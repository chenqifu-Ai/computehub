## 三、🚨 事故与教训

### 🔴 WANLIDA-ACCIDENT-002 — wanlida-temp 自动升级掉线（6/19 03:01）

**严重程度**: 节点离线，需人工物理恢复

**时间线**:
```
22:10 首次检测到升级（v1.3.35 → v1.3.38）
22:15~02:55 50+次升级检查，全部 skip
03:00 SHA256 变化（Gateway 可能重启）
03:01 最后一次成功注册
03:04 IP 冲突驱逐（wanlida-temp ↔ phone-agent 抢注册）
03:05 升级触发 → 旧 Worker 死 → 新 binary 起不来 → ❌ 变砖
03:06 HealthMonitor 标记 OFFLINE
```

**根因**: 升级即自杀模式 — 旧 Worker 先死，新 Worker 再跑。如果新 binary 是坏的，没人验证，没人回滚。

**核心铁律**:
> 🚫 **绝对禁止 Windows Worker 自动升级**
> 必须先下载→SHA256验证→确认版本→再kill旧进程
> 旧 Worker 必须作为"守护者"活着，直到新 Worker 确认上线

**文档**: `memory/topics/经验总结/WANLIDA-ACCIDENT-002_wanlida-temp自动升级掉线复盘.md`

### 🟡 6/23 节点排查踩坑
