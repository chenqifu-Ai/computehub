## 🚀 ComputeHub v0.7.6 Gateway 下载端点 (08:35)

### 发现：Gateway 自带下载端点
- **端点**: `GET /api/v1/download?file=<文件名>`
- **作用**: 直接从 Gateway 部署目录提供二进制文件下载，无需额外 HTTP 服务
- **端口**: 8282

### 发现问题
- 旧版 Gateway 的 `allowedPrefixes` 没有 `computehub-` 前缀
- 新版 (v0.7.6) 已包含: `"compute-worker-"`, `"compute-gateway-"`, `"compute-tui-"`, `"computehub-tui"`, `"computehub-"`, `"opc-"`
- 旧版请求返回 `file not allowed`

### 解决方案
- 用新版二进制重启 Gateway：`bin/linux-arm64/computehub-gateway --port 8282 ...`
- ✅ 验证成功：9.5MB Windows 文件可正常下载

### 给老大的 Windows 下载命令
```cmd
curl -sL "http://192.168.1.7:8282/api/v1/download?file=computehub-gateway-windows-amd64.exe" -o D:\computehub\computehub-gateway.exe
curl -sL "http://192.168.1.7:8282/api/v1/download?file=computehub-worker-windows-amd64.exe" -o D:\computehub\computehub-worker-windows-amd64.exe
curl -sL "http://192.168.1.7:8282/api/v1/download?file=computehub-tui-windows-amd64.exe" -o D:\computehub\computehub-tui-windows-amd64.exe
```

### 相关文件
- `projects/computehub/src/gateway/gateway.go` → `handleFileDownload()` 函数
- `deploy/` → 各平台二进制文件
- `deploy/computehub-gateway-windows-amd64.exe` (9.5MB, 已就绪)

## 🔧 ComputeHub 结果回传 Bug 修复 (11:00-12:00)

### 问题
- 短任务（快速完成的 Python/shell 命令）通过 `/api/v1/tasks/progress` 查询时 stdout/stderr 始终为空
- 原因是 Worker 只通过 `sendStreamProgress()` 推流式输出，但短任务完成太快，来不及推流
- 最终结果通过 `submitTaskResult()` 存到 `ts.Result`，但 `GetTaskStream()` 只读 `ts.StreamStdout`/`ts.StreamStderr`

### 修复
- **文件**: `src/kernel/actions.go` → `GetTaskStream()`（~line 210）
- **改动**: 当 `ts.StreamStdout`/`ts.StreamStderr` 为空且 `ts.Result` 存在时，回退返回 `ts.Result.Stdout`/`ts.Result.Stderr`
- **提交命令用 `command` 字段**（payload 不映射到 Worker 的执行命令，Gateway 侧有 `payload → command` 映射但 API 调用需传 `command`）

### 验证
- 提交 `echo hello-computehub && uname -a` 到 worker-localhost
- ✅ progress 返回 stdout + exit_code=0 + duration=154ms
- ✅ detail 也有完整内容

### 编译 & 部署
- 全平台重新编译（linux-arm64 6.7MB，其他 11MB）
- `sync-deploy.sh` 同步到 `deploy/`
- 本地 Gateway 重启用新二进制
