# 2026-06-08 工作日志

## 📦 v1.3.19 — 两个线上 Bug 修复

### Bug 1: ARM64 Worker 下载 x86 binary → exec format error
- **根因**: `DownloadWithChecksum()` 和 `fetchSHA256()` 下载 URL 没传 platform 参数，Gateway 返回默认 amd64
- **修复**: 下载 URL + SHA256 查询都加 `&platform=linux/arm64`，Gateway 校验 API 解析 platform 做架构感知匹配
- **commit**: `c039bee` + `c1e69f8`

### Bug 2: local-arm Worker SIGSYS crash (faccessat2)
- **现象**: v1.3.17→v1.3.19 升级过程中，HallClient 收到消息后触发 Agent → `exec.Command("bash")` → `LookPath` → `faccessat2` → SIGSYS
- **教训**: v1.3.13 SafeCommand 只修了 `upgrade_manager.go`，漏了 `worker_agent.go` 和 `worker_kernel.go` 两处
- **修复**: 两处都改 `executil.SafeLookPath("bash"/"sh")` → 绝对路径
- **commit**: `b2d8ed7`

### 其他

## 🧠 经验教训

1. **SafeCommand 扫尾不干净** — 下次类似全面排查要全项目 grep，不只是改发现问题的文件
2. **升级链消息触发 Agent** — UpgradeManager 完成 → HallClient 通知 → Agent 执行 shell，这个链上的每个 exec.Command 都要排查
3. **Android seccomp 感知** — Go 1.26 对 `exec.Command` bare name 强制走 faccessat2，Termux proot 必崩
4. **ECS 双目录问题** — `ComputeHub/` 和 `OPC/` 两个目录都有 deploy，systemd 指向 `ComputeHub/deploy/`，升级时要确认正确路径
