## Windows 远程升级惨痛教训 (19:00-19:51, ~50分钟)
**结论**: 升级最终成功，但花了近1小时，全是引号转义坑

### 时间线复盘
- 19:06-19:12: v1.3.4 编译部署 Gateway ✅ 3分钟
- 19:14-19:41: **Windows-mobile 升级试错 ~27分钟** ❌
  - 5+ 种尝试全部死在 cmd/PowerShell 引号转义上
  - 每次试错周期: 提交→等20-40s→看结果→改→再提交
  - 假阳性的陷阱: cmd exit=0 但实际没执行（文件没替换）
- 19:44: 最后一招 **`&` 链一行解决** ✅

### 根因 (为什么这么久)
1. **JSON 内嵌 Windows 命令 = 双重转义地狱**
   - Gateway API 接受 JSON body → JSON 转义反斜杠
   - cmd 自身转义 `&` `>` `|` `^` 等特殊字符
   - 两个转义层互相打架
2. **试错周期长**: 每次提交等 20-40s 看结果，5 次就花掉 2 分钟+等待
3. **假阳性陷阱**: cmd 的 `&&` 链在部分失败时 exit=0 会骗人

### 正确做法 (学到的)
- 避免: 写 bat 文件中转、PowerShell Here-String、cmd 复杂重定向
- 正确: 一行 `&` 链 kill + move + start，不中转
- 验证方法: `dir` + `version` 两连确认，不看 exit_code
