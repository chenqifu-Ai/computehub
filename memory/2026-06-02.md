## 🐛 修复：ActiveTasks 双倍递增 Bug (10:08-10:18)

**问题**: 任务全完成后 ecs-p2ph 显示 `7/8 ████████░░`
- 昨天跑的 7 个任务都完成了，但 ActiveTasks 仍显示 7

**根因**: `gateway_worker.go` 里的 `findPendingTaskForNode()`
- `SubmitTask()` 创建任务时已 `ActiveTasks++`（标记为 `pending`）
- Worker 轮询时 `pending→running` 又 `ActiveTasks++` ❌
- `CompleteTask()` 只 `ActiveTasks--`
- 净效果：**每个任务完成后多 +1**，7个任务多7

**修复**: 删掉 `gateway_worker.go:74` 那行重复的 `state.Metrics.ActiveTasks++`

**部署**: 编译 linux-amd64 → SCP 到 ECS → 重启 Gateway+Worker → 确认 active_tasks=0 ✅

## 🚀 Windows 节点升级 (10:28-11:00)

**目标**: 将 Windows-mobile 从旧 WMI agent 升级为统一 binary `computehub.exe worker --agent`

**步骤**:
1. **编译**: `CGO_ENABLED=0 GOOS=windows GOARCH=amd64` → `computehub-windows-amd64.exe` (10.6MB, v1.3.1)
2. **更新 deploy/**:
   - 复制 `computehub.exe` → `deploy/` + `deploy/windows-worker/` + `deploy/windows-amd64/`
   - 复制 `computehub-linux-amd64` → `deploy/` + `deploy/linux-amd64/`

   - 更新 `version.txt` → `1.3.1`

   - 难点：cmd 的 `&` 会把 URL参数截断，需用引号包裹 URL

   - 10.6MB 下载成功 ✅
   - 版本验证 `v1.3.1` ✅
4. **替换&重启**: 逐个步骤提交任务（避免 `&&` 逃逸问题）

5. **验证**: `platform=windows/amd64` ✅，新 Worker 自带自动升级机制
