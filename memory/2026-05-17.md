# 2026-05-17 日志

## 🔧 ECS 服务器故障排查

### 问题
SSH 连接 `36.250.122.43` 超时（Connection timed out）
同时 ping、网关端口全不通

### 排查过程
1. ❌ 误判 fail2ban → IP 未被封
2. ❌ SSHD 正常、端口正常监听
3. ✅ **真凶：iptables INPUT 策略 DROP**
4. ✅ 添加放行规则后连通，但 SSH 密钥认证失败
5. ✅ 添加 Ed25519 公钥到 authorized_keys → 完全恢复

### 修复命令
```bash
# iptables
iptables -I INPUT 1 -m state --state ESTABLISHED,RELATED -j ACCEPT
iptables -I INPUT 1 -p tcp --dport 22 -j ACCEPT
mkdir -p /etc/iptables && iptables-save > /etc/iptables/rules.v4

# SSH 密钥
echo 'ssh-ed25519 ...' >> /home/computehub/.ssh/authorized_keys
chown computehub:computehub ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys

# fail2ban 调优
fail2ban-client set sshd bantime 300
fail2ban-client set sshd addignoreip 112.48.49.13
```
