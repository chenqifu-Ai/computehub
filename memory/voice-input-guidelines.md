# 语音输入处理规范

**创建时间**: 2026-03-24
**用户**: 老大
**输入方式**: 语音转文字
**用户背景**: 闽南人，普通话不太标准

---

## 一、背景说明

用户使用语音输入，文字是语音转写的。由于用户是**闽南人**，普通话不标准，可能出现：
- 闽南语发音习惯导致的识别偏差
- 同音字错误（"陪审" → "Python"）
- 发音相近的词被误识别
- 方言或口音导致的识别偏差

### 闽南语发音特点
- 某些音节发音不同
- 可能夹杂闽南语词汇
- 声调和标准普通话有差异
- 常见：f/h 不分、zh/z 不分等

---

## 二、识别规则

### 重要说明
- 用户使用语音输入，**不会修改转写的文字**
- 遇到发音问题，直接根据上下文理解
- 不确定的词，先推测，实在不行再问
- 通过长期积累理解发音规律

### 规则1：遇到奇怪关键词时
**立即检查**：
1. 该词是否在技术/编程领域常用？
2. 尝试**发音读出来**，联想可能的同音词
3. 结合上下文判断真实意思

### 规则2：优先级处理
1. **先查发音映射表**（见下方）
2. 再结合上下文
3. 不确定时**立即询问**用户确认

### 规则3：记录新映射
遇到新的发音错误时：
1. 立即记录到映射表
2. 标注首次出现时间
3. 添加必要的备注

## 四、处理流程

```
收到消息
    ↓
遇到不理解的关键词？
    ↓ 是
查发音映射表
    ↓
找到映射？
    ↓ 是              ↓ 否
使用映射意思    发音读出来
    ↓                ↓
继续处理      联想到可能词？
                 ↓ 是        ↓ 否
              使用推测    询问用户确认
```

## 六、示例

### 示例1
**用户说**："让AI陪审来做事"
**分析**：
- "陪审"是法律术语，但这里不像
- 发音读出来："péi shěn"
- 联想：Python？陪审？
- 上下文：AI + 做事
- **判断**：陪审 = Python

### 示例2
**用户说**："用那个什么框架"
**分析**：
- 不确定具体框架
- 不确定时**立即询问**
