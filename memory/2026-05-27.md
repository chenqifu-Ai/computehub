## 13:00-14:00 🛠️ Windows Worker 自动升级故障修复

### 事件
老大截图显示 Windows Worker 升级失败。Gateway 13:22 有段时间重启，Windows Worker 报 "No connection"。
之后 Worker 下载新版本时失败，404 Not Found。

### 根因分析（两问题叠加）
1. **Gateway 端口冲突重启**：旧编译方式（`cmd/gateway/`）不包含 worker 子命令。运行时 `computehub worker` 实际上启动了第二个 gateway，端口冲突，Gateway 反复重启。
2. **二进制命名不一致**：Gateway `upgradeBinaryName()` 返回 `computehub-worker-win-amd64.exe`，Worker `workerExecutable()` 返回 `computehub-worker-amd64.exe`，Worker 的 `strings.Contains(URL, expectedBinary)` 校验不通过，直接 `return fmt.Errorf("平台不匹配")` 退出进程。

### 修复措施
- **源码修复**：`gateway_upgrade.go` + `auto_upgrade.go` 统一命名 → `computehub-worker-amd64.exe`（去掉 `win-` 前缀）
- **新增标准**：`WIN-BIN-001_Windows二进制命名一致性标准.md` — Gateway / Worker 代码 / Deploy 目录三处命名必须一致
- **新增标准**：`WIN-DBG-001_Windows节点截图调试标准流程.md` — 老大截图发来 → 小智分析 → 直接修
- **合版编译修复**：之前误用 `cmd/gateway/` 编译（只有 gateway 无 worker 子命令），改回 `cmd/computehub/` 编译合版 binary
