## 🧠 经验：Base64 绕过 JSON 转义地狱（老大指定标准 WIN-COM-002）
**问题**: JSON API 里传 Windows cmd 命令，`\`、`"`、`&` 等特殊字符导致 JSON 解析失败
**解法**: Base64 编码命令字符串 → 通过 JSON API 传输 → 目标端 PowerShell iex 解码执行

**标准流程**:
```python
# 发送端: cmd 原样写，base64 编码
cmd = 'taskkill /F /IM computehub.exe /T'
b64 = base64.b64encode(cmd.encode()).decode()

payload = json.dumps({
    "task_id": "xxx",
    "node_id": "windows-node",
    "command": f'powershell -Command "$b=\'{b64}\';$d=[System.Text.Encoding]::UTF8.GetString([System.Convert]::FromBase64String($b));iex $d"'
})
```

**注意事项**:
- cmd 命令太长可能被 `iex` 截断（PowerShell 命令长度限制）
- 替代方案: certutil base64 decode → 写 .bat 文件 → 调度执行
- 经典组合: `schtasks` 延迟执行 + `.bat` 文件，绕过当前 Worker 进程被杀的问题
