## 重要发现：阿里百炼 Qwen 模型 Token 浪费问题

### 问题
阿里百炼的 Qwen 系列模型（qwen3.5-plus, qwen3.5-flash, qwen3-max, qwen3-coder-next）**默认开启 thinking 推理模式**，导致简单问题也会输出大量思考过程，浪费约 90% 的 tokens。

### 实测数据
| 模型 | 问题 | 总 tokens | 推理 tokens | 浪费比例 |
|------|------|----------|------------|---------|
| qwen3.5-plus | "用一句话介绍你自己" | 235 | 210 | 89% |
| qwen3.5-flash | "hi" | 417 | 390 | 93% |

### 解决方案
在模型配置中添加 `defaultParams: {"thinking": {"enabled": false}}`

### 配置修复
文件：`~/.openclaw/openclaw.json`

所有 modelstudio provider 下的模型都已添加禁用 thinking 配置。

### 建议
- 日常使用 → `ollama-cloud/glm-5:cloud`（不浪费 tokens）
- 需要 Qwen 能力 → `modelstudio/qwen3.5-plus`（已配置禁用 thinking）

## 18:28 OpenClaw 配置修复

### 问题
嵌入式 Agent 运行 `glm-5:cloud` 模型时报 `terminated` 错误。

### 根因
模型 ID 不匹配：
- `agents.defaults.models` 引用: `ollama-cloud/glm-5:cloud`
- `ollama-cloud` provider 中定义: `glm-5`（缺少 `:cloud` 后缀）

### 修复
修改 `~/.openclaw/openclaw.json` 中以下模型的 ID：
- `glm-5` → `glm-5:cloud`
- `kimi-k2.5` → `kimi-k2.5:cloud`
- `minimax-m2.5` → `minimax-m2.5:cloud`

### 工具警告
`apply_patch`, `memory_search`, `memory_get` 在 coding profile allowlist 中配置，但当前 runtime 不支持这些 core tools。这只是警告，不影响正常运行。
