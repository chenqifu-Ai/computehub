## 🧠 ComputeHub 源码分析（05:46-05:49）

### 关键文件清单
- `main.go` — Gateway 入口，加载配置、初始化 Composer/Gateway
- `cmd/` — 4 个入口：`computehub`（全功能）、`gateway`、`worker`、`tui`、`node`

- `src/kernel/kernel.go` — 线性化指令队列 + 状态机，防竞态
- `src/composer/composer.go` — TaskComposer：大模型拆任务 → 小模型并行干 → 大模型汇总
- `src/scheduler/scheduler.go` — 调度器：熔断器、抢占式调度、区域感知
- `src/executor/` — 沙箱执行环境
- `src/workercmd/` — Worker 节点逻辑
- `src/gateway/gallery.go` — 作品展示/Gallery
- `src/gateway/video.go` — 视频处理
- `src/gateway/auth.go` — 鉴权
- `src/pure/pipeline.go` — 纯管线处理
- `src/visualizer/` — Prometheus 可视化
- `src/discover/` — 节点自动发现
- `src/health/` — 健康检查
- `src/gene/` — 状态/基因存储
- `Makefile` — 构建系统：本地编译/交叉编译/部署

### 核心技术
- **Composer 模式**：大模型（qwen3.6-35b-common）拆任务，小模型并行执行，大模型汇总
- **NewAPI 集成**：默认 API URL `https://ai.zhangtuokeji.top:9090/v1`，Key `sk-3RgMq1COL9uqn29hCBwXOt5X3d5TpIddaRKH44chQ2QcAybl`
- **Kernel 线性化**：确保多节点并行下的指令顺序一致性
- **Scheduler**：含熔断器、延迟感知负载均衡、区域评分
- **交叉编译**：`make linux-amd64` / `make linux-arm64` / `make release`

### 项目本质
AI Agent 分布式计算平台 — 用大模型拆任务、多节点并行执行、结果合并的完整系统。
