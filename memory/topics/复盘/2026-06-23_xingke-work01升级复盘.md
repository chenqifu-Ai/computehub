## 关键经验

### 1. exit_code=7 永远是网络问题
curl exit_code=7 = "Failed to connect to host"。**不是命令问题**。下次再遇到，等15s重试，不改方向。

### 2. 方案对了就不换
文件传JSON是对的方案（避开所有转义），失败是因为节点掉线。如果当时停下来查nodes/list，就不会白折腾。

### 3. 语法探测先于一切
PowerShell包装的Worker（out-file噪音、&被吞）和直接cmd的Worker，命令格式完全不同。不做探测直接猜 = 浪费时间。

### 4. 节点掉线≠命令不行
"assigned node not registered" 是节点状态问题，不是命令格式问题。

### 5. kill用同机新Worker
不同Worker在同一台机器时，通过新Worker杀旧Worker的PID最可靠（不走中转节点）。
