# 图片识别操作规范 (IMG-REC-001)

> **制定日期**: 2026-04-29
> **适用范围**: AI 智能体（小智）执行图片识别任务
> **版本**: v1.0

---

## 一、核心方式：直接调 qwen3.6-35b API

**不用当前 session 模型！**

当前 session 的默认模型（ollama-cloud-2/deepseek-v4-flash）**不支持图片输入**。所有图片识别必须：

1. **直接** 调用 qwen3.6-35b API（`http://58.23.129.98:8001/v1`）
2. 将图片编码为 **base64**
3. 通过 `image_url` 字段传入 `data:image/...;base64,...` 格式
4. API 返回后，从 reasoning 字段读取结果

> 💡 简单说：**调用 qwen3.6-35b → base64 传图 → 读 reasoning**

---

## 二、概述

当用户要求识别图片内容时，AI 必须按本规范的标准流程操作，确保一致性、准确性和效率。

## 三、标准操作流程（7 步）

### 第 1 步：确认图片存在

```bash
# 检查文件是否存在
ls -la <图片路径>

# 检查文件类型
file <图片路径>
```

**预期输出示例：**
```
-rw------- 1 root root 62129 Apr 29 14:54 /path/to/image
/path/to/image: PNG image data, 1866 x 663, 8-bit/color RGB, non-interlaced
```

**关键检查项：**
- [ ] 文件是否存在
- [ ] 文件是否为图片格式（PNG / JPG / JPEG / WEBP / GIF）
- [ ] 记录图片尺寸（宽×高）

### 第 2 步：获取图片大小

```python
# 计算文件大小（KB）
import base64
with open(img_path, "rb") as f:
    b64 = base64.b64encode(f.read()).decode()
file_size = len(base64.b64decode(b64)) / 1024
print(f"图片大小: {file_size:.0f} KB")
```

**注意：** 单张图片 base64 后的请求大小建议控制在 **5MB 以内**，过大的图片会导致 API 超时或 token 浪费。

### 第 3 步：构建 API 请求

**endpoint 地址：** `http://58.23.129.98:8001/v1/chat/completions`

**请求格式：**
```python
payload = {
    "model": "qwen3.6-35b",
    "messages": [{"role": "user", "content": [
        {"type": "text", "text": "请非常详细地描述这张图片里的所有内容。如果有文字，请完整读出所有文字内容。"},
        {"type": "image_url", "image_url": {"url": f"data:image/png;base64,{b64}"}}
    ]}],
    "max_tokens": 4096,
    "temperature": 0.3
}
```

**参数说明：**
| 参数 | 推荐值 | 说明 |
|------|--------|------|
| model | `qwen3.6-35b` | 固定，唯一支持图片输入的模型 |
| max_tokens | 2048~4096 | 根据图片复杂度调整，含大量文字时用 4096 |
| temperature | 0.3 | 低温度确保描述准确客观 |
| timeout | 60~180s | 大图片/复杂图片需要更长推理时间 |

### 第 4 步：设置 prompt 模板

根据场景选择 prompt 模板：

**通用模板（默认）：**
```
请非常详细地描述这张图片里的所有内容。
如果有文字，请完整读出所有文字内容。
描述画面、颜色、布局、物体等所有细节。
```

**数据分析/图表模板：**
```
请详细分析这张图表/截图的内容。
- 如果有坐标轴，请描述坐标含义
- 如果有数据趋势，请描述变化规律
- 如果有文字，请完整读出
```

**产品/物品识别模板：**
```
请详细描述这张图片中的物品：
- 外观、颜色、形状
- 上面的所有文字/标签
- 品牌、型号（如果有）
```

**屏幕截图模板：**
```
请描述这张截图的内容：
- 这是什么界面/应用
- 有哪些功能按钮和文字
- 当前状态和数据
- 时间、数字等重要信息
```

### 第 5 步：发送请求并获取结果

```python
start = time.time()
r = requests.post(url, json=payload, headers=headers, timeout=180)
elapsed = time.time() - start
data = r.json()

content = data.get('choices',[{}])[0].get('message',{}).get('content','')
reasoning = data.get('choices',[{}])[0].get('message',{}).get('reasoning','')
tokens = data.get('usage',{}).get('total_tokens','?')

print(f"⏱ {(elapsed*1000):.0f}ms | HTTP {r.status_code} | tokens={tokens}")
```

### 第 6 步：读取结果

**注意：** qwen3.6-35b API 的 content 字段始终为 null，所有输出在 reasoning 字段。

```python
if content:
    result = content
elif reasoning:
    result = reasoning   # 从 reasoning 字段读取
else:
    result = "❌ 无输出"
```

### 第 7 步：组织回复给用户

回复格式应包含：
1. ✅ 确认已分析图片
2. 🔑 提炼关键信息（用表格/列表）
3. 📝 完整描述（如有必要）

## 三、完整代码模板

```python
import requests, json, time, base64

img_path = "<图片路径>"
url = "http://58.23.129.98:8001/v1/chat/completions"
headers = {
    "Content-Type": "application/json",
    "Authorization": "Bearer sk-78sadn09bjawde123e"
}

# 读取并编码图片
with open(img_path, "rb") as f:
    b64 = base64.b64encode(f.read()).decode()

# 构建请求
payload = {
    "model": "qwen3.6-35b",
    "messages": [{"role": "user", "content": [
        {"type": "text", "text": "请详细描述这张图片的所有内容"},
        {"type": "image_url", "image_url": {"url": f"data:image/png;base64,{b64}"}}
    ]}],
    "max_tokens": 4096,
    "temperature": 0.3
}

# 发送
start = time.time()
r = requests.post(url, json=payload, headers=headers, timeout=180)
data = r.json()

# 获取结果
content = data.get('choices',[{}])[0].get('message',{}).get('content','')
reasoning = data.get('choices',[{}])[0].get('message',{}).get('reasoning','')
result = content or reasoning or "❌ 无输出"
```

## 五、注意事项

### ⚠️ 已知问题
1. **content 字段始终为空** — qwen3.6-35b API 所有输出在 `reasoning` 字段
2. **sharp 模块在 Termux 不可用** — dlopen libvips 失败，无法在本地用 Node.js 读取图片
3. **必须走 API** — 图片分析必须通过 qwen3.6-35b API 的 base64 图片输入方式

### ⚠️ 限制
| 项 | 说明 |
|----|------|
| 最大图片 | 建议单张 base64 编码后 < 5MB |
| 超时 | 大图片建议 timeout=180s |
| token | 复杂图片建议 max_tokens=4096 |
| 不支持 PDF | 需要先截图/转换为图片 |
| 不支持 video | 仅静态图片 |

### ⚠️ 不要做什么
- ❌ **不要用当前 session 模型（ollama-cloud-2/deepseek-v4-flash）去分析图片** — 它不支持图片输入
  - ✅ **正确做法：直接调 qwen3.6-35b API（58.23.129.98:8001）走 base64 传图**
- ❌ 不要尝试在当前系统安装 sharp / libvips — Termux 环境不支持
- ❌ 不要问用户"我能看图片吗" — 直接用 API 走 base64 分析
