## 一、配置架构总览

OpenClaw 模型配置分 **三层**，任何修改必须保证三层链路完整一致：

```
Layer 1: models.providers      ← 物理连接（URL + API Key + 模型定义）
         ↓
Layer 2: agents.defaults.model.primary  ← 默认模型指向
         ↓
Layer 3: agents.defaults.models          ← 别名映射（alias）
```

### 1. Provider 层（物理连接）

```json
{
  "models": {
    "providers": {
      "provider_name": {
        "baseUrl": "https://...",
        "apiKey": "sk-xxx",
        "api": "openai-completions",
        "models": [
          {
            "id": "model-id",
            "name": "Model Name",
            "reasoning": false,
            "maxTokens": 8192,
            "contextWindow": 256000
          }
        ]
      }
    }
  }
}
```

**关键点：**
- `provider_name` 是别名映射中的第一级（`provider_name/model_id` 中的 provider_name）
- `models[].id` 是别名映射中的第二级
- 每个 provider 是一个独立的 API 端点 + Key

### 2. Agent 层（默认 + 别名）

```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "provider_name/model_id"    // 默认模型
      },
      "models": {
        "provider_name/model_id": {
          "alias": "快捷别名"     // 可选，方便使用
        }
      }
    }
  }
}
```

**关键点：**
- `primary` 的值格式：`provider_name/model_id`
- 别名映射的 key 必须是 `provider_name/model_id` 格式
- key 中的 provider_name 必须在 `models.providers` 中存在
- key 中的 model_id 必须在对应 provider 的 `models` 列表中存在

### 3. 别名完整性约束（v2.0 新增 ⭐）

**核心规则：Provider 层注册的每个模型，必须至少在 Alias 层注册一次。**

- **主别名**: 常用模型在 `agents.defaults.models` 中配置 `alias` 字段
- **隐式注册**: 冷门模型在 `agents.defaults.models` 中配置空对象 `{}`
- **不允许**: Provider 有模型，但 Alias 层完全不存在对应条目

**为什么需要这个规则？**

v2.0 前版本（2026-06-11 之前）的问题：手动添加 alias 时只配了常用模型，冷门模型在 provider 层注册了但 alias 层没有，导致在 `/model` 列表里看不到。实际验证发现 **zhangtuo-ai 有 40 个模型，但 alias 层只有 20 个条目**，50% 不可见。

## 二、完整修改流程（标准操作 SOP）

### 第 0 步：全量扫描（v2.0 新增 ⭐）

**任何修改模型配置前，必须先运行全量扫描，确认三层链路完整性。**

```bash
python3 << 'EOF'
import json

with open('/root/.openclaw/openclaw.json') as f:
    config = json.load(f)

providers = config['models']['providers']
agent_models = config['agents']['defaults']['models']

print("=== Provider 模型扫描 ===")
issues = []
for prov_name, prov in providers.items():
    model_ids = {m['id'] for m in prov.get('models', [])}
    aliased = {k.split('/')[1] for k in agent_models.keys() if k.startswith(f'{prov_name}/')}
    
    missing_in_alias = model_ids - aliased
    extra_in_alias = aliased - model_ids
    
    print(f"\n{prov_name}: {len(model_ids)} models, {len(aliased)} aliased")
    if missing_in_alias:
        issues.append(f"  ❌ {prov_name}: {len(missing_in_alias)} 个模型未在 alias 层注册: {sorted(missing_in_alias)}")
        for m in sorted(missing_in_alias):
            print(f"  ❌ 缺失: {m}")
    if extra_in_alias:
        issues.append(f"  ⚠️ {prov_name}: {len(extra_in_alias)} 个别名指向不存在的模型: {sorted(extra_in_alias)}")
        for m in sorted(extra_in_alias):
            print(f"  ⚠️ 多余: {m}")

# 检查废弃别名
print("\n=== 废弃别名检查 ===")
for full_path, info in agent_models.items():
    if 'alias' in info:
        alias = info['alias']
        # 检查 alias 是否指向已删除的 provider 中的模型
        provider_name, model_id = full_path.split('/', 1)
        if provider_name not in providers:
            issues.append(f"  ❌ [{alias}] 指向不存在的 provider: {full_path}")

if issues:
    print("\n=== ⚠️ 发现问题 ===")
    for issue in issues:
        print(issue)
else:
    print("\n✅ 链路完整，无问题")
EOF
```

**扫描结果判定：**
- ✅ 无输出 → 链路完整
- ⚠️ 有 "多余" 条目 → 别名指向不存在的模型，需清理
- ❌ 有 "缺失" 条目 → 必须补上，否则这些模型不可见

### 第 1 步：确认目标 provider 和 model

明确：
1. **哪个 provider**（必须已存在于 `models.providers`）
2. **哪个 model id**（必须存在于该 provider 的 `models` 列表）
3. **要设为什么别名**（如 `zhangtuo-common`）
4. **是否要改 primary**
5. **是否新增 provider**（新增 provider 时，必须同步添加 alias 注册）

### 第 2 步：执行修改

```python
import json
with open('/root/.openclaw/openclaw.json') as f:
    config = json.load(f)

# === 场景 A: 修改 primary ===
config['agents']['defaults']['model']['primary'] = 'new_provider/new_model'

# === 场景 B: 添加/修改别名 ===
config['agents']['defaults']['models']['new_provider/new_model'] = {
    'alias': 'new_alias'
}

# === 场景 C: 删除别名 ===
if 'old_provider/old_model' in config['agents']['defaults']['models']:
    del config['agents']['defaults']['models']['old_provider/old_model']

# === 场景 D: 注册冷门模型（v2.0 新增 ⭐）===
# 如果 provider 新增了模型但 alias 层没有注册，必须补上
# 即使是冷门模型，也至少注册一个空条目
config['agents']['defaults']['models']['provider/cold_model'] = {}

with open('/root/.openclaw/openclaw.json', 'w') as f:
    json.dump(config, f, indent=2, ensure_ascii=False)
```

### 第 3 步：验证链路（必须执行！）

```bash
python3 << 'EOF'
import json

with open('/root/.openclaw/openclaw.json') as f:
    config = json.load(f)

providers = config['models']['providers']
agent_models = config['agents']['defaults']['models']

print("=== 别名链路验证 ===")
all_ok = True
for full_path, info in agent_models.items():
    if 'alias' not in info:
        continue
    alias = info['alias']
    provider_name = full_path.split('/')[0]
    model_id = full_path.split('/')[1]
    
    if provider_name not in providers:
        print(f"  ❌ [{alias}] provider \"{provider_name}\" NOT FOUND!")
        all_ok = False
    else:
        provider = providers[provider_name]
        model_found = any(m['id'] == model_id for m in provider['models'])
        if model_found:
            reasoning = next((m.get('reasoning', False) for m in provider['models'] if m['id'] == model_id), False)
            print(f"  ✅ [{alias}] -> {full_path} (reasoning={reasoning})")
        else:
            print(f"  ❌ [{alias}] model \"{model_id}\" NOT in provider!")
            all_ok = False

# 验证 primary
print(f"\n=== Primary 验证 ===")
primary = config['agents']['defaults']['model']['primary']
prov, model = primary.split('/', 1)
if prov in providers and any(m['id'] == model for m in providers[prov]['models']):
    print(f"  ✅ primary: {primary}")
else:
    print(f"  ❌ primary: {primary} NOT VALID!")
    all_ok = False

print(f"\n{'✅ 所有链路验证通过！' if all_ok else '❌ 存在错误链路，请修正！'}")
EOF
```

### 第 4 步：验证 alias 完整性（v2.0 新增 ⭐）

**修改后必须执行此步骤，确保没有漏掉冷门模型。**

```bash
python3 << 'EOF'
import json

with open('/root/.openclaw/openclaw.json') as f:
    config = json.load(f)

providers = config['models']['providers']
agent_models = config['agents']['defaults']['models']

print("=== Alias 完整性检查（v2.0） ===")
all_ok = True
for prov_name, prov in providers.items():
    model_ids = {m['id'] for m in prov.get('models', [])}
    aliased = {k.split('/')[1] for k in agent_models.keys() if k.startswith(f'{prov_name}/')}
    
    missing = model_ids - aliased
    if missing:
        print(f"  ❌ {prov_name}: {len(missing)} 个模型未在 alias 层注册")
        all_ok = False
    else:
        print(f"  ✅ {prov_name}: {len(model_ids)} models, 全部注册")

print(f"\n{'✅ 别名完整性验证通过！' if all_ok else '❌ 存在未注册的模型！'}")
EOF
```

## 三、别名注册规范

### 注册原则

| 模型类型 | 注册方式 | 示例 |
|---------|---------|------|
| **主力模型** | `{"alias": "简短别名"}` | `{"alias": "ds-v4-flash"}` |
| **常用模型** | `{"alias": "别名"}` | `{"alias": "kimi-k2.6"}` |
| **冷门模型** | `{}`（空对象） | 无别名但条目存在 |
| **废弃模型** | 从 alias 层 **删除** | `qwen3.6-35b-common` 废弃后删除 |

### 别名命名规范

- **简短清晰**：不超过 20 个字符
- **无歧义**：同一 provider 内别名不重复
- **可读**：使用模型名缩写或全称
- **避免特殊字符**：不用 `-`, `.` 等可能引起问题的字符

### 自动补全算法（v2.0 新增 ⭐）

当新增 provider 或新增模型时，如果 alias 层缺失，按以下规则补全：

```python
import json

with open('/root/.openclaw/openclaw.json') as f:
    config = json.load(f)

providers = config['models']['providers']
agent_models = config['agents']['defaults']['models']

# 自动补全缺失的 alias 注册
added = 0
for prov_name, prov in providers.items():
    for model in prov.get('models', []):
        model_id = model['id']
        full_path = f'{prov_name}/{model_id}'
        
        if full_path not in agent_models:
            agent_models[full_path] = {}
            added += 1

if added:
    print(f"✅ 自动补全 {added} 个缺失的 alias 注册")
    with open('/root/.openclaw/openclaw.json', 'w') as f:
        json.dump(config, f, indent=2, ensure_ascii=False)
else:
    print("✅ 无缺失，别名完整性完好")
```

## 四、常见错误模式

### ❌ 错误模式 1：别名中的 provider 不存在

```json
// 错误：zhangtuo-ai provider 存在，但别名指向 zhangtuo-ai/provider（不存在）
"zhangtuo-ai/qwen3.6-35b-common": { "alias": "zhangtuo-common" }
```

**修正：** 检查 provider 名称是否精确匹配
```json
"zhangtuo-ai-common/qwen3.6-35b-common": { "alias": "zhangtuo-common" }
```

### ❌ 错误模式 2：别名中的 model id 不存在

```json
// 错误：model id 拼写错误或不在 provider 的 models 列表中
"some-provider/wrong-model-id": { "alias": "test" }
```

**修正：** 确认 model id 与 provider 定义完全一致

### ❌ 错误模式 3：alias 层缺失冷门模型（v2.0 修复 ⭐）

```json
// 问题：provider 有 40 个模型，但 alias 层只注册了 20 个
// 结果：20 个冷门模型在 /model 列表中不可见
```

**修正：** 使用自动补全算法或手动补全缺失的 alias 注册

### ❌ 错误模式 4：废弃别名未清理

```json
// 错误：模型已废弃，但别名映射还残留
"zhangtuo-ai/qwen3.6-35b-common": { "alias": "zhangtuo-common" }
// → qwen3.6-35b-common 已不在 provider models 列表中
```

**修正：** 模型废弃后，必须从 alias 层同步删除

### ❌ 错误模式 5：primary 指向不存在的 provider

```json
// 错误：primary 中的 provider 未在 models.providers 中定义
"primary": "non-existent-provider/model"
```

### ❌ 错误模式 6：provider 层重复定义

```json
// 多个 provider 使用相同的 baseUrl 和 key，容易混淆
"zhangtuo-ai": { "baseUrl": "https://a.com", ... }
"zhangtuo-ai-common": { "baseUrl": "https://a.com", ... }
```

**说明：** 同一个 API 端点可以注册多个 provider，只要 model id 不同就行。但要确保别名正确指向对应 provider。

## 五、修改前检查清单

修改任何模型配置前，先确认：

- [ ] 运行全量扫描（第 0 步），确认当前链路完整性
- [ ] 目标 provider 存在于 `models.providers`
- [ ] 目标 model id 存在于对应 provider 的 `models` 列表
- [ ] reasoning 字段是否正确设置
- [ ] maxTokens / contextWindow 是否合理
- [ ] API Key 是否正确且未过期
- [ ] 别名是否与其他别名冲突
- [ ] primary 指向是否符合预期

## 七、运维操作

### 定期维护（每月一次）

1. 运行全量扫描，检查是否有遗漏
2. 清理废弃别名（模型已从 provider 删除的）
3. 自动补全缺失的 alias 注册
4. 检查 alias 命名是否规范

### 快速诊断命令

```bash
# 一键诊断所有 provider 的 alias 状态
python3 << 'EOF'
import json

with open('/root/.openclaw/openclaw.json') as f:
    config = json.load(f)

providers = config['models']['providers']
agent_models = config['agents']['defaults']['models']

print(f"{'Provider':<20} {'Models':>6} {'Aliased':>8} {'Missing':>8} {'Status'}")
print("-" * 60)

for prov_name in sorted(providers.keys()):
    model_ids = {m['id'] for m in providers[prov_name].get('models', [])}
    aliased = {k.split('/')[1] for k in agent_models.keys() if k.startswith(f'{prov_name}/')}
    missing = len(model_ids - aliased)
    extra = len(aliased - model_ids)
    
    status = "✅" if missing == 0 and extra == 0 else ("⚠️" if missing == 0 else "❌")
    print(f"{prov_name:<20} {len(model_ids):>6} {len(aliased):>8} {missing:>8} {status}")

print()
print(f"Total: {sum(len(p.get('models',[])) for p in providers.values())} models, "
      f"{len(agent_models)} aliased entries")
EOF
```
