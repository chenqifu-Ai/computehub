# OpenClaw 模型配置修改标准流程 (MCP-STD-001)

> **制定日期**: 2026-04-28  
> **适用范围**: OpenClaw 模型 provider/别名/默认模型配置  
> **版本**: v1.0

---

## 一、配置架构总览

OpenClaw 模型配置分 **三层**，任何修改必须保证三层链路完整一致：

```
Layer 1: models.providers      ← 物理连接（URL + API Key + 模型定义）
         ↓
Layer 2: agents.defaults.model.primary  ← 默认模型指向
         ↓
Layer 3: agents.defaults.models          ← 别名映射（alias）
```

### 1. Provider 层（物理连接）

```json
{
  "models": {
    "providers": {
      "provider_name": {
        "baseUrl": "https://...",
        "apiKey": "sk-xxx",
        "api": "openai-completions",
        "models": [
          {
            "id": "model-id",
            "name": "Model Name",
            "reasoning": false,
            "maxTokens": 8192,
            "contextWindow": 256000
          }
        ]
      }
    }
  }
}
```

**关键点：**
- `provider_name` 是别名映射中的第一级（`provider_name/model_id` 中的 provider_name）
- `models[].id` 是别名映射中的第二级
- 每个 provider 是一个独立的 API 端点 + Key

### 2. Agent 层（默认 + 别名）

```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "provider_name/model_id"    // 默认模型
      },
      "models": {
        "provider_name/model_id": {
          "alias": "快捷别名"     // 可选，方便使用
        }
      }
    }
  }
}
```

**关键点：**
- `primary` 的值格式：`provider_name/model_id`
- 别名映射的 key 必须是 `provider_name/model_id` 格式
- key 中的 provider_name 必须在 `models.providers` 中存在
- key 中的 model_id 必须在对应 provider 的 `models` 列表中存在

## 二、完整修改流程（标准操作 SOP）

### 第 0 步：读取当前配置

```bash
python3 -c "
import json
with open('/root/.openclaw/openclaw.json') as f:
    config = json.load(f)
# 打印所有 provider 和 alias
for k, v in config['models']['providers'].items():
    print(f'{k}: {v[\"baseUrl\"]} -> {[m[\"id\"] for m in v[\"models\"]]}')
for p, i in config['agents']['defaults']['models'].items():
    if 'alias' in i:
        print(f'  [{i[\"alias\"]}] -> {p}')
"
```

### 第 1 步：确认目标 provider 和 model

明确：
1. **哪个 provider**（必须已存在于 `models.providers`）
2. **哪个 model id**（必须存在于该 provider 的 `models` 列表）
3. **要设为什么别名**（如 `zhangtuo-common`）
4. **是否要改 primary**

### 第 2 步：执行修改

```python
import json
with open('/root/.openclaw/openclaw.json') as f:
    config = json.load(f)

# 如果需要修改 primary
config['agents']['defaults']['model']['primary'] = 'new_provider/new_model'

# 如果需要修改别名
config['agents']['defaults']['models']['new_provider/new_model'] = {
    'alias': 'new_alias'
}

# 如果需要删除旧别名
if 'old_provider/old_model' in config['agents']['defaults']['models']:
    del config['agents']['defaults']['models']['old_provider/old_model']

with open('/root/.openclaw/openclaw.json', 'w') as f:
    json.dump(config, f, indent=2, ensure_ascii=False)
```

### 第 3 步：验证链路（必须执行！）

```bash
python3 -c "
import json
with open('/root/.openclaw/openclaw.json') as f:
    config = json.load(f)

all_ok = True
for full_path, info in config['agents']['defaults']['models'].items():
    if 'alias' not in info:
        continue
    alias = info['alias']
    provider_name = full_path.split('/')[0]
    model_id = full_path.split('/')[1]
    
    if provider_name not in config['models']['providers']:
        print(f'  ❌ {alias}: provider \"{provider_name}\" NOT FOUND!')
        all_ok = False
    else:
        provider = config['models']['providers'][provider_name]
        model_found = any(m['id'] == model_id for m in provider['models'])
        if model_found:
            reasoning = next((m.get('reasoning', False) for m in provider['models'] if m['id'] == model_id), False)
            print(f'  ✅ {alias} -> {full_path} (reasoning={reasoning})')
        else:
            print(f'  ❌ {alias}: model \"{model_id}\" NOT in provider \"{provider_name}\"!')
            all_ok = False

if all_ok:
    print()
    print('✅ 所有别名链路验证通过！')
else:
    print()
    print('❌ 存在错误链路，请修正！')
    exit(1)
"
```

### 第 4 步：验证 primary

```bash
python3 -c "
import json
with open('/root/.openclaw/openclaw.json') as f:
    config = json.load(f)
primary = config['agents']['defaults']['model']['primary']
provider_name, model_id = primary.split('/')
provider = config['models']['providers'].get(provider_name)
if provider and any(m['id'] == model_id for m in provider['models']):
    print(f'✅ primary 验证通过: {primary}')
else:
    print(f'❌ primary 验证失败: {primary}')
"
```

## 三、常见错误模式

### ❌ 错误模式 1：别名中的 provider 不存在

```json
// 错误：zhangtuo-ai-common provider 存在，但别名指向 zhangtuo-ai/provider（不存在）
"zhangtuo-ai/qwen3.6-35b-common": { "alias": "zhangtuo-common" }
```

**修正：** 检查 provider 名称是否精确匹配
```json
"zhangtuo-ai-common/qwen3.6-35b-common": { "alias": "zhangtuo-common" }
```

### ❌ 错误模式 2：别名中的 model id 不存在

```json
// 错误：model id 拼写错误或不在 provider 的 models 列表中
"some-provider/wrong-model-id": { "alias": "test" }
```

**修正：** 确认 model id 与 provider 定义完全一致

### ❌ 错误模式 3：primary 指向不存在的 provider

```json
// 错误：primary 中的 provider 未在 models.providers 中定义
"primary": "non-existent-provider/model"
```

### ❌ 错误模式 4：provider 层重复定义

```json
// 错误：多个 provider 使用相同的 baseUrl 和 key，容易混淆
"zhangtuo-ai": { "baseUrl": "https://a.com", ... }
"zhangtuo-ai-common": { "baseUrl": "https://a.com", ... }  // 同一个端点
```

**说明：** 同一个 API 端点可以注册多个 provider，只要 model id 不同就行。但要确保别名正确指向对应 provider。

## 四、修改前检查清单

修改任何模型配置前，先确认：

- [ ] 目标 provider 存在于 `models.providers`
- [ ] 目标 model id 存在于对应 provider 的 `models` 列表
- [ ] reasoning 字段是否正确设置
- [ ] maxTokens / contextWindow 是否合理
- [ ] API Key 是否正确且未过期
- [ ] 别名是否与其他别名冲突
- [ ] primary 指向是否符合预期

---

## 五、修改后验证清单

修改后必须执行：

- [ ] 运行链路验证脚本（Section 二·第3步）
- [ ] 验证 primary 指向正确
- [ ] 确认所有 alias 都指向正确的 provider+model
- [ ] 无 ❌ 标记
- [ ] （可选）重启 OpenClaw 使配置完全生效

## 六、案例记录

### 案例 1：2026-04-28 zhangtuo-common 别名修正

**问题：** 别名 `zhangtuo-common` 指向 `zhangtuo-ai/qwen3.6-35b-common`，但 `qwen3.6-35b-common` 定义在 `zhangtuo-ai-common` provider 中，不在 `zhangtuo-ai` 中。

**根因：** 修改别名时 provider 名称写错（漏了 `-common` 后缀）。

**修正：**
```diff
- "zhangtuo-ai/qwen3.6-35b-common": { "alias": "zhangtuo-common" }
+ "zhangtuo-ai-common/qwen3.6-35b-common": { "alias": "zhangtuo-common" }
```

**教训：** 修改别名时，provider 名称必须与 `models.providers` 中的 key **逐字匹配**。
