## 2. 发送策略（优先级）

```
QQ邮箱可用? → 是 → 从QQ发到目标地址
      ↓ 否
163邮箱可用? → 是 → 从163发到目标地址
      ↓ 否
记录到文件等待下次尝试
```

⚠️ **重要**: QQ邮箱 SMTP 连续认证失败会触发安全封锁（535 error），此时换 163 邮箱发送，等 QQ 邮箱恢复后再用。

## 3. 标准发送脚本

### 3.1 通用发送脚本

```python
#!/usr/bin/env python3
"""
STD-003: 邮件发送标准脚本
Usage: python3 send_email.py <接收邮箱> <主题> <内容文件>
"""
import smtplib
from email.mime.text import MIMEText
from email.mime.multipart import MIMEMultipart

def send_email(to_addr, subject, content_path, 
               from_addr='chenqifu_fzu@163.com',
               smtp_server='smtp.163.com',
               smtp_port=465,
               password='AWZBPidhza74EbV8'):
    """
    发送邮件
    参数:
        to_addr: 接收邮箱
        subject: 邮件主题
        content_path: 邮件内容文件路径 (.md 或 .txt)
        from_addr: 发件邮箱（默认163）
        smtp_server: SMTP服务器（默认smtp.163.com）
        smtp_port: SMTP端口（默认465）
        password: SMTP授权码
    返回:
        bool: 是否发送成功
    """
    with open(content_path, 'rb') as f:
        content = f.read()
    
    # 构建 HTML 邮件体
    ext = content_path.split('.')[-1].lower()
    if ext in ('md', 'txt'):
        html_body = ('<html><head><meta charset="utf-8"></head><body>'
                     '<pre style="font-family:monospace;font-size:14px;white-space:pre-wrap">'
                     + content.decode('utf-8') +
                     '</pre></body></html>')
    else:
        # 二进制文件作为附件
        # TODO: 附件支持
        return False
    
    msg = MIMEMultipart('mixed')
    msg['From'] = from_addr
    msg['To'] = to_addr
    msg['Subject'] = subject
    msg.attach(MIMEText(html_body, 'html', 'utf-8'))
    
    try:
        smtp = smtplib.SMTP_SSL(smtp_server, smtp_port, timeout=10)
        smtp.login(from_addr, password)
        smtp.sendmail(from_addr, [to_addr], msg.as_string())
        smtp.quit()
        return True
    except Exception as e:
        print(f"❌ 发送失败: {e}")
        return False


# 如果邮箱被封锁，走备用通道
def send_email_fallback(to_addr, subject, content_path):
    """
    备用邮箱发送（自动切换）
    """
    # 先用主邮箱（QQ）
    print("→ 尝试 QQ 邮箱...")
    ok = send_email(
        to_addr, subject, content_path,
        from_addr='19525456@qq.com',
        smtp_server='smtp.qq.com',
        smtp_port=465,
        password='ormxhluuafwnbgei'
    )
    if ok:
        return True
    
    # 主邮箱失败，切备用（163）
    print("→ 切换 163 邮箱...")
    ok = send_email(
        to_addr, subject, content_path,
        from_addr='chenqifu_fzu@163.com',
        smtp_server='smtp.163.com',
        smtp_port=465,
        password='AWZBPidhza74EbV8'
    )
    return ok


# CLI 入口
if __name__ == '__main__':
    import sys
    if len(sys.argv) < 4:
        print("Usage: python3 send_email.py <to> <subject> <content_file>")
        sys.exit(1)
    ok = send_email_fallback(sys.argv[1], sys.argv[2], sys.argv[3])
    print('✅ 发送成功' if ok else '❌ 发送失败')
```

## 4. 常见问题处理

### 4.1 QQ邮箱 535 Login Fail

```
错误: 535 Login fail. Account is abnormal, service is not open, 
      password is incorrect, login frequency limited, or system is busy.
```

**原因**: 连续多次认证失败触发 QQ 邮箱安全保护  
**解决方法**: 
1. ✅ 立即切 163 邮箱发送（备选方案）
2. ✅ 等 1-2 小时后再尝试 QQ 邮箱
3. ❌ 不要连续重试（会延长封锁时间）

### 4.2 163邮箱 Connection closed

```
错误: SMTPServerDisconnected: Connection unexpectedly closed
```

**原因**: 
- 授权码过期
- 网络问题
- IMAP/SMTP 服务未开启

**解决方法**: 
- 登录 163 网页版 → 设置 → POP3/SMTP → 确认开启
- 重新生成授权码

### 4.3 附件发送

```python
from email.mime.base import MIMEBase
from email import encoders

# 添加附件
part = MIMEBase('application', 'octet-stream')
with open(file_path, 'rb') as f:
    part.set_payload(f.read())
encoders.encode_base64(part)
part.add_header('Content-Disposition', f'attachment; filename="{filename}"')
msg.attach(part)
```
