## 🧠 核心 API 格式异常

### 问题
vLLM 部署的 qwen3.6-35b 响应格式**非标准 OpenAI**：

| 字段 | 标准 OpenAI | qwen3.6-35b 实际值 |
|------|------------|-------------------|
| `content` | 回答文本 | **始终 null** |
| `reasoning` | 推理过程 | **全部输出**（推理+回答混在一起） |

### 原因
vLLM 将 qwen 的 thinking 机制直接映射到了 `reasoning` 字段，
而标准 OpenAI 接口的 `content` 和 `reasoning` 是分开的。

### 解决方案（适配层逻辑）
```
1. 读 reasoning 字段（因为那里有全部内容）
2. 过滤元信息（"Thinking Process:"、"Final Output:" 等）
3. 提取实际回答
```

## 🔴 踩坑记录（v1→v10 全记录）

### v1 (16:15) — 发现 content=null
- **现象**: 所有测试 0 分
- **原因**: 直接读 content 字段
- **修复**: 创建适配层，优先读 reasoning

### v2→v3 (18:20) — 81分 🥇
- **修复**: content 字段优先策略
- **结果**: 34/42 (81%)
- **经验**: 这是最关键的修复，不改这个一切白搭

### v4 (19:26) — 86分 🥇 巅峰
- **修复**: 代码执行从 exec() 改为文件方式
- **结果**: 12/14 (86%)
- **经验**: exec() 缩进太容易出错，文件执行更稳

### v5→v9 (19:50→20:09) — 连续下降 71%→53%
- **v5 (71%)**: 深度专项测试，代码执行继续失败
- **v6 (57%)**: 修复+多语言，代码执行仍然不行
- **v7 (53%)**: UTF8 编码问题
- **v8 (68%)**: 跳过代码，纯知识测试回升
- **教训**: 代码执行是环境问题，不是模型能力问题

### v10 (22:35) — 82.6% 🥈
- **结果**: 38/46 (82.6%)
- **发现重大缺陷**: 7个"失败"项，模型其实答了，但 adapter 输出提取丢内容
  - 化学 H₂O → 没匹配到 H2O（下标字符）
  - TCP/UDP → adapter 只保留后半段
  - ACID → adapter 只保留"持久性"，漏了前三个
  - 死锁 → adapter 只保留"循环等待"
  - 设计模式 → adapter 只保留"观察者"
- **教训**: adapter 的 `_clean_output()` 太激进，丢失了结构化回答的多项内容

## 🛠️ adapter 输出提取缺陷

### 问题描述
`_clean_output()` 方法过于激进，会截断结构化回答的多项内容：

```
问题: ACID 四个特性？
模型回答: 原子性(A) + 一致性(C) + 隔离性(I) + 持久性(D)
adapter 输出: 持久性（因为匹配到最后一行含"持久性"的行）
```

### 当前过滤规则（过于严格）
```python
skip_patterns = [
    "thinking process", "here's a thinking", "let me",
    "好的，", "check:", "verify:", "confirmation:",
]
```

### 改进建议
1. **缩短内容时保留多项**: 如果回答有编号（1. 2. 3. 或 A B C），保留全部
2. **下标字符处理**: H₂O 应同时匹配 H2O
3. **结构感知**: 遇到列表/枚举时不要只取最后一行
4. **长度阈值**: 当前 100 字符阈值太低，结构化回答通常较长

## 🎯 后续使用指南

### 日常调用
```python
# 普通问答 — 用 ask()
# 推理题 — 用 ask_detailed() 
# 代码题 — 用 ask_code()
```

### 测试建议
1. **避免过度精确匹配**: 模型回答格式多变，关键词匹配要宽松
2. **区分模型能力 vs adapter 缺陷**: 很多"失败"是 adapter 提取问题
3. **代码测试用 ask_code()**: 不要手动执行生成代码
4. **记录 adapter 缺陷**: 如果回答正确但 adapter 提取失败，记为 adapter 问题而非模型问题

- **中项**: 架构知识、编程概念、科学常识
- **注意**: 化学式下标（H₂O）、结构化回答的多项内容
