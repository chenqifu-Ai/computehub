
## 二、向其他节点派发任务

### 方式A：Shell命令任务
```bash
curl -X POST http://36.250.122.43:8282/api/v1/tasks/submit \
  -H 'Content-Type: application/json' \
  -d '{"node_id":"目标节点ID","command":"要执行的命令","timeout":30,"priority":5}'
```

### 方式B：让其他节点的Agent思考
```bash
curl -X POST http://36.250.122.43:8282/api/v1/tasks/submit \
  -H 'Content-Type: application/json' \
  -d '{"node_id":"目标节点ID","command":"curl -s -X POST http://localhost:8383/api/v1/worker/think -H '\''Content-Type: application/json'\'' -d '\''{\"task\":\"你的自然语言指令\"}'\''","timeout":60}'
```

### 方式C：查询任务结果
```bash
curl -s 'http://36.250.122.43:8282/api/v1/tasks/detail?task_id=任务ID'
```

## 三、集群广播

向所有在线节点广播消息：
```bash
curl -X POST http://36.250.122.43:8282/api/v1/cluster/broadcast \
  -H 'Content-Type: application/json' \
  -d '{"type":"arc_net_broadcast","arc_net":{"version":"1.0","event":"broadcast","sender":{"node_id":"wanlida-ubuntu","label":"达智"},"seq":1,"timestamp":1234567890},"role":"user","content":"你的广播内容"}'
```

## 六、重要规则

1. **唯一通道**: 所有Agent通讯必须通过 ComputeHub Gateway (36.250.122.43:8282)，禁止SSH/Tailscale/VNC/RDP
2. **Worker端口**: 每个Worker的Agent监听 localhost:8383
3. **任务超时**: 简单命令30s，Agent思考60s，长任务120s+
4. **不要杀Worker**: 先spawn新进程验证在线，再杀旧进程
5. **共享知识**: 重要经验写入集群记忆，让所有Agent受益
