### 二、Git 查找流程（铁律）

```
收到任务/查询
    ↓
先 git ls-files | grep <关键词>
    ↓
找到文件 → 用 read 工具读取
    ↓
未找到 → git grep -l <关键词>
    ↓
找到文件 → 用 read 工具读取
    ↓
仍未找到 → find 文件系统确认
    ↓
确认不存在 → 告知老大"文件不存在"
```

**禁止**:
- ❌ 直接猜路径
- ❌ 用绝对路径盲读
- ❌ 跳过 git 查找

### 三、Commit 规范

#### Commit 类型

| 类型 | 前缀 | 适用场景 | 示例 |
|------|------|----------|------|
| 更新 | `更新:` | 修改已有文件 | `更新: SOP.md - 新增Git记忆系统` |
| 记录 | `记录:` | 新增每日记录 | `记录: 2026-04-24 - 天气查询` |
| 教训 | `教训:` | 经验教训 | `教训: 先git查再操作 - weather路径` |
| 快照 | `快照:` | 每日快照 | `快照: 2026-04-24 - 系统状态` |
| 配置 | `配置:` | 修改配置 | `配置: TOOLS.md - API更新` |
| 修复 | `修复:` | 修复问题 | `修复: .gitignore - 排除嵌套git` |

#### Commit 格式

```
<类型>: <文件/主题> - <具体内容> - <补充说明>
```

**示例**:
```
更新: SOP.md - 新增Git记忆管理系统铁律 - 先git查再操作
教训: memory/2026-04-24.md - 先git查再操作 - weather skill路径错误
快照: 2026-04-24 - 每日系统状态记录
记录: memory/2026-04-24.md - 厦门天气查询
```

### 四、操作分级与 Git 要求

#### 🔴 红色操作（修改核心文件）

**文件**: AGENTS.md, SOUL.md, SOP.md, MEMORY.md, IDENTITY.md, USER.md

**要求**:
1. 必须 commit，且必须有详细描述
2. commit 后必须告知老大变更了什么
3. commit 格式: `更新: <文件> - <变更说明> - <影响范围>`

#### 🟡 黄色操作（修改配置/工具）

**文件**: TOOLS.md, HEARTBEAT.md, 配置文件

**要求**:
1. 必须 commit
2. commit 格式: `配置: <文件> - <变更说明>`

#### 🟢 绿色操作（日常记录）

**文件**: memory/YYYY-MM-DD.md, memory/topics/

**要求**:
1. 每次交互后记录关键信息
2. 累积 3-5 条或当日结束时 commit
3. commit 格式: `记录: <主题> - <具体内容>`

## ⚠️ 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| git 仓库过大 | push/pull 慢 | 定期清理大文件，使用 LFS |
| commit 过多 | 历史冗余 | 每日快照合并，不频繁 commit |
| 嵌套 git 冲突 | 提交失败 | ai_agent/code/.git 不纳入 |
| 分支混乱 | 难以维护 | 只用 master，不随意切分支 |

## 🔄 Git 工作流程总结

```
日常操作流程:
  查询文件 → git ls-files | grep → 读取 → 操作 → 验证 → commit
                                      ↓
                              未找到 → git grep -l → 读取 → 操作 → 验证 → commit

修改文件流程:
  找到文件 → 用 edit 工具精确修改 → 验证结果 → git add + commit

学习记录流程:
  学到新知识 → 写入 memory/YYYY-MM-DD.md → 必要时写入 topics/ → commit
```
