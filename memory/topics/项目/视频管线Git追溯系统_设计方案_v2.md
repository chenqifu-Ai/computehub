## 一、系统概述

### 1.1 这是什么

一个 **PDF/PPT → 短视频** 的自动化生产线。你上传文档 → 系统自动解析内容 → 生成带语音解说的短视频 → 展示在作品广场。

### 1.2 整体架构

```
┌──────────────────────────────────────────────────────────┐
│                     浏览器用户                              │
│  (Gallery 网页界面)                                        │
│  - 上传PDF/PPT/音频文件                                    │
│  - 查看作品列表                                            │
│  - 播放/下载视频                                           │
└──────────────────────────┬───────────────────────────────┘
                           │ HTTP API (8282端口)
                           ▼
┌──────────────────────────────────────────────────────────┐
│              ECS 服务器 (36.250.122.43)                    │
│                                                           │
│  ┌──────────────────┐   ┌──────────────────────────────┐ │
│  │   Gateway API    │   │   Gallery (HTML/JSON页面)      │ │
│  │  (Go 服务)       │   │   http://36.250.122.43:8282   │ │
│  │                  │   │                                │ │
│  │  POST /api/v1/   │   │  作品展示 / 上传 / 删除        │ │
│  │  video/generate  │   │  文件预览 / 播放               │ │

│           ▼                                                │
│  ┌──────────────────────────────────────────────┐         │
│  │  video_worker.py → video_pipeline.py          │         │
│  │  (管线引擎: PDF解析 → TTS → 视频合成)         │         │
│  └──────────────────────────────────────────────┘         │
│           │                                                │
│           ▼                                                │
│  ┌──────────────────┐   ┌──────────────────────────────┐ │
│  │  Pipeline Repo   │   │   Gallery 存储                 │ │

```

### 1.3 核心特性

| 特性 | 说明 |
|------|------|
| **一键生成** | 上传PDF → 自动解析 → 自动配音 → 产出短视频 |
| **全自动** | 提交后后台处理，无需等待 |
| **可追溯** | 每次任务自动git记录，3个commit |
| **多格式** | PDF/PPT → 视频；语音文件 → 音频作品 |
| **作品广场** | 网页浏览、播放、下载、删除 |

### 2.3 操作步骤详解

#### 步骤1：上传文件

| 方式 | 操作 |
|------|------|
| 点击上传 | 点击上传区域 → 选择文件（可多选） |
| 拖拽上传 | 从桌面拖拽文件到上传区域 |

**支持的文件类型：**

| 类型 | 扩展名 | 系统自动识别为 | 说明 |
|------|--------|---------------|------|
| 📄 文档 | `.pdf` `.pptx` `.docx` | **文档内容源** | 转换成视频主体内容 |
| 🎤 语音 | `.mp3` `.wav` `.aac` `.m4a` `.flac` | **配音文件** | 自动作为视频背景语音 |
| 🖼 图片 | `.jpg` `.png` `.webp` `.gif` | **图片素材** | 自动匹配到视频页面 |
| 🎬 视频 | `.mp4` `.avi` `.mov` | **视频素材** | 作为视频片段嵌入 |

#### 步骤2：文件识别状态

上传后系统自动识别文件类型，显示在列表中：

```
状态指示:
✅ 已识别 — 系统知道这是什么文件类型
⏳ 分析中 — 正在解析文件内容
❌ 不支持 — 格式不被支持
```

#### 步骤3：点击"生成视频"

- 系统将所有上传的文件打包成一个任务
- 自动提交到后台管线处理
- 不需要等待，可以继续上传其他任务

#### 步骤4：查看进度

提交后进度面板实时显示：
- 当前阶段（解析中 / TTS语音合成中 / 视频合成中 / 完成）
- 百分比进度条
- 预计剩余时间

#### 步骤5：查看作品

完成后作品自动出现在 Gallery 网格中，可以：
- ▶ 播放视频
- ⬇ 下载文件
- 🗑 删除作品

## 三、系统文件自动识别标准

### 3.1 文件类型判断规则

上传的每个文件，系统自动判断其角色：

```
上传文件
    │
    ├── .pdf  →  ⏩ 文档内容     → 进入 pipeline: 解析+编排
    ├── .pptx →  ⏩ 文档内容     → 同上
    ├── .docx →  ⏩ 文档内容     → 同上
    │
    ├── .mp3  →  ⏩ TTS参考音色   → 供 TTS 模块模仿语音
    ├── .wav  →  ⏩ TTS参考音色   → 同上
    ├── .m4a  →  ⏩ TTS参考音色   → 同上
    ├── .aac  →  ⏩ TTS参考音色   → 同上
    │
    ├── .jpg  →  ⏩ 图片素材      → 匹配到视频页面的背景图
    ├── .png  →  ⏩ 图片素材      → 同上
    │
    ├── .mp4  →  ⏩ 视频素材      → 作为视频片段嵌入
    │
    └── 其他  →  ⏩ 二进制        → 仅存储，不做处理
```

### 3.2 智能关联逻辑

当同时上传多个文件时：

```
同一次上传的多个文件:
  ├── 农产品联农.pdf        ← 文档主体
  ├── 企业简介.pptx         ← 第二个文档（会追加）
  ├── 女声_参考.wav         ← TTS配音音色参考
  └── logo.png              ← 添加到视频水印

系统自动:
  1. 识别"文档"类型为主内容
  2. 识别"语音"为配音参考
  3. 识别"图片"为素材
  4. 编排成一个视频任务
```

### 3.3 生成参数智能推导

| 文件特征 | 自动推导的参数 |
|---------|--------------|
| 上传了 PDF | → 模板用 `business`，每页5秒 |
| 上传了 PPT/PDF + 语音文件 | → 用上传的语音做TTS参考音色 |
| PDF页数 > 10 | → 自动加速到每页4秒 |
| 上传了图片 | → 用作视频背景/封面 |

## 六、Git 追溯系统的使用

### 6.1 查看历史

```bash
# 通过 API 查询历史（无需 SSH）
GET http://36.250.122.43:8282/api/v1/video/list
→ 返回所有任务列表

# 通过 SSH 查 git log（仅管理员）
ssh computehub@36.250.122.43
cd /home/computehub/pipeline-repo
git log --oneline
```

### 6.2 Git 提交规范

每次任务自动产生 3 个 git commit：

```
[commit 1] task: 厦门联农 — 开始
  记录了: 输入PDF、参数、时间

[commit 2] output: liangnong_v2.mp4 (12MB)
  记录了: 产出大小、耗时、日志摘要

[commit 3] publish: liangnong_v2.mp4 → Gallery
  记录了: 是否成功同步到Gallery
```

### 6.3 问题排查

```bash
# 查某次任务的所有记录
git log --all --grep="联农"

# 查产物是否进入了 Gallery
git log --oneline | grep publish

# 查谁删了文件
git log --all --diff-filter=D -- "*.mp4"
```

## 七、API 接口文档

### 7.1 Gallery 页面

```http
GET / → 默认页面入口
```

```http
POST /api/v1/video/generate
Content-Type: application/json
```
