### E-001 | 改IP只改缓存没改主配置

- **严重度**: 🟡重要
- **场景**: 老大要求把 vllm-proxy 的 IP 从 192.168.2.54 改为 192.168.1.17

- **影响**: 
  - 主配置一直是旧IP，服务不可用
  - 老大发现后才补救，浪费约 10 分钟

- **教训**: 
  - 改配置前必须读主配置确认文件关系
  - 任何配置修改必须走 MCP-STD-001 4步流程
  - 修改后必须检查所有相关文件
- **标准化**: → **STD-SOP-001**（标准匹配检查机制）

- **状态**: ✅ 已标准化

### E-003 | Go深拷贝 make+append 导致节点翻倍
- **日期**: 2026-05-02 09:00-09:12
- **严重度**: 🟢一般
- **场景**: ComputeHub Phase 3 启动，`go test ./src/...` 发现 visualizer 包 3 个 test fail
- **根因**:
  1. `GetRegionData()` 和 `GetAllRegions()` 深拷贝时用了 `make([]T, len)` 再 `append`
  2. `make([]T, n)` 创建 n 个零值 → `append` 再追加 n 个 → 翻倍
  3. 这是 Go 新手常见错误，`make + append` 不能混用
- **影响**:
  - 3 个测试失败（TestRegisterNode / TestAddAlert / TestRegisterNodeDiscovery）
  - 阻塞 Docker 化流程约 12 分钟
- **修复**:
  - `make([]T, n) + append` → `append([]T{}, src...)` 一行搞定
  - 2 处深拷贝同时修复
- **教训**:
  - Go 切片深拷贝：用 `append([]T{}, src...)` 或 `copy(dst, src)`，不要 `make + append`
  - test fail 第一时间看共性（3 个都是 `got 2` 明显翻倍模式）
- **标准化**: → **STD-GO-001**（Go 切片深拷贝规范）
- **重复次数**: 1
- **状态**: ✅ 已修复

### E-004 | Gateway入口代码重复两份
- **日期**: 2026-05-02 08:44-09:00
- **严重度**: 🟡重要
- **场景**: ComputeHub Phase 3 启动时检查代码结构
- **根因**:
  1. `main.go`（270行，gateway+TUI混合）和 `cmd/gateway/main.go`（92行，gateway纯版）几乎完全一样
  2. `main.go` 有 `--tui` flag 但 TUI 入口没有独立
  3. 编译入口不确定到底用哪个（`go run .` vs `go run ./cmd/gateway`）
- **影响**:
  - 入口混乱，代码维护困难
  - 编译二进制时不清楚哪个是最新版
  - 已有旧版 `opc-gateway-v2` 等历史编译产物
- **修复**:
  - 保留 `cmd/gateway/main.go`（92行纯版）作为正式入口
  - `main.go` 替换为 `cmd/gateway/main.go` 内容
  - TUI 独立分离到 `cmd/tui/main.go`
  - 旧版完整 `main.go` 备份到 `_archive/main.go.full`
- **教训**:
  - Go 项目入口应该按功能分目录（`cmd/gateway/`、`cmd/tui/`）
  - 根目录 `main.go` 只做跳板或入口选择
  - 重复代码必须清除，不能留存"两个一样的文件"
- **标准化**: —
- **重复次数**: 1
- **状态**: ✅ 已修复
