#### ECS 节点启动
1. 检查环境: OpenClaw v2026.3.13 已装在 `/usr/local/bin/openclaw`
2. **nohup 失败** — 任务结束时子进程被杀
3. **setsid + disown 失败** — ECS 的 /bin/sh 没有 disown
4. **Python daemonize 成功**: `subprocess.Popen(start_new_session=True)`
5. ✅ 端口 18789 监听 (PID 401371)

### 踩坑记录

#### ComputeHub 任务执行特性
- **关键**: ComputeHub 任务进程退出时，**所有子进程一起被杀**
- `nohup ... &` → 子进程仍被杀（同一进程组）
- `setsid` → 脱离 TTY 但不脱离进程组
- **唯一解法**: Python `start_new_session=True`（创建新会话+新进程组）

#### Windows cmd 转义
- JSON → cmd → PowerShell 三层嵌套，引号转义地狱
- **解法**: 先写 .bat 文件再执行，避免嵌套转义

#### Task API 字段
- ❌ `target_node` → 无效
- ✅ `node` → 指定目标节点
- ✅ `node_id` / `assigned_node` → 也有效（三者等价）
