## 🚨 严重事故：WANLIDA-ACCIDENT-002 — wanlida-temp 自动升级掉线

### 时间
**03:01 - 03:06** — 节点从存活到完全失联仅 5 分钟

### 根因
Worker v1.3.35 内置 UpgradeManager 自动升级 v1.3.38 失败。
- 旧 Worker 检测到新版本 → 下载新 binary → kill 旧进程 → 启动新 binary
- **新 binary 损坏或环境不对 → 起不来 → 节点变砖**
- 旧 Worker 已死，无回滚机会

### 关键发现
- Gateway 从 22:10 到 03:00 催了 50+ 次升级，Worker 一直 skip，凌晨 3 点突然触发
- SHA256 在 03:00 变了（`756a14f2` → `ee416a4a`），可能是 Gateway 重启导致 checksum 更新
- IP 冲突驱逐（wanlida-temp ↔ phone-agent ↔ windows-mobile 反复抢注册）可能干扰了 Worker 状态

### 教训
**绝对禁止 Windows Worker 自动升级**。必须先下载→SHA256验证→确认版本→再kill旧进程。

### 补救
wanlida-temp 需人工到机器前恢复。
