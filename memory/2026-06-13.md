## 🚨 wanlida-opc01 下载失败深度排查 (12:05-12:40)

### 问题背景
多次尝试向 wanlida-opc01 部署 computehub.exe 全部失败。

### 尝试过的方案
| # | 方式 | 命令 | 结果 |
|---|------|------|------|
| 1 | curl (cmd & 截断) | `curl.exe ... download?file=computehub.exe&platform=...` | ❌ `&` 被 cmd 截断，URL 丢失 platform 参数 |
| 2 | powershell Invoke-WebRequest | `Invoke-WebRequest -Uri "...&platform=..."` | ❌ `&amp;` HTML 编码未解码，PowerShell 解析错误 |
| 3 | curl -H 头传平台 | `curl.exe -H platform: windows/amd64` | ❌ exit_code 1, 无输出 |
| 4 | curl -o NUL 探测连通性 | 探测 curl | ✅ 收到二进制数据但 exit_code 1 |
| 5 | curl 简单下载 | `curl.exe -o wanlidaworker3.exe .../download?file=computehub.exe` | ❌ exit_code 1, 无输出 |
| 6 | curl 带超时 + 输出路径 | `curl.exe -o C:\Users\Public\test_dl.exe` | ❌ 超时 hang 死 |
| 7 | echo 测试 | `echo hello_world` | ✅ exit 0, 成功 |
| 8 | where curl.exe | `where curl.exe` | ✅ 找到 curl |

### 根因确认
**wanlida-opc01 (183.251.21.92) 到 ECS 公网 36.250.122.43:8282 的网络质量极差**：
- 能收到 TCP 连接（第4次尝试拿到了部分二进制数据）
- 但 curl 下载大文件（computehub.exe ~10MB）时 hang 死
- 简单 echo 命令可以正常执行（不需要网络）
- Gateway 日志 6/11 之后无新日志（Gateway 6/12 重启后可能日志轮转）

### 结论
**wanlida-opc01 节点暂不可用**——不是命令格式问题，是物理网络问题。
- WS 连接反复断开重连（每秒级别）
- HTTP 下载大文件超时
- 简单 shell 命令可执行（走已有 WS 连接）
