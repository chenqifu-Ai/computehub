# 🛡️ 操作审计系统规范

## 📋 系统概述
为落实"所有操作都要有迹可循，留痕汇报给米董"的要求，建立本操作审计系统。

## 🎯 审计原则

### 1. 全面留痕原则
- 每个操作都必须记录
- 记录包括：时间、操作类型、内容、结果、风险等级
- 操作前后状态对比

### 2. 实时汇报原则  
- 重要操作立即向米董汇报
- 每日操作摘要汇报
- 风险操作专项汇报

### 3. 安全边界原则
- 严格遵守权限边界
- 危险操作三重确认
- 越权操作零容忍

## 📊 操作分类体系

### 🟢 绿色操作 (低风险)
- 本地文件读写
- 信息查询搜索  
- 计划任务管理
- **无需特殊审批**

### 🟡 黄色操作 (中风险) 
- 系统配置修改
- 网络连接测试
- 需要**事前报备**

### 🔴 红色操作 (高风险)
- 远程SSH操作
- 文件删除操作  
- 系统级修改
- **禁止执行**，必须米董明确授权

## 📝 审计日志格式

```markdown
## [时间] 操作类型 - 操作描述

### 操作详情
- **操作ID**: [唯一标识]
- **操作类型**: [绿色/黄色/红色] 
- **执行内容**: [具体命令/操作]
- **目标对象**: [操作目标]
- **执行结果**: [成功/失败/部分成功]

### 安全评估  
- **风险等级**: [低/中/高]
- **权限检查**: [通过/未通过]
- **边界合规**: [是/否]

### 汇报记录
- **汇报时间**: [时间]
- **汇报对象**: 米董
- **汇报内容**: [摘要]
```

## 🔄 汇报机制

### 即时汇报 (重要操作)
- 操作完成后立即汇报
- 通过session_send向米董发送摘要

### 每日汇报 (操作摘要)  
- 每日20:00发送当日操作摘要
- 包含操作统计、风险分析、改进建议

### 专项汇报 (风险事件)
- 发现风险操作立即专项汇报
- 包含事件分析、处理措施、预防方案

## ⚠️ 违规处理

### 轻度违规
- 操作未留痕记录
- 处理：补充记录，警告一次

### 中度违规  
- 越界操作未造成后果
- 处理：立即停止，专项汇报，安全复习

### 严重违规
- 越权操作造成影响
- 处理：全面停止，深度复盘，系统加固
