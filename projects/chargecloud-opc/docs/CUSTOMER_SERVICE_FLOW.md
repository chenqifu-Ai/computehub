# 充电云 - 客服流程文档

## 一、客服架构

### 1.1 客服分级

| 级别 | 渠道 | 响应时间 | 服务时间 |
|------|------|---------|---------|
| L1 | 在线客服 | 30分钟 | 9:00-21:00 |
| L1 | 企业微信 | 10分钟 | 9:00-21:00 |
| L2 | 电话客服 | 即时 | VIP客户 7x24 |
| L3 | 工单系统 | 2小时 | 24小时 |

### 1.2 问题分级

| 级别 | 问题类型 | 处理时限 | 处理人 |
|------|---------|---------|--------|
| P0 | 系统故障、数据丢失 | 30分钟 | 技术团队 |
| P1 | 充电桩故障、用户无法充电 | 2小时 | 运维团队 |
| P2 | 订单异常、退款问题 | 24小时 | 客服团队 |
| P3 | 功能咨询、使用指导 | 48小时 | 客服团队 |

## 二、客服流程

### 2.1 在线客服流程

```
用户发起咨询 → 机器人自动回复（FAQ匹配）
                    ↓ 无匹配
            转人工客服 → 客服接收 → 问题分类
                    ↓
            问题解决 → 用户满意度评价 → 归档
                    ↓
            问题未解决 → 升级处理 → 转相关部门
```

### 2.2 电话客服流程

```
用户拨打客服电话 → IVR语音导航
                    ↓
    按1：充电问题 → 按1：充电桩故障 → 转运维
                按2：充电操作指导 → 客服解答
    按2：订单问题 → 按1：退款问题 → 财务处理
                按2：订单查询 → 客服解答
    按3：账户问题 → 按1：登录问题 → 技术支持
                按2：账户安全 → 客服解答
    按0：人工服务 → 客服接听
```

### 2.3 工单处理流程

```
用户提交工单 → 系统自动分类 → 分配处理人
                    ↓
            处理人接单 → 问题排查 → 问题处理
                    ↓
            处理完成 → 回复用户 → 用户确认
                    ↓
            用户满意 → 工单关闭
            用户不满意 → 重新处理
```

## 三、常见场景处理

### 3.1 充电桩故障

#### 场景描述
用户反映充电桩无法使用，扫码后无反应或显示故障。

#### 处理流程
1. **确认问题**
   - 询问充电桩编号
   - 确认故障现象（离线/无法扫码/充电中断）
   
2. **远程排查**
   - 查看系统充电桩状态
   - 尝试远程重启
   
3. **处理方案**
   - 可远程修复：指导用户重试
   - 无法远程修复：引导到附近其他桩，派运维人员现场维修
   
4. **后续跟进**
   - 故障充电桩标记维修状态
   - 运维人员维修后更新状态
   - 用户回访确认

#### 话术参考
> "您好，请问是哪个充电桩出现问题？麻烦您提供一下充电桩编号，我帮您查看一下...根据系统显示，该充电桩确实存在故障，我这边已经安排维修人员处理。您可以选择附近的XX站点充电，给您造成不便非常抱歉。"

### 3.2 订单退款

#### 场景描述
用户申请退款，或订单金额异常需要处理。

#### 处理流程
1. **核实订单**
   - 查询订单号
   - 确认订单状态和金额
   
2. **判断退款原因**
   - 充电未完成：全额退款
   - 充电桩故障：全额退款+补偿
   - 用户主动取消：按实际充电量扣费
   
3. **执行退款**
   - 系统发起退款
   - 确认退款到账
   
4. **用户通知**
   - 短信/消息通知
   - 确认用户收到

#### 话术参考
> "您好，我帮您查了一下订单，订单号C20260321XXXX，充电量15度，金额45元。请问您申请退款的原因是？...好的，我这边帮您申请退款，预计1-3个工作日到账，请您留意查收。"

### 3.3 账户问题

#### 场景描述
用户无法登录、账户被盗、余额异常等。

#### 处理流程
1. **身份验证**
   - 核对手机号
   - 核对最近充值/消费记录
   
2. **问题处理**
   - 无法登录：发送重置链接，引导重置密码
   - 账户被盗：冻结账户，核实后解冻
   - 余额异常：查询消费记录，解释明细
   
3. **安全提醒**
   - 提醒修改密码
   - 开启二次验证

#### 话术参考
> "您好，请先确认一下您的注册手机号...好的，我这边帮您查询一下账户状态...根据系统显示，您的账户正常。请问具体遇到了什么问题？...好的，我这边已经帮您发送了密码重置链接，请查收邮件/短信。"

## 四、升级处理流程

### 4.1 升级条件

| 情况 | 升级级别 |
|------|---------|
| 用户情绪激动 | L2 → L3 |
| 问题超过处理时限 | 自动升级 |
| 用户要求投诉 | 转投诉处理 |
| 涉及金额>500元 | 转财务审核 |

### 4.2 升级流程

```
客服处理 → 无法解决 → 填写升级原因 → 转交上级
                                    ↓
                            上级接收 → 优先处理
                                    ↓
                            处理完成 → 回复用户 → 记录归档
```

## 五、投诉处理流程

### 5.1 投诉接待

1. **安抚情绪**
   - 表示理解
   - 承诺解决
   
2. **记录投诉**
   - 投诉内容
   - 投诉原因
   - 用户诉求
   
3. **分析原因**
   - 查阅相关记录
   - 确定责任方

### 5.2 投诉处理

1. **内部处理**
   - 责任方为平台：道歉+补偿
   - 责任方为用户：解释说明
   
2. **用户沟通**
   - 电话回访
   - 说明处理结果
   - 征求用户意见

### 5.3 投诉跟进

1. **满意度确认**
2. **改进措施**
3. **流程优化**

## 六、服务标准

### 6.1 服务态度

- **热情**：主动问候，语气亲切
- **耐心**：认真倾听，不急不躁
- **专业**：业务熟练，解答准确
- **高效**：快速响应，及时处理

### 6.2 服务禁语

❌ "我不知道"
❌ "这不是我们的问题"
❌ "你去找别人"
❌ "没办法处理"
❌ "你听不懂吗"

### 6.3 服务用语

✅ "您好，请问有什么可以帮您？"
✅ "我理解您的心情，我们马上处理"
✅ "感谢您的反馈，我们会改进"
✅ "很抱歉给您带来不便"
✅ "请问还有其他需要帮助的吗？"
