## 1. 架构概述

### 1.1 设计原则
- **模块化**：五大模块解耦，独立开发部署
- **可扩展**：支持未来新增行业方案快速接入
- **高可用**：7×24小时不间断运行
- **安全性**：内容加密防泄密
- **易用性**：零门槛语音操作

### 1.2 架构风格
采用 **分层微服务架构**，分为：
- 终端接入层（边缘设备）
- API网关层
- 业务服务层（微服务）
- 数据持久层
- AI引擎层

## 7. 数据安全架构

### 7.1 加密策略

| 数据类别 | 加密方式 | 存储位置 |
|----------|----------|----------|
| 招商全案内容 | AES-256 + 密钥派生 | 终端本地加密存储 |
| 用户凭证 | bcrypt哈希 | 服务端MySQL |
| API通信 | TLS 1.2+ (HTTPS) | 网络传输 |
| 会话令牌 | JWT + Redis黑名单 | 服务端 |
| 日志数据 | 脱敏存储 | Elasticsearch |

### 7.2 防泄密机制

1. **内容加密**：全案数据AES-256加密存储，密钥动态派生
2. **防截屏**：Android Window FLAG_SECURE
3. **防录屏**：SurfaceView硬件加速渲染
4. **防导出**：禁止文件复制/分享/下载
5. **完整性校验**：内容文件MD5/SHA256校验
