# ComputeHub 多平台节点能力管理规划

> **版本**: v0.1 (草案)
> **日期**: 2026-05-12
> **状态**: 📋 待实施

---

## 一、背景与问题

### 1.1 现状

| 节点 | 环境 | OS/Arch | 真实能力 | 调度器认知 |
|------|------|---------|----------|-----------|
| `fedora-node-140` | 原生 Fedora 40 | linux/amd64 | 全栈编译、任意 syscall | ✅ online |
| `cqf-worker-22` | 原生 Fedora 40 | linux/amd64 | 全栈编译、任意 syscall | ✅ online |
| `worker-189-1` | **Termux proot** | linux/arm64 | seccomp 限制, 无法 faccessat2 | ✅ online (实际受限) |

### 1.2 典型案例

分布式编译测试时，5/15 编译失败全部集中在 TUI 组件。

原因：`go build ./cmd/tui` 调用 `faccessat2`(Linux 5.8+) 系统调用，被 proot seccomp 规则拦截，进程直接被 `SIGSYS` 杀死。

**根因**: 调度器不知道节点"能做什么"，只看得见"在线/离线"，导致把不适合的任务发到受限节点。

---

## 二、总体方案：Capability-Aware Scheduling (能力感知调度)

### 2.1 核心思路

```
注册: worker 启动时上报"我是谁、我能干什么"
调度: 任务指定"我需要什么"，调度器过滤不匹配节点
回馈: 任务失败自动标记，同类任务后续避让
```

### 2.2 架构层次

```
┌──────────────────────────────────────────────────┐
│  第三层：运行时动态探测 + 自适应调度 (远期)          │
│  ┌────────────────────────────────────────────┐   │
│  │ 第二层：调度器能力匹配过滤 (短期)             │   │
│  │ ┌──────────────────────────────────────┐   │   │
│  │ │ 第一层：节点平台信息注册 + 标签系统    │   │   │
│  │ │ (紧急·立刻需要)                      │   │   │
│  │ └──────────────────────────────────────┘   │   │
│  └────────────────────────────────────────┘   │   │
└──────────────────────────────────────────────────┘
```

### 3.2 `TaskSubmit` 任务约束扩展

```go
type TaskSubmit struct {
    // ... 现有字段

    // 🆕 任务能力需求
    RequiredPlatform string            `json:"required_platform,omitempty"` // "linux/amd64"
    RequiredCaps    map[string]bool    `json:"required_caps,omitempty"`
    // 例: {"seccomp_compat":true, "go":true}

```

### 3.3 自动检测清单

Worker 启动时自动检测并注册：

| 检测项 | 方法 | 结果示例 |
|--------|------|---------|
| OS | `runtime.GOOS` | `linux` |
| Arch | `runtime.GOARCH` / `uname -m` | `amd64`, `arm64` |
| 环境类型 | 检测 `/dev/pts`、`/proc/1/status`、`.termux` | `native`/`proot`/`docker` |
| CPU 核数 | `runtime.NumCPU()` | `4` |
| 内存 | `/proc/meminfo` | `7.75 GB` |
| GPU | `nvidia-smi` / `rocm-smi` | `RTX4090` 或 `CPU` |
| Go 版本 | `go version` | `1.24.1` |
| GCC 版本 | `gcc --version` | `14.1.1` |
| seccomp 兼容性 | 测试性调用 `faccessat2` | `true`/`false` |
| Docker | `docker info` | `true`/`false` |
| 网络 | 是否能访问外网 | `true`/`false` |

## 四、第二层：调度器能力匹配过滤 (短期)

### 4.1 调度流程升级

```
┌─ Task 提交 ─────────────────────────────────┐
│  task_id: "build-tui-linux-amd64"           │
│  required_platform: "linux/amd64"            │
│  required_caps: {"go":true}                  │
│  required_env_types: ["native", "docker"]    │
├─ Scheduler.Schedule() ───────────────────────┤
│  1. 过滤 status != "online" 的节点            │
│  2. 过滤 负载 ≥ MaxTasks 的节点               │
│  3. 🌟 过滤 平台不匹配的节点                   │  ← 新增
│  4. 🌟 过滤 能力不满足的节点                   │  ← 新增
│  5. 🌟 过滤 环境类型不匹配的节点                │  ← 新增
│  6. 按策略(least_loaded/latency_aware) 选最优 │
└──────────────────────────────────────────────┘
```

### 4.2 能力匹配逻辑

```go
func canNodeRunTask(node *NodeInfo, task *TaskSubmit) bool {
    // 1. 平台检查
    if task.RequiredPlatform != "" && node.Platform != task.RequiredPlatform {
        return false
    }

    // 2. 环境类型检查
    if len(task.RequiredEnvTypes) > 0 {
        match := false
        for _, env := range task.RequiredEnvTypes {
            if node.EnvType == env {
                match = true
                break
            }
        }
        if !match {
            return false
        }
    }

    // 3. 能力标签检查
    for capKey, required := range task.RequiredCaps {
        nodeVal, exists := node.Capabilities[capKey]
        if !exists || nodeVal != required {
            return false
        }
    }

    return true
}
```

### 4.3 自动任务标注

编译类任务自动添加能力约束：

| 任务模式 | 自动标注 |
|----------|---------|
| `go build ./cmd/tui` | `required_env_types: ["native","docker"]` |
| `go build ./cmd/gateway` | 无特殊约束 |
| `go build ./cmd/worker` | 无特殊约束 |
| `docker build ...` | `required_caps: {"docker":true}` |
| `nvidia-smi` | `required_caps: {"gpu":true}` |

### 5.2 失败回溯 + 节点降级

```
Task "go build ./cmd/tui" on worker-189-1
    ↓ exit_code=-1 (SIGSYS)
    ↓ stderr 包含 "bad system call" / "faccessat2"
Kernel:
    ↓ 识别错误类型 = "seccomp_block"
    ↓ 为该节点添加降级标签: caps["seccomp_compat"] = false
    ↓ 自动规避后续同类任务
```

**降级策略**：

| 失败模式 | 检测特征 | 降级动作 |
|----------|---------|---------|
| SIGSYS / bad system call | stderr 含系统调用名 | 标记 `seccomp_compat=false` |
| GPU OOM | `CUDA OOM` / `out of memory` | 降低 GPU 并发数 |
| 磁盘满 | `no space left on device` | 标记磁盘不足，分配任务数减半 |
| 工具链缺失 | `command not found` | 标记该工具不可用 |
| 网络超时 | `connection refused` | 标记网络不可达 |

## 八、影响评估

### 向后兼容性

- `NodeRegister` 新增字段均为 `omitempty` → 旧 Worker 注册不受影响
- `TaskSubmit` 新增字段均为 `omitempty` → 旧任务照常调度
- 调度器：无约束任务走原有逻辑，有约束才触发过滤

### 风险

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| 能力检测误判（proot 未检测到） | 低 | 提供 CLI 手动覆盖 |
| 调度器性能下降（多一次 map 遍历） | 低 | 节点数少 (<100) 无感 |
| 任务永远匹配不到节点 | 低 | 调度器 fallback 到无约束调度 |

## 九、附录

### A. 环境类型检测参考

```go
func detectEnvType() string {
    // 1. 检查是否为 Termux
    if _, err := os.Stat("/data/data/com.termux"); err == nil {
        return "termux"
    }

    // 2. 检查是否为 proot (pid 1 不是 init/systemd)
    if data, err := os.ReadFile("/proc/1/status"); err == nil {
        if !strings.Contains(string(data), "Name:\tinit") &&
           !strings.Contains(string(data), "Name:\tsystemd") {
            return "proot"
        }
    }

    // 3. 检查是否为 Docker (/.dockerenv)
    if _, err := os.Stat("/.dockerenv"); err == nil {
        return "docker"
    }

    // 4. 检查是否为 WSL
    if data, err := os.ReadFile("/proc/version"); err == nil {
        if strings.Contains(string(data), "Microsoft") || 
           strings.Contains(string(data), "WSL") {
            return "wsl"
        }
    }

    return "native"
}
```

### B. seccomp 兼容性探测

```go
func detectSeccompCompat() bool {
    // 尝试调用一个常见被 seccomp 限制的系统调用
    // 使用 exec.Command 执行一个已知会触发 faccessat2 的操作
    cmd := exec.Command("go", "env", "GOROOT")
    if out, err := cmd.Output(); err == nil {
        goroot := strings.TrimSpace(string(out))
        testCmd := exec.Command("go", "tool", "link")
        testCmd.Env = append(os.Environ(), "GOROOT="+goroot)
        if err := testCmd.Run(); err != nil {
            if exitErr, ok := err.(*exec.ExitError); ok {
                if exitErr.ExitCode() == -1 { // signal
                    return false
                }
            }
        }
    }
    return true
}
```
