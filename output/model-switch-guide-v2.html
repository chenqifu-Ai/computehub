一、模型切换概述

为什么需要切换？

🏠 在家时 → 用本地
  • 响应速度更快
  • 不消耗云端额度
  • 隐私数据更安全
  • 离线也能使用
🚗 外出时 → 用云端
  • 随时随地可用
  • 连接稳定可靠
  • 模型能力更强
  • 不受网络限制
3

本地 vs 云端 对比

🖥️ 本地模型
  • ✓ 响应速度快
  • ✓ 零云端费用
  • ✓ 数据不出本地
  • ✓ 离线可用
  • ✗ 需要本地服务器
  • ✗ 模型能力有限
  • ✗ 需要硬件支持
☁️ 云端模型
  • ✓ 随时随地访问
  • ✓ 模型能力强
  • ✓ 无需本地硬件
  • ✓ 连接稳定
  • ✗ 消耗额度
  • ✗ 依赖网络
  • ✗ 响应稍慢

二、配置文件说明

配置文件位置

~/.openclaw/openclaw.json

关键参数：primary

{ "agent": { "primary": "ollama/llama3:latest" // 本地模型 } }
参数格式说明
  • 本地模型：ollama/模型名:标签
  • 云端模型：ollama-cloud/模型名
5

常用模型配置示例

本地模型（在家用）

// llama3 - 通用大模型，能力强 "primary": "ollama/llama3:latest" // phi3 - 轻量模型，速度快 "primary": "ollama/phi3:latest" // deepseek-coder - 代码专用 "primary": "ollama/deepseek-coder:latest"

云端模型（外出用）

// 云端 glm-5 - 智谱云端模型 "primary": "ollama-cloud/glm-5" // 云端 glm-4.7-flash - 快速响应 "primary": "ollama-cloud/glm-4.7-flash"

四、踩坑记录 ⚠️

❌ 错误1：本地没有你想要的模型

问题：切换到 glm-5:cloud 本地模型时，发现本地服务器没有这个模型。

# 错误配置（本地没有这个模型） "primary": "ollama/glm-5:cloud" # ← 本地服务器不存在此模型！

原因：glm-5 是云端模型，本地服务器只有：llama3, phi3, deepseek-coder

✅ 正确做法：先查看本地可用模型
# 查看本地服务器有哪些模型 curl -s http://192.168.1.7:11434/api/tags # 本地可用模型： - llama3:latest (4.3GB) - phi3:latest (2.0GB) - deepseek-coder:latest (0.7GB)

然后选择本地有的模型，或者用云端模型。

❌ 错误2：混淆本地和云端模型格式

问题：把云端模型格式用成本地格式，或反过来。

# 错误写法 "primary": "ollama/glm-5" # ← glm-5不在本地！ "primary": "ollama-cloud/llama3" # ← llama3不在云端！ # 正确写法 "primary": "ollama/llama3:latest" # ← 本地模型 "primary": "ollama-cloud/glm-5" # ← 云端模型

五、常见问题

Q1: 本地模型列表为空？

检查本地服务器是否启动：
curl http://192.168.1.7:11434/api/tags
如果无法连接，确认服务器IP和端口是否正确。

Q2: 切换后模型不可用？

确认模型名称正确。本地模型需要已在服务器上安装。
可用命令拉取模型：ollama pull llama3

Q3: 云端模型响应慢？

云端模型依赖网络，响应时间比本地长是正常的。
建议在家用本地，外出用云端。

Q4: 如何判断当前用的是本地还是云端？

cat ~/.openclaw/openclaw.json | grep "primary"
ollama/ 开头 = 本地模型
ollama-cloud/ 开头 = 云端模型
