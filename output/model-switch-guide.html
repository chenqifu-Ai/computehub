一、模型切换概述

为什么需要切换？

🏠 在家时 → 用本地
  • 响应速度更快
  • 不消耗云端额度
  • 隐私数据更安全
  • 离线也能使用
🚗 外出时 → 用云端
  • 随时随地可用
  • 连接稳定可靠
  • 模型能力更强
  • 不受网络限制
3

本地 vs 云端 对比

🖥️ 本地模型
  • ✓ 响应速度快
  • ✓ 零云端费用
  • ✓ 数据不出本地
  • ✓ 离线可用
  • ✗ 需要本地服务器
  • ✗ 模型能力有限
  • ✗ 需要硬件支持
☁️ 云端模型
  • ✓ 随时随地访问
  • ✓ 模型能力强
  • ✓ 无需本地硬件
  • ✓ 连接稳定
  • ✗ 消耗额度
  • ✗ 依赖网络
  • ✗ 响应稍慢

二、配置文件说明

配置文件位置

~/.openclaw/openclaw.json

关键参数：primary

{ "agent": { "primary": "ollama/llama3:latest" // 本地模型 } }
参数格式说明
  • 本地模型：ollama/模型名:标签
  • 云端模型：ollama-cloud/模型名
5

常用模型配置示例

本地模型（在家用）

// llama3 - 通用大模型，能力强 "primary": "ollama/llama3:latest" // phi3 - 轻量模型，速度快 "primary": "ollama/phi3:latest" // deepseek-coder - 代码专用 "primary": "ollama/deepseek-coder:latest"

云端模型（外出用）

// 云端 glm-5 - 智谱云端模型 "primary": "ollama-cloud/glm-5" // 云端 glm-4.7-flash - 快速响应 "primary": "ollama-cloud/glm-4.7-flash"

三、切换操作步骤

1
查看可用模型
检查本地服务器有哪些模型可用
curl -s http://192.168.1.7:11434/api/tags | python3 -c "import json,sys; d=json.load(sys.stdin); [print(f\" - {m['name']}\") for m in d['models']]"
2
编辑配置文件
修改 primary 参数为想要的模型
# 切换到本地 llama3 "primary": "ollama/llama3:latest" # 切换到云端 glm-5 "primary": "ollama-cloud/glm-5"

四、常见问题

Q1: 本地模型列表为空？

检查本地服务器是否启动：
curl http://192.168.1.7:11434/api/tags
如果无法连接，确认服务器IP和端口是否正确。

Q2: 切换后模型不可用？

确认模型名称正确。本地模型需要已在服务器上安装。
可用命令拉取模型：ollama pull llama3

Q3: 云端模型响应慢？

云端模型依赖网络，响应时间比本地长是正常的。
建议在家用本地，外出用云端。
