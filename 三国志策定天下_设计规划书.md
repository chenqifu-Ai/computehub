## 1. 项目概述

### 1.1 游戏定位

| 项目 | 内容 |
|------|------|
| 游戏名称 | 《三国志·策定天下》 |
| 游戏类型 | 策略战棋 + 角色扮演 + 城池经营 |
| 目标平台 | PC (Windows/Mac/Linux) + 移动端 (iOS/Android) |
| 目标用户 | 策略游戏爱好者、三国文化爱好者、战棋玩家 |
| 核心卖点 | 武将即棋子，谋略即武器 |
| 开发引擎 | Unity / Godot 4 |
| 预计开发周期 | 10个月 |
| 团队规模 | 核心团队 5-8 人 |

### 1.2 设计理念

1. **深度策略** — 每个决策都有分量，没有"最优解"
2. **历史沉浸** — 尊重历史框架，但给玩家改变历史的能力
3. **武将灵魂** — 每个武将都有独特性和记忆点
4. **易学难精** — 上手简单，精通需要数百小时
5. **无内购** — 买断制，完整体验

### 1.3 核心循环

## 2. 核心玩法设计

### 2.1 回合制框架

| 周期 | 内容 | 持续时间 |
|------|------|----------|
| 每月 | 内政指令 + 外交 + 人事 | 玩家操作时间 |
| 每季 | 收入结算 + 人口增长 + 科技研发 | 自动 |
| 每年 | 武将寿命判定 + 势力评价 | 自动 |

### 2.2 胜利条件

| 模式 | 胜利条件 |
|------|----------|
| 统一天下 | 占领所有城池，消灭所有势力 |
| 称霸一方 | 占领一州以上，维持30年不被灭 |
| 历史目标 | 完成剧本特定目标（如官渡之战击败袁绍） |
| 自定义 | 玩家自设目标 |

### 2.3 失败条件

- 所有城池被占领
- 君主被俘且无继承人
- 所有武将阵亡/叛逃
- 特定剧本时间限制内未达成目标

### 4.3 收入结算（每季度）

```
季度收入 = 商业值 × 税率 × 人口系数
季度粮产 = 农业值 × 人口系数 × 天气修正
季度支出 = 兵力 × 军饷 + 武将俸禄 + 城池维护
```

**税率设置**:
| 税率 | 收入倍率 | 民心影响 |

| 低 (10%) | ×0.5 | +2/季 |
| 中 (20%) | ×1.0 | 0 |
| 高 (30%) | ×1.5 | -5/季 |
| 暴政 (40%) | ×2.0 | -15/季 |

| 属性 | 计算公式 | 作用 |

### 6.4 伤害计算公式

```
基础伤害 = 攻击方攻击力 × (1 + 兵种克制修正)
         × (1 + 士气修正) × (1 + 地形修正)
         × 随机因子 (0.9~1.1)

最终伤害 = 基础伤害 - 防御方防御力 × 0.5

兵力损失 = 最终伤害 / 10 (每点伤害损失10兵)

士气影响:
  士气>80: 伤害×1.2
  士气60-80: 伤害×1.0
  士气40-60: 伤害×0.8
  士气20-40: 伤害×0.5
  士气<20: 可能溃逃
```

### 6.6 单挑系统

#### 触发条件
- 两支部队相邻
- 双方武将武力均≥70
- 双方兵力均>500

#### 单挑模式（三局两胜）

```
每局选择: 攻击 / 防御 / 技能

攻击 → 克制 → 防御 (攻击破防)
防御 → 克制 → 技能 (防御挡技能)
技能 → 克制 → 攻击 (技能破攻)

胜者: 对败者造成武力差×10的伤害
败者: 士气-20，概率受伤/被俘
```

#### 单挑结果

| 结果 | 概率 | 效果 |
|------|------|------|
| 平局 | 武力差<10时 | 双方无事 |
| 轻伤 | 武力差10-20 | 败者全属性-10%，持续3回合 |
| 重伤 | 武力差20-30 | 败者全属性-30%，持续5回合 |
| 被俘 | 武力差>30 | 败者被俘，可招降/斩首/交换 |
| 斩杀 | 武力差>40或触发技能 | 败者阵亡 |


## 9. AI 设计

### 9.1 AI 决策框架

```
每个AI势力每回合决策流程:

1. 战略评估阶段
   ├─ 评估自身实力（兵力/经济/武将/城池）
   ├─ 评估各邻国威胁等级
   └─ 确定当前战略目标（扩张/防守/发展）

2. 资源分配阶段
   ├─ 内政预算分配（发展/军备/外交）
   ├─ 武将任命（太守/军师/将军）
   └─ 科技研发选择

3. 军事决策阶段
   ├─ 判断进攻目标（优先级排序）
   ├─ 兵力调配（前线/后方/预备队）
   └─ 出征准备（武将/兵种/粮草）

4. 外交决策阶段
   ├─ 评估同盟价值
   ├─ 判断背刺时机
   └─ 应对玩家外交提案
```

### 12.2 数据架构

```
数据层:
├─ 静态数据 (JSON)
│   ├─ 武将数据: name, stats, skills, relations
│   ├─ 城池数据: name, position, type, resources
│   ├─ 兵种数据: name, stats, cost, upgrades
│   ├─ 科技数据: name, effects, cost, prerequisites
│   ├─ 事件数据: trigger, conditions, effects
│   └─ 剧本数据: year, factions, initial_state
│
├─ 动态数据 (SQLite)
│   ├─ 游戏存档: save_id, timestamp, game_state
│   ├─ 势力状态: faction_id, resources, tech, relations
│   ├─ 武将状态: general_id, loyalty, exp, items
│   └─ 城池状态: city_id, stats, buildings, garrison
│
└─ 配置数据 (JSON)
    ├─ 游戏设置: difficulty, speed, UI preferences
    └─ 键位设置: key_bindings
```
