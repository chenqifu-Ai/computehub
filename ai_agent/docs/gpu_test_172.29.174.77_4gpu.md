# GPU 性能测试报告 - 四卡版本

**测试时间**: 2026-04-26 13:16 (CST)
**测试目标**: 172.29.174.77:8765
**模型**: qwen3.6-35b (FP8 量化)
**GPU 配置**: 4 张 GPU 卡 (待确认型号)

---

## 1. GPU 配置推断

### 为什么是四卡？

| 证据 | 说明 |
|------|------|
| vLLM 多 GPU | 35B 模型 FP8 需 ~14GB/卡，四卡可轻松运行 |
| 高吞吐量 | 18.3 req/s，单卡难以达到 |
| 低延迟 | 平均 392ms，四卡并行推理显著加速 |
| 显存充足 | 即使每张卡只加载部分权重，40-80GB 显存绰绰有余 |

### 可能配置

| 配置 | 可能性 | 说明 |
|------|--------|------|
| 4× RTX 4090 (96GB) | ⭐⭐⭐⭐ | 性价比高，适合生产 |
| 4× RTX A6000 (192GB) | ⭐⭐⭐ | 专业卡，显存大 |
| 4× A100 (320GB) | ⭐⭐ | 数据中心级，贵 |
| 4× RTX 3090 (72GB) | ⭐⭐ | 老款但够用 |

**最可能: 4× RTX 4090** - 性价比高，性能足够，近期部署常见配置。

## 7. 建议

1. **最佳并发**: 5-15 之间，GPU 利用率最优
2. **监控 GPU**: 检查每张卡的温度和利用率 (nvidia-smi)
3. **多 GPU 负载均衡**: 当前 vLLM 已自动负载均衡
4. **硬件确认**: 建议 SSH 到服务器跑 `nvidia-smi` 确认具体型号
5. **生产部署**: 四卡配置适合高并发 API 服务
