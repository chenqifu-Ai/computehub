# 附件四：预留权益池与员工激励方案（草案）

**（Token聚合平台三方合作协议之附件四）**

---

## 第一部分：预留权益池管理制度

### 一、预留权益池概况

| 项目 | 内容 |
|------|------|
| 占项目总权益比例 | **10%** |
| 用途 | 引入对项目有重大价值的未来合作方 |
| 单个合作方持股上限 | 不超过 **5%** |
| 管理方式 | 由三方按原权益比例 **共同管理** |

### 二、引入对象标准

#### 2.1 适用对象
在技术、市场、政策或资本等维度能为项目带来显著价值的个人或机构，包括但不限于：

| 类别 | 价值标准 | 参考持股 |
|------|---------|---------|
| **技术类** | 拥有核心AI技术、稀缺算法能力或重要技术资源 | 1%-3% |
| **市场类** | 拥有大规模B端客户渠道或行业垄断级市场资源 | 1%-5% |
| **政策类** | 拥有关键政策资源或行业准入资质通道 | 1%-3% |
| **资本类** | 能带来项目发展所需的资金或融资渠道 | 按投资额协商 |

#### 2.2 引入决策流程
1. **提议**：任意一方提出引入建议，说明合作方的价值、拟授权益比例及条件
2. **评估**：三方共同评估合作方的价值贡献
3. **决策**：三方 **一致同意** 方可引入
4. **签约**：签署《合作方权益授予协议》明确双方权利义务

### 三、授出机制

#### 3.1 权益授予方式
1. **增资模式**（推荐）：新合作方向项目增资，权益比例按增资额计算
2. **转让模式**：从预留权益池中直接转让，三方不稀释

#### 3.2 权益兑现条件
1. 一次性授予：签署合作协议即授予全部约定权益
2. 分期授予（推荐）：分 **____期** 授予，每期达成约定条件后自动解锁

### 四、退出机制

1. 预留权益合作方如需退出，其权益由 **三方按原权益比例优先受让**
2. 三方均放弃受让的，可引入新合作方替代
3. 合作方未达成约定价值的，三方有权以约定价格回购其权益

## 第二部分：员工激励方案（草案）

### 一、员工激励池概况

| 项目 | 内容 |
|------|------|
| 占项目总权益比例 | **15%** |
| 用途 | 激励核心技术骨干、运营骨干及其他对公司有重大贡献的员工 |
| 管理方式 | 通过 **有限合伙持股平台** 或 **虚拟期权** 方式进行管理 |

### 二、激励对象

#### 2.1 激励对象范围

| 层级 | 对象 | 激励上限（个人） |
|------|------|----------------|
| **核心层** | CTO、技术总监、核心架构师 | ≤3% |
| **骨干层** | 高级开发、核心运营人员 | ≤1.5% |
| **潜力层** | 优秀开发、运营人员 | ≤0.5% |

#### 2.2 进入条件
激励对象须满足以下至少两项条件：
1. 连续在职不少于 **6个月**
2. 在关键技术或业务领域有突出贡献
3. 担任核心岗位且不可或缺
4. 经三方一致认可的其他条件

### 三、激励方式

#### 方案A：有限合伙持股平台（推荐）

```
              ┌───────────────────────┐
              │     创始人GP          │ ← 由三方指定，负责管理
              │ （持有平台0.1%份额）  │
              └─────────┬─────────────┘
                        │
              ┌─────────▼─────────────┐
              │   有限合伙持股平台     │ ← 持有项目15%权益
              │  （XX合伙企业）        │
              └─────────┬─────────────┘
                        │
          ┌─────────────┼─────────────┐
          │             │             │
     ┌────▼────┐  ┌────▼────┐  ┌────▼────┐
     │ 员工LP1 │  │ 员工LP2 │  │ 员工LP3 │
     │ (1%)    │  │ (0.5%)  │  │ (0.3%)  │  ...
     └─────────┘  └─────────┘  └─────────┘
```

#### 方案B：虚拟期权
1. 不实际变更股权结构
2. 达到约定条件时，按虚拟期权的约定份额参与分红
3. 离职后自动失效

### 四、激励计划要素

#### 4.1 授予流程
1. **提名**：任一合伙人或部门负责人提名激励对象
2. **考核**：评估考核对象过往贡献及未来价值
3. **决策**：经 **三方一致同意** 确定授予份额
4. **签约**：与激励对象签署《权益授予协议》
5. **登记**：在权益池台账中登记

#### 4.2 成熟（解锁）安排

| 成熟方式 | 说明 | 推荐场景 |
|---------|------|---------|
| **阶梯式** | 分4年成熟：第1年25%、第2年25%、第3年25%、第4年25% | 核心技术员工 |
| **绩效式** | 达成约定KPI后一次性成熟 | 运营/销售骨干 |
| **混合式** | 基础部分按时间成熟，额外部分按绩效成熟 | 综合岗位 |

#### 4.3 行权方式
1. 授予时无需支付对价（无偿授予）
2. 退出时按约定价格回购

### 五、退出与回购

#### 5.1 回购触发条件

| 情形 | 回购价格 | 说明 |
|------|---------|------|
| **正常离职** | 按项目净资产评估价回购 | 工作交接完成 |
| **辞退** | 按授予价格回购 | 公司解除劳动关系 |
| **违约行为** | 1元回购 | 违反保密/竞业约定 |
| **公司控制权变更** | 加速成熟并可选择退出 | 收购/合并情形 |

#### 5.2 回购程序
1. 激励对象退出条件触发后 **30日内** 完成回购
2. 回购款一次性或分期支付（经双方协商）
3. 回购权益重新归入员工激励池

### 六、管理机制

#### 6.1 管理主体
由 **三方共同管理** 员工激励池，日常事务由指定负责人处理。

#### 6.2 台账管理
1. 建立《员工激励池权益台账》，详细记录每笔授予、成熟、回购情况
2. 每半年向三方报告激励池变动情况
3. 台账内容包括：

```
激励对象：__________
岗位：__________
授予日期：__________
授予份额：____%（占总权益）
成熟时间表：__________________
当前成熟状态：__________________
授予价格：¥____元
回购约定：__________________
```

## 第三部分：附则

1. 预留权益池和员工激励池的 **未授出权益**，其对应的收益暂由三方按各自权益比例 **代持并参与分配**，或经三方一致同意留存于项目作为发展基金。

2. 本方案作为《Token聚合平台三方合作协议》附件四，与主协议具有同等法律效力。

3. 本方案的具体实施细则由三方协商制定，经三方一致同意后可修改完善。
