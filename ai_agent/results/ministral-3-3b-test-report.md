## 四、根因分析

### 4.1 直接原因
**内存布局无法分配** (`memory layout cannot be allocated`) — Ollama在尝试为模型分配推理内存时失败。

### 4.2 深度分析

```
模型内存需求：   ministral-3:3B ≈ 2.8GB (模型权重)
实际内存需求：   ≈ 2.8GB × 2 ~ 3 ≈ 6~8GB (含KV cache + 计算开销)

当前服务器状态：
├── 4个模型同时常驻 (Ollama默认行为 keep_alive=-1)
├── 已占用内存：~20.5GB (模型权重)
└── 加上推理开销，实际需求可能 > 30GB
```

**核心矛盾**: 模型文件(2.8GB) ≠ 实际运行时内存(6~8GB)，服务器可用内存不足以支撑。
