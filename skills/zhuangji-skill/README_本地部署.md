## 部署流程

1. **设备检测** - 检查SSH连接和系统信息
2. **OpenClaw安装** - 使用本地资源安装指定版本
3. **配置初始化** - 运行openclaw setup
4. **配置同步** - 复制本地完整配置到目标设备
5. **服务启动** - 启动Gateway服务
6. **验证检查** - 确认部署成功

## 本地资源策略

### 优先使用本地包
脚本会检查以下位置的OpenClaw安装包：
- `~/.npm/_cacache/content-v2/sha512/` - npm缓存目录
- 本地下载的tar.gz包

### 回退机制
如果本地包不存在，会自动回退到网络安装指定版本

## 注意事项

1. **网络要求**: 目标设备必须能够SSH连接
2. **权限要求**: 需要root或sudo权限安装npm包
3. **版本兼容**: 确保本地和目标设备架构兼容
4. **备份机制**: 部署前会自动备份目标设备现有配置

## 故障排除

### SSH连接失败
- 检查目标设备IP地址
- 确认SSH服务正常运行
- 验证用户名和密码

### 安装失败
- 检查网络连接
- 确认npm权限
- 查看详细错误日志

### 服务启动失败
- 检查端口冲突
- 查看openclaw日志

1. 脚本执行权限 `chmod +x *.sh`
2. sshpass安装 `apt install sshpass`

4. 目标设备可达性
