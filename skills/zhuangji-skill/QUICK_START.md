# 🚀 装机技能 - 快速开始指南

## 一键部署命令

### 基础部署
```bash
./deploy.sh 192.168.1.19 8022 u0_a207 123 18789
```

### 参数说明
| 参数 | 说明 | 默认值 |
|------|------|--------|
| 目标IP | 要部署的设备IP地址 | (必填) |
| SSH端口 | SSH连接端口 | 8022 |
| 用户名 | 目标设备用户名 | u0_a207 |
| 密码 | 目标设备密码 | 123 |
| Gateway端口 | OpenClaw网关端口 | 18789 |

## 常用部署场景

### 1. 部署到Android设备 (Termux)
```bash
./deploy.sh 192.168.1.100 8022 u0_a355 123456 18789
```

### 2. 部署到Linux服务器
```bash
./deploy.sh 192.168.1.200 22 root password123 18888
```

### 3. 自定义端口部署
```bash
./deploy.sh 192.168.1.150 2222 admin mypassword 19999
```

## 部署流程验证

### 手动验证步骤
1. **检查SSH连接**: 
   ```bash
   sshpass -p "密码" ssh -p 端口 用户名@IP "echo 连接成功"
   ```

2. **检查服务状态**:
   ```bash
   curl http://IP:端口/health
   ```

3. **检查配置文件**:
   ```bash
   sshpass -p "密码" ssh -p 端口 用户名@IP "ls -la ~/.openclaw/"
   ```

## 故障排除

### 常见问题
1. **SSH连接失败**: 检查网络、端口、用户名密码
2. **权限不足**: 确保目标设备有安装权限
3. **端口冲突**: 更换Gateway端口号
4. **依赖缺失**: 确保目标设备有Node.js环境

### 日志查看
部署过程中的详细日志会在终端输出，包含每个步骤的执行结果。

## 高级功能

### 批量部署脚本
创建 `batch_deploy.sh` 文件:
```bash
#!/bin/bash
# 批量部署多个设备

devices=(
    "192.168.1.101 8022 u0_a207 123 18789"
    "192.168.1.102 8022 u0_a208 123 18790" 
    "192.168.1.103 8022 u0_a209 123 18791"
)

for device in "${devices[@]}"; do
    echo "部署设备: $device"
    ./deploy.sh $device
    echo "---"
    sleep 2
done
```

### 自定义配置
编辑 `deploy.sh` 文件中的以下部分来自定义:
- 部署超时时间
- 备份策略
- 验证检查项
- 日志输出格式
