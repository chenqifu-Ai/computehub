# 信用风险评级模型

## 一、信用风险概述

### 1. 定义
信用风险是指债务人未能履行合同义务而给债权人造成损失的风险。是金融机构最主要的风险类型之一。

### 2. 信用风险构成
- **违约风险**：借款人无法按期还本付息
- **信用利差风险**：因信用质量变化导致的利差波动
- **信用降级风险**：信用评级下调导致的损失

### 3. 影响因素
| 因素类型 | 具体内容 |
|----------|----------|
| 宏观因素 | 经济周期、利率水平、行业景气度 |
| 企业因素 | 财务状况、经营能力、管理层素质 |
| 合同因素 | 担保方式、期限结构、还款安排 |

## 二、信用评级体系

### 1. 外部评级

#### 国际评级机构（三大评级机构）
- **穆迪(Moody's)**：Aaa至C评级体系
- **标准普尔(S&P)**：AAA至D评级体系
- **惠誉(Fitch)**：AAA至D评级体系

#### 评级符号对照
| 穆迪 | 标普/惠誉 | 信用质量 |
|------|-----------|----------|
| Aaa | AAA | 最高级 |
| Aa | AA | 高级 |
| A | A | 中上级 |
| Baa | BBB | 中级（投资级分界线） |
| Ba | BB | 中下级（投机级） |
| B | B | 投机级 |
| Caa | CCC | 高风险 |
| Ca | CC | 极度投机 |
| C | C/D | 违约或接近违约 |

#### 评级分类
- **投资级**：AAA至BBB-（风险较低）
- **投机级/垃圾级**：BB+至D（风险较高）

### 2. 内部评级体系（IRB）

#### 巴塞尔协议要求
根据巴塞尔协议，银行可采用内部评级法计量信用风险资本要求：

- **初级法**：银行自行估计违约概率(PD)，其他参数采用监管规定值
- **高级法**：银行自行估计全部参数（PD、LGD、EAD、M）

#### 关键参数
| 参数 | 名称 | 定义 |
|------|------|------|
| PD | 违约概率 | 借款人在一定期限内违约的可能性 |
| LGD | 违约损失率 | 违约时损失的金额占风险暴露的比例 |
| EAD | 违约风险暴露 | 违约时的风险敞口金额 |
| M | 有效期限 | 信贷产品的有效期限 |

## 三、信用评分模型

### 1. 传统评分模型

#### 专家判断法（5C模型）
- **Character（品德）**：借款人诚信度、还款意愿
- **Capacity（能力）**：还款能力、现金流状况
- **Capital（资本）**：净资产、所有者权益
- **Collateral（担保）**：抵押物价值、担保条件
- **Conditions（环境）**：经济环境、行业状况

#### Z-score模型（Altman模型）
适用于上市公司的破产预测模型：

**Z = 1.2X1 + 1.4X2 + 3.3X3 + 0.6X4 + 1.0X5**

| 变量 | 含义 | 计算 |
|------|------|------|
| X1 | 营运资本/总资产 | (流动资产-流动负债)/总资产 |
| X2 | 留存收益/总资产 | 累计未分配利润/总资产 |
| X3 | 息税前利润/总资产 | (利润总额+利息支出)/总资产 |
| X4 | 股权市值/总负债 | 股价×股数/总负债 |
| X5 | 销售收入/总资产 | 营业收入/总资产 |

**判断标准**：
- Z > 2.99：安全区
- 1.81 < Z < 2.99：灰色区（不确定）
- Z < 1.81：危险区

### 2. 现代评分模型

#### Logistic回归模型
- **原理**：用逻辑函数预测违约概率
- **公式**：P(Y=1) = 1 / (1 + e^(-z))
- **优点**：解释性强、计算简单
- **缺点**：假设线性关系

#### 决策树模型
- **原理**：通过一系列规则对客户分类
- **优点**：直观易懂、处理非线性
- **缺点**：容易过拟合

#### 机器学习模型
| 模型类型 | 特点 | 适用场景 |
|----------|------|----------|
| 随机森林 | 多棵决策树集成 | 特征复杂、非线性强 |
| XGBoost | 梯度提升集成 | 大数据、高精度需求 |
| 神经网络 | 深度学习 | 海量数据、复杂模式 |
| 支持向量机 | 核函数映射 | 小样本、高维数据 |

### 3. 模型评估指标

#### 混淆矩阵
| | 预测违约 | 预测正常 |
|---|----------|----------|
| 实际违约 | TP（真正例） | FN（假反例） |
| 实际正常 | FP（假正例） | TN（真反例） |

#### 关键指标
- **准确率(Accuracy)**：(TP+TN) / (TP+TN+FP+FN)
- **精确率(Precision)**：TP / (TP+FP)
- **召回率(Recall)**：TP / (TP+FN)
- **F1分数**：2 × Precision × Recall / (Precision + Recall)

#### 信用评分特有指标
- **KS值(Kolmogorov-Smirnov)**：衡量好客户与坏客户评分分布的最大差异
  - KS > 0.3：模型区分能力较好
  - KS > 0.5：模型区分能力很强
- **AUC值(Area Under Curve)**：ROC曲线下面积
  - AUC > 0.7：模型有效
  - AUC > 0.8：模型良好

## 四、个人信用评分

### 1. 征信报告
- **内容**：个人基本信息、信贷记录、公共记录、查询记录
- **来源**：中国人民银行征信中心（央行征信）
- **更新频率**：金融机构按月报送

### 2. 个人征信评分维度
| 维度 | 权重示例 | 考核内容 |
|------|----------|----------|
| 还款历史 | 35% | 是否按时还款、逾期次数 |
| 负债水平 | 30% | 信用卡使用率、负债比 |
| 信用历史长度 | 15% | 开户时间、最老账户年龄 |
| 新信用申请 | 10% | 近期申请次数、查询频率 |
| 信用类型 | 10% | 信用产品多样性 |

### 3. 信用评分应用
- **信用卡审批**：准入判断、额度核定
- **贷款审批**：房贷、车贷、消费贷
- **利率定价**：差异化定价策略
- **额度管理**：授信额度调整

## 五、企业信用评级

### 1. 财务指标分析

#### 偿债能力指标
- **流动比率**：流动资产/流动负债（>2为佳）
- **速动比率**：(流动资产-存货)/流动负债（>1为佳）
- **资产负债率**：负债/资产（<70%为佳）

#### 盈利能力指标
- **净资产收益率(ROE)**：净利润/净资产
- **总资产收益率(ROA)**：净利润/总资产
- **毛利率**：(收入-成本)/收入

#### 营运能力指标
- **应收账款周转率**：收入/应收账款
- **存货周转率**：成本/存货
- **总资产周转率**：收入/总资产

### 2. 非财务因素
- **管理层素质**：经验、诚信、决策能力
- **行业地位**：市场份额、竞争优势
- **外部环境**：政策支持、经济环境
- **或有事项**：担保、诉讼、承诺

### 3. 企业信用评级流程
```
企业申请 → 资料收集 → 现场调查 → 初评分析 → 
评级委员会审定 → 评级公布 → 跟踪评级
```

## 六、风险管理应用

### 1. 贷前准入
- **准入门槛**：设置最低信用评分要求
- **授信额度**：根据评级结果核定额度
- **定价策略**：风险定价，高风险高利率

### 2. 贷中监控
- **预警机制**：评分下降触发预警
- **额度调整**：动态调整授信额度
- **贷后检查**：定期跟踪客户状况

### 3. 贷后管理
- **风险分类**：五级分类（正常、关注、次级、可疑、损失）
- **拨备计提**：根据风险分类计提减值准备
- **催收处置**：逾期账户的催收策略

## 七、合规与监管

### 1. 监管要求
- 《商业银行资本管理办法（试行）》
- 《商业银行大额风险暴露管理办法》
- 《征信业务管理办法》
- 《个人信息保护法》

### 2. 数据合规
- **数据采集**：合法合规、最小必要
- **数据使用**：授权使用、目的限制
- **数据安全**：加密存储、访问控制
- **数据留存**：保存期限、销毁机制

### 3. 消费者保护
- **知情权**：告知评分用途和影响
- **异议权**：对评分结果提出异议
- **隐私权**：个人信息保护

## 八、实务要点

### 1. 模型开发注意事项
- 样本代表性：确保样本覆盖各类客户
- 特征工程：选择有效、合规的特征变量
- 模型验证：样本内外验证、时间外验证
- 模型监控：定期监测模型表现衰减

### 2. 信用评分局限性
- **历史数据局限**：历史不代表未来
- **黑天鹅事件**：突发事件难以预测
- **模型假设**：假设可能与现实偏离
- **数据质量**：垃圾进垃圾出

### 3. 模型迭代优化
- **定期校准**：每年或每半年重新校准
- **特征更新**：纳入新的有效特征
- **模型升级**：引入新技术、新方法
- **A/B测试**：新模型与旧模型对比
