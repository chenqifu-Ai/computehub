# 网络安全 - 零信任网络架构

## 一、零信任概述

### 定义
零信任（Zero Trust）是一种安全模型，核心原则是"永不信任，始终验证"。不默认信任任何用户、设备或网络，所有访问请求都需要经过身份验证和授权。

### 传统模型 vs 零信任

| 特性 | 传统边界模型 | 零信任模型 |
|------|------------|-----------|
| 信任基础 | 内网可信 | 无默认信任 |
| 边界防护 | 防火墙为边界 | 身份为边界 |
| 访问控制 | 基于网络 | 基于身份 |
| 验证频率 | 一次验证 | 持续验证 |

---

## 二、零信任核心原则

### 1. 最小权限原则
- 用户只获得完成任务所需的最小权限
- 权限按需分配，用完即收回

### 2. 持续验证
- 每次访问都需要验证
- 验证身份、设备、环境

### 3. 微分段
- 网络细粒度划分
- 限制横向移动

### 4. 假设被攻破
- 设计时假设已被入侵
- 最小化攻击影响范围

## 五、关键技术实现

### 1. SDP（软件定义边界）
```
客户端 → 控制器（认证） → 网关（授权） → 资源
```

### 2. 微分段实现
- 基于VLAN
- 基于SDN
- 基于主机代理

### 3. 持续验证流程
```
请求 → 身份验证 → 设备检查 → 策略评估 → 授权 → 访问 → 监控
```

## 九、常见误区

### 1. 零信任=无信任
- 实际是"动态信任"
- 基于上下文评估信任度

### 2. 替代防火墙
- 零信任是架构理念
- 防火墙仍是重要组件

### 3. 一次实施永久有效
- 零信任是持续过程
- 需要不断优化

### 4. 只适用于大企业
- 小企业也能实施
- 可从简单方案开始

## 十、零信任最佳实践

### 1. 分阶段实施
- 从最关键的资产开始
- 逐步扩大范围

### 2. 用户培训
- 零信任改变使用习惯
- 需要用户理解配合

### 3. 监控优先
- 先建立监控能力
- 再逐步收紧权限

### 4. 应急预案
- 零信任可能阻断正常访问
- 需要应急恢复机制
