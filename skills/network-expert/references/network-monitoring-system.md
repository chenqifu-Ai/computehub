# 网络监控体系 - 企业级监控架构设计

## 概述

网络监控是企业IT运维的核心，通过实时监控网络设备、流量、性能指标，实现故障快速发现、定位和恢复。

## 二、监控技术详解

### 1. SNMP监控

```
SNMP架构：

┌─────────────────┐      SNMP      ┌─────────────────┐
│  管理工作站     │ ◄───────────▶ │  网络设备       │
│  (NMS)         │               │  (Agent)        │
│                │               │                 │
│ · SNMP Manager │               │ · SNMP Agent    │
│ · MIB Browser  │               │ · MIB           │
│ · Trap Receiver│               │ · OID           │
└─────────────────┘               └─────────────────┘

SNMP版本对比：
┌──────────┬─────────────────────────────────────────────────┐
│  版本    │  特点                                           │
├──────────┼─────────────────────────────────────────────────┤
│ SNMPv1   │ 基础功能，明文传输，安全性低                     │
│ SNMPv2c  │ 增强功能，Community认证，安全性低               │
│ SNMPv3   │ 认证加密，用户权限，安全性高（推荐）             │
└──────────┴─────────────────────────────────────────────────┘

常用OID：
├── 系统信息：1.3.6.1.2.1.1
│   └── sysDescr：系统描述
│   └── sysUpTime：运行时间
├── 接口信息：1.3.6.1.2.1.2
│   └── ifTable：接口表
│   └── ifInOctets：入流量
│   └── ifOutOctets：出流量
├── IP信息：1.3.6.1.2.1.4
│   └── ipForwarding：IP转发
│   └── ipDefaultTTL：默认TTL
└── CPU/内存：厂商特定OID
    └── Cisco CPU：1.3.6.1.4.1.9.9.109
    └── 华为 CPU：1.3.6.1.4.1.2011.6.3
```

### 2. NetFlow/sFlow流量监控

```
NetFlow工作原理：

┌─────────────────────────────────────────────────────────────────────────────┐
│                           NetFlow流程                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────┐  │
│  │  流量入口    │────▶│  流识别    │────▶│  流记录    │────▶│ 流输出  │  │
│  │             │     │             │     │             │     │         │  │
│  │ 数据包      │     │ 5元组识别   │     │ 统计信息    │     │ NetFlow │  │
│  │             │     │             │     │             │     │ Collector│  │
│  └─────────────┘     └─────────────┘     └─────────────┘     └─────────┘  │
│                                                                             │
│  5元组（5-tuple）：                                                         │
│  ├── 源IP地址（Source IP）                                                   │
│  ├── 目的IP地址（Destination IP）                                           │
│  ├── 源端口（Source Port）                                                   │
│  ├── 目的端口（Destination Port）                                           │
│  └── 协议（Protocol）                                                        │
│                                                                             │
│  流记录包含：                                                               │
│  ├── 5元组信息                                                               │
│  ├── 开始/结束时间                                                           │
│  ├── 包数/字节数                                                             │
│  ├── TCP标志                                                                │
│  └── 输入/输出接口                                                           │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

NetFlow版本：
├── NetFlow v5：最常用，基础流记录
├── NetFlow v9：支持模板，可扩展
└── IPFIX：IETF标准，兼容NetFlow v9

sFlow vs NetFlow：
┌──────────┬─────────────────────┬─────────────────────┐
│  特性    │  NetFlow            │  sFlow              │
├──────────┼─────────────────────┼─────────────────────┤
│ 采样方式 │ 流级采样            │ 包级采样            │
│ 资源消耗 │ 高（需要维护流表）   │ 低（无状态）         │
│ 精度     │ 100%流信息          │ 统计估计            │
│ 实时性   │ 流结束后才输出       │ 实时输出            │
│ 适用场景 │ 流量分析、计费       │ DDoS检测、实时监控   │
└──────────┴─────────────────────┴─────────────────────┘
```

## 四、告警管理

### 告警设计原则

```
告警设计原则：

1. 分级告警
┌─────────────────────────────────────────────────────────────────────────────┐
│                          告警级别                                            │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  P1 - 紧急（Critical）                                                       │
│  ├── 影响：核心业务中断                                                      │
│  ├── 响应：5分钟内响应                                                       │
│  ├── 通知：电话+短信+邮件                                                    │
│  └── 示例：核心交换机down、主链路中断                                        │
│                                                                             │
│  P2 - 严重（Major）                                                         │
│  ├── 影响：部分业务受影响                                                    │
│  ├── 响应：15分钟内响应                                                      │
│  ├── 通知：短信+邮件                                                         │
│  └── 示例：备用链路中断、单设备故障                                          │
│                                                                             │
│  P3 - 警告（Warning）                                                        │
│  ├── 影响：性能下降但未中断                                                  │
│  ├── 响应：30分钟内响应                                                      │
│  ├── 通知：邮件+IM                                                           │
│  └── 示例：CPU高、带宽利用率高                                               │
│                                                                             │
│  P4 - 信息（Info）                                                          │
│  ├── 影响：无直接影响                                                        │
│  ├── 响应：工作时间处理                                                      │
│  ├── 通知：邮件                                                             │
│  └── 示例：配置变更、设备重启                                                │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

2. 告警收敛
┌─────────────────────────────────────────────────────────────────────────────┐
│                          告警收敛策略                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  时间收敛：                                                                 │
│  ├── 同一告警，5分钟内只告警一次                                             │
│  ├── 告警风暴时，按时间窗口合并                                               │
│  └── 恢复后自动关闭相关告警                                                  │
│                                                                             │
│  关联收敛：                                                                 │
│  ├── 父子告警：设备down导致端口down，只告警设备                               │
│  ├── 根因分析：自动识别根因，过滤衍生告警                                     │
│  └── 拓扑收敛：按网络拓扑合并相关告警                                        │
│                                                                             │
│  业务收敛：                                                                 │
│  ├── 按业务系统分组告警                                                      │
│  ├── 按影响范围过滤告警                                                      │
│  └── 维护窗口抑制告警                                                       │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

3. 告警通知
┌─────────────────────────────────────────────────────────────────────────────┐
│                          告警通知方式                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐          │
│  │ 邮件        │ │ 短信       │ │ 电话       │ │ IM          │          │
│  │ Email      │ │ SMS        │ │ Call       │ │ 钉钉/企业微信│          │

│  ├── P2告警未响应 → 15分钟后升级                                            │

│                                                                             │
```
