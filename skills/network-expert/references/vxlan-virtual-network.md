# VXLAN虚拟网络 - 数据中心网络虚拟化

## 概述

VXLAN（Virtual Extensible LAN，虚拟可扩展局域网）是一种网络虚拟化技术，用于解决大型数据中心中的VLAN数量限制问题，实现大规模多租户网络隔离。

---

```
│  ┌─────────────────────────────────────────────────────────────────────┐   │
```

## 二、VXLAN技术原理

### VXLAN概述

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          VXLAN技术概述                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  定义：                                                                     │
│  VXLAN是一种网络虚拟化协议，通过MAC-in-UDP封装，在三层网络上                  │
│  构建二层 overlay网络，实现大规模多租户隔离。                                 │
│                                                                             │
│  核心特点：                                                                 │
│  ├── 扩展VLAN：24位VNI，支持1600万个虚拟网络                                  │
│  ├── Overlay：在三层网络上构建二层overlay                                     │
│  ├── 多租户：支持大规模多租户隔离                                            │
│  └── 自动化：支持自动化网络配置                                              │
│                                                                             │
│  RFC标准：                                                                  │
│  ├── RFC 7348：VXLAN规范                                                    │
│  └── IETF标准                                                               │
│                                                                             │
│  应用场景：                                                                 │
│  ├── 云数据中心多租户                                                       │
│  ├── 虚拟机迁移                                                             │
│  ├── 容器网络（Kubernetes）                                                  │
│  └── 软件定义网络（SDN）                                                     │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### VNI虚拟网络标识符

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          VNI详解                                            │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  VNI（VXLAN Network Identifier）：                                          │
│  ├── 长度：24位                                                            │
│  ├── 范围：0 - 16,777,215（约1600万）                                        │
│  ├── 作用：标识虚拟网络，实现租户隔离                                        │
│  └── 类比：类似VLAN ID，但数量远超VLAN                                       │
│                                                                             │
│  VNI vs VLAN：                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │               │ VLAN          │ VNI                   │            │   │
│  │  ─────────────────────────────────────────────────────────────────  │   │
│  │  ID位数      │ 12位          │ 24位                  │            │   │
│  │  最大数量    │ 4094          │ 16,777,215            │            │   │
│  │  封装        │ 以太网帧       │ UDP封装               │            │   │
│  │  隔离方式    │ 二层隔离       │ 三层overlay           │            │   │
│  │  适用场景    │ 传统网络       │ 云数据中心            │            │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  VNI分配方式：                                                              │
│  ├── 手动配置：管理员指定VNI                                                │
│  ├── 自动分配：VTEP自动分配                                                 │
│  ├── SDN控制器：控制器统一分配                                               │
│  └── 映射：VLAN ↔ VNI映射                                                   │
│                                                                             │
│  VNI映射示例：                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  租户    │ VLAN   │ VNI        │ 说明                              │   │
│  │  ─────────────────────────────────────────────────────────────────  │   │
│  │  租户A   │ 100    │ 100000     │ 租户A的生产环境                   │   │
│  │  租户A   │ 200    │ 100001     │ 租户A的测试环境                   │   │
│  │  租户B   │ 100    │ 200000     │ 租户B的生产环境（VLAN可重叠）      │   │
│  │  租户B   │ 200    │ 200001     │ 租户B的测试环境                   │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  注意：不同租户可以使用相同的VLAN，但映射到不同的VNI，实现隔离                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### VTEP（VXLAN Tunnel Endpoint）

## 四、VXLAN工作流程

### 单播转发流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        VXLAN单播转发流程                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  场景：VM1 (192.168.1.10) 发送数据给 VM4 (192.168.1.20)                      │
│                                                                             │
│  步骤1：ARP解析                                                              │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  VM1需要解析VM4的MAC地址                                             │   │
│  │  ├── VM1发送ARP请求（广播）                                          │   │
│  │  ├── VTEP1接收ARP请求                                                │   │
│  │  ├── VTEP1通过控制平面（如EVPN）解析VM4的位置                         │   │
│  │  └── 或VTEP1泛洪ARP请求到其他VTEP                                    │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  步骤2：封装与转发                                                           │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  VTEP1封装VXLAN报文：                                                │   │
│  │  ├── 原始帧：目的MAC=VM4的MAC，源MAC=VM1的MAC                         │   │
│  │  ├── VXLAN头：VNI=100000                                             │   │
│  │  ├── 外层UDP：目的端口=4789                                          │   │
│  │  ├── 外层IP：源IP=VTEP1_IP，目的IP=VTEP2_IP                          │   │
│  │  └── 外层以太网：目的MAC=下一跳MAC                                    │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  步骤3：Underlay转发                                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  物理网络根据外层IP转发：                                            │   │
│  │  ├── 源IP：VTEP1_IP                                                  │   │
│  │  ├── 目的IP：VTEP2_IP                                                │   │
│  │  ├── 路由：三层路由转发                                               │   │
│  │  └── ECMP：支持等价多路径负载均衡                                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  步骤4：解封装                                                               │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  VTEP2收到VXLAN报文后：                                              │   │
│  │  ├── 检查VNI=100000                                                  │   │
│  │  ├── 解封装，提取原始二层帧                                           │   │
│  │  ├── 根据VNI查找对应的虚拟网络                                       │   │
│  │  └── 根据目的MAC转发给VM4                                            │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  流程图：                                                                   │
│  ┌───────────┐   ARP   ┌───────────┐  VXLAN  ┌───────────┐         ┌───────┐│
│  │   VM1     │───────▶│  VTEP1    │────────▶│  VTEP2    │────────▶│  VM4  ││
│  │192.168.1.10│       │           │         │           │         │192.168││
│  └───────────┘         └───────────┘         └───────────┘         └───────┘│
│        │                     │                     │                      │   │
```

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        VXLAN广播/未知单播处理                                 │
├─────────────────────────────────────────────────────────────────────────────┤

│  问题：VXLAN如何处理广播、未知单播、组播（BUM流量）？                          │
│                                                                             │
│  方案1：泛洪（Head-End Replication）                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                     │   │
│  │  原理：入口VTEP复制报文，发送给所有VTEP                              │   │
│  │                                                                     │   │
│  │          ┌───────────┐                                             │   │
│  │          │   VTEP1   │ (源)                                         │   │
│  │          └─────┬─────┘                                             │   │
│  │                │                                                   │   │
│  │        ┌───────┼───────┐                                          │   │
│  │        ▼       ▼       ▼                                          │   │
│  │  ┌─────────┐ ┌─────────┐ ┌─────────┐                             │   │
│  │  │ VTEP2   │ │ VTEP3   │ │ VTEP4   │ (复制)                      │   │
│  │  └─────────┘ └─────────┘ └─────────┘                             │   │
│  │                                                                     │   │
│  │  优点：简单，不需要组播支持                                          │   │
│  │  缺点：复制开销大，带宽浪费                                          │   │

│                                                                             │

│  │                                                                     │   │
│  │  原理：使用IP组播在Underlay网络传输                                  │   │
│  │                                                                     │   │
│  │  映射：VNI → 组播组地址                                              │   │
│  │  例：VNI 100000 → 组播组 239.1.1.100                                │   │
│  │                                                                     │   │
│  │  优点：网络层复制，效率高                                            │   │
│  │  缺点：需要组播网络支持，复杂                                        │   │
│  │                                                                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  方案3：控制平面学习（EVPN）                                                 │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                     │   │
│  │  原理：使用EVPN（Ethernet VPN）控制平面学习MAC                        │   │
│  │                                                                     │   │
│  │  ┌─────────┐         ┌─────────┐         ┌─────────┐             │   │
│  │  │ VTEP    │────────▶│ EVPN    │◀────────│ VTEP    │              │   │
│  │  │         │  MP-BGP  │ 控制    │  MP-BGP  │         │              │   │
│  │  │ (MAC)   │         │ 平面    │         │ (MAC)   │              │   │
│  │  └─────────┘         └─────────┘         └─────────┘             │   │
│  │                                                                     │   │
│  │  优点：无需泛洪，精确转发，支持大规模                                │   │
│  │  缺点：需要EVPN支持                                                  │   │
│  │                                                                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  推荐方案：EVPN + Head-End Replication                                      │
│  ├── EVPN学习MAC位置，减少未知单播                                           │
│  │   └── Head-End Replication处理必要的广播                                  │
│  └── 大多数现代数据中心采用                                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 八、总结

### VXLAN核心要点

```
1. VXLAN解决VLAN数量限制，支持1600万虚拟网络
2. 使用MAC-in-UDP封装，在三层网络构建二层Overlay
3. VTEP负责封装/解封装，VNI标识虚拟网络
4. EVPN提供控制平面，无需泛洪学习MAC
5. Spine-Leaf架构适合大规模数据中心
```

### 记忆口诀

```
VLAN不够用VXLAN，二十四位VNI算。
MAC封装进UDP，三层网络二层建。
VTEP端点来做桥，租户隔离靠VNI。
EVPN做控制面，无需泛洪学MAC。
Spine Leaf架构好，数据中心首选案。
```
