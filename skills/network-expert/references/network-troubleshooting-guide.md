# 网络故障排除实战指南

## 概述

网络故障排除是网络运维的核心技能，需要系统化的方法和丰富的经验积累。

---

## 一、故障排除方法论

### 1. 故障排除流程

```
网络故障排除七步法：

┌─────────────────────────────────────────────────────────────────────────────┐
│                           故障排除流程                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐      │
│  │ 第1步   │──▶│ 第2步   │──▶│ 第3步   │──▶│ 第4步   │──▶│ 第5步   │      │
│  │ 定义问题│   │ 收集信息│   │ 分析原因│   │ 制定方案│   │ 实施修复│      │
│  └─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘      │
│       │                                                        │           │
│       │                                                        ▼           │
│       │                                              ┌─────────┐           │
│       │                                              │ 第6步   │           │
│       │                                              │ 验证结果│           │
│       │                                              └─────────┘           │
│       │                                                   │                │
│       │                                                   ▼                │
│       │                                              ┌─────────┐           │
│       │                                              │ 第7步   │           │
│       │                                              │ 文档记录│           │
│       └──────────────────────────────────────────────└─────────┘           │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

详细步骤：

第1步：定义问题
├── 症状是什么？
├── 影响范围？
├── 何时开始？
└── 有何变化？

第2步：收集信息
├── 网络拓扑
├── 配置文件
├── 日志记录
├── 用户反馈
└── 监控数据

第3步：分析原因
├── 列出可能原因
├── 排除法
├── 分层分析
└── 确定根因

第4步：制定方案
├── 解决方案
├── 回滚方案
├── 影响评估
└── 时间窗口

第5步：实施修复
├── 备份配置
├── 执行变更
├── 监控效果
└── 必要时回滚

第6步：验证结果
├── 功能验证
├── 性能验证
├── 用户确认
└── 持续观察

第7步：文档记录
├── 故障描述
├── 根本原因
├── 解决方案
└── 预防措施
```

### 2. 分层故障排除

```
OSI七层故障排除：

┌─────────────────────────────────────────────────────────────────────────────┐
│                          OSI分层故障排除                                     │
├──────────┬────────────────┬────────────────────────────────────────────────┤
│  第7层   │   应用层        │  检查：应用程序配置、API、认证授权            │
│  第6层   │   表示层        │  检查：数据格式、加密、压缩                   │
│  第5层   │   会话层        │  检查：会话管理、连接状态                      │
│  第4层   │   传输层        │  检查：端口、TCP/UDP、防火墙规则              │
│  第3层   │   网络层        │  检查：IP地址、路由、ACL                       │
│  第2层   │   数据链路层    │  检查：MAC地址、VLAN、STP、交换机配置          │
│  第1层   │   物理层        │  检查：线缆、接口、电源、硬件状态              │
└──────────┴────────────────┴────────────────────────────────────────────────┘

推荐顺序：从底层向上层排除
原因：底层故障会导致上层问题
```

## 二、常用故障排除命令

### 1. 连通性测试

```
ping命令：
┌─────────────────────────────────────────────────────────────────────────────┐
│  ping <目标IP>                     # 基本连通测试                           │
│  ping -c 5 <目标IP>                # 发送5个包                              │
│  ping -s 1000 <目标IP>              # 指定包大小（测试MTU）                  │
│  ping -i 0.5 <目标IP>               # 间隔0.5秒                              │
│  ping -t 64 <目标IP>                # 设置TTL                                │
└─────────────────────────────────────────────────────────────────────────────┘

traceroute/tracert：
┌─────────────────────────────────────────────────────────────────────────────┐
│  traceroute <目标IP>              # 跟踪路由路径                             │
│  traceroute -n <目标IP>           # 不解析DNS                                │
│  traceroute -I <目标IP>           # 使用ICMP                                 │
│  traceroute -T <目标IP>           # 使用TCP                                  │
└─────────────────────────────────────────────────────────────────────────────┘

常用诊断：
┌─────────────────────────────────────────────────────────────────────────────┐
│  # 检查本地网络接口                                                         │
│  ip addr show                     # Linux                                   │
│  ifconfig                         # Linux/Unix                              │
│  ipconfig /all                    # Windows                                 │
│                                                                             │
│  # 检查路由表                                                               │
│  ip route show                    # Linux                                   │
│  route -n                         # Linux                                   │
│  netstat -rn                      # Linux/macOS                             │
│  route print                      # Windows                                 │
│                                                                             │
│  # 检查ARP表                                                               │
│  ip neigh show                    # Linux                                   │

│  # 检查DNS解析                                                              │
```

### 3. 抓包分析

```
tcpdump命令：
┌─────────────────────────────────────────────────────────────────────────────┐
│  # 基本抓包                                                                 │
│  tcpdump -i eth0                   # 抓取eth0接口                           │
│  tcpdump -i eth0 -w capture.pcap   # 保存到文件                              │
│                                                                             │
│  # 过滤条件                                                                 │
│  tcpdump -i eth0 host 192.168.1.1  # 特定主机                               │
│  tcpdump -i eth0 port 80            # 特定端口                               │
│  tcpdump -i eth0 tcp               # TCP流量                                │
│  tcpdump -i eth0 udp               # UDP流量                                │
│  tcpdump -i eth0 icmp              # ICMP流量                               │
│                                                                             │
│  # 组合过滤                                                                 │
│  tcpdump -i eth0 'host 192.168.1.1 and port 80'                             │
│  tcpdump -i eth0 'tcp and port 443'                                         │
│  tcpdump -i eth0 'net 192.168.1.0/24'                                       │
│                                                                             │
│  # 显示选项                                                                 │
│  tcpdump -i eth0 -n               # 不解析DNS                                │
│  tcpdump -i eth0 -nn              # 不解析DNS和端口                          │
│  tcpdump -i eth0 -v               # 详细输出                                │
│  tcpdump -i eth0 -vv              # 更详细输出                                │
│  tcpdump -i eth0 -X               # 十六进制和ASCII显示                       │
└─────────────────────────────────────────────────────────────────────────────┘

Wireshark分析：
┌─────────────────────────────────────────────────────────────────────────────┐
│  常用过滤器：                                                               │
│  ├── ip.addr == 192.168.1.1       # 特定IP                                │
│  ├── tcp.port == 80                # HTTP端口                               │
│  ├── http                          # HTTP流量                               │
│  ├── dns                           # DNS流量                                │
│  ├── tcp.flags.syn == 1            # SYN包                                  │
│  ├── tcp.flags.reset == 1          # RST包                                  │
│  └── http.request.method == "GET"  # GET请求                                │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 三、常见故障场景

### 1. 连通性问题

```
故障：无法ping通目标

排除步骤：
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  1. 检查物理层                                                              │
│     ├── 线缆是否连接？                                                      │
│     ├── 接口是否UP？                                                        │
│     ├── LED状态是否正常？                                                   │
│     └── 硬件是否工作？                                                      │
│                                                                             │
│  2. 检查数据链路层                                                          │
│     ├── VLAN是否正确？                                                      │
│     ├── STP是否阻塞？                                                       │
│     ├── MAC地址是否学习？                                                   │
│     └── 端口安全是否阻止？                                                  │
│                                                                             │
│  3. 检查网络层                                                              │
│     ├── IP地址是否正确？                                                    │
│     ├── 子网掩码是否正确？                                                  │
│     ├── 网关是否可达？                                                      │
│     ├── 路由是否正确？                                                      │
│     ├── ACL是否阻止？                                                       │
│     └── NAT是否配置？                                                       │
│                                                                             │
│  4. 检查传输层及以上                                                         │
│     ├── 防火墙是否阻止ICMP？                                                │
│     └── 目标主机是否响应ping？                                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

常见原因：
├── 物理连接问题（线缆、接口）
├── IP配置错误
├── VLAN配置错误
├── 路由问题
├── ACL阻止
├── 防火墙阻止
└── 目标主机禁用了ICMP响应
```

### 2. DNS问题

```
故障：无法解析域名

排除步骤：
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│  1. 检查DNS配置                                                             │
│     cat /etc/resolv.conf           # Linux                                  │
│     ipconfig /all                  # Windows                                │
│                                                                             │
│  2. 测试DNS服务器                                                           │
│     ping <DNS服务器IP>                                                      │
│     nslookup <域名> <DNS服务器IP>                                           │
│                                                                             │
│  3. 测试DNS解析                                                             │
│     nslookup <域名>                                                         │
│     dig <域名>                                                              │
│     host <域名>                                                             │
│                                                                             │
│  4. 检查DNS缓存                                                             │
│     # Windows                                                               │
│     ipconfig /displaydns                                                    │
│     ipconfig /flushdns                                                     │
│                                                                             │
│     # Linux (systemd-resolved)                                              │
│     systemd-resolve --statistics                                            │
│     systemd-resolve --flush-caches                                         │
│                                                                             │
│  5. 检查hosts文件                                                           │
│     cat /etc/hosts                 # Linux                                  │
│     type C:\Windows\System32\drivers\etc\hosts  # Windows                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

常见原因：
├── DNS服务器不可达
├── DNS配置错误
├── hosts文件覆盖
├── DNS缓存污染
└── 域名不存在或过期
```

### 3. 性能问题

```
故障：网络延迟高

排除步骤：
┌─────────────────────────────────────────────────────────────────────────────┐

│  2. 检查带宽                                                               │
│     # 测试带宽                                                              │
│     iperf3 -s                      # 服务端                                  │
│     iperf3 -c <服务器IP>           # 客户端                                  │
│                                                                             │

│     show processes cpu             # CPU使用率                              │
│     show memory                    # 内存使用                                │
│     show interface counters        # 接口计数                               │
│                                                                             │
│  4. 检查链路状态                                                           │
│     show interfaces                # 错误计数                                │

│  5. 抓包分析                                                               │

常见原因：
├── 带宽不足
├── 网络拥塞
├── 设备性能问题
├── 错误配置（双工不匹配）
├── 线缆问题
├── 无线干扰
└── 路由环路
```

## 六、总结

### 故障排除核心原则

```
1. 从底层开始：物理层 → 数据链路层 → 网络层 → 应用层

2. 分而治之：缩小问题范围，隔离故障点

3. 收集证据：查看日志、抓包分析、收集用户反馈

4. 先易后难：先检查常见问题，再考虑复杂问题

5. 变更控制：记录所有操作，准备回滚方案

6. 文档记录：记录故障现象、原因、解决方案

7. 预防为主：分析根因，防止再次发生
```

### 记忆口诀

```
网络故障排除法，分层诊断是关键。
物理数据网络层，传输应用往上排。
Ping测连通traceroute，抓包分析找根源。
日志配置看仔细，变更记录要留存。
先易后难分而治，文档记录莫忘记。
```
