四、问题分析

问题1：认证速率限制未配置

⚠️ 中等风险

现象：Gateway绑定在非loopback地址，但没有认证速率限制

风险：可能遭受暴力破解攻击

影响：攻击者可以无限次尝试密码

问题2：命令拒绝规则无效

⚠️ 中等风险

现象：denyCommands使用精确匹配，某些条目可能不起作用

风险：命令过滤不准确

影响：可能无法阻止某些危险命令

问题3：插件安全配置未设置

⚠️ 低风险

现象：发现feishu插件，但plugins.allow未设置

风险：任何发现的插件都可能自动加载

影响：潜在的安全隐患

9

四、问题分析（续）

问题4：systemd服务不可用

ℹ️ 环境限制

现象：Termux环境不支持systemd

原因：Android/Termux环境限制

影响：无法使用systemd管理Gateway服务

解决：使用前台模式运行Gateway

问题5：Memory未存储

ℹ️ 信息

现象：Memory: 0 files · 0 chunks

原因：记忆系统未初始化或未使用

影响：长期记忆功能可能受限

问题6：Session上下文使用率高

⚠️ 注意

现象：Tokens: 100k/127k (78%)

影响：当前会话已使用78%的上下文窗口

解决：定期压缩会话或重启会话

10

五、解决方案

修复1：配置认证速率限制

# 编辑配置文件 vim ~/.openclaw/openclaw.json # 添加以下配置： { "gateway": { "auth": { "rateLimit": { "maxAttempts": 10, // 最多尝试10次 "windowMs": 60000, // 60秒窗口 "lockoutMs": 300000 // 锁定5分钟 } } } }

修复2：设置插件安全白名单

# 添加插件白名单 { "plugins": { "allow": ["feishu"] // 只允许feishu插件 } }
11

五、解决方案（续）
