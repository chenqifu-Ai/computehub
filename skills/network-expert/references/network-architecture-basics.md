# 网络架构基础

## 概述
网络架构是指计算机网络的整体设计和结构，包括网络拓扑、设备布局、协议选择和连接方式。良好的网络架构是网络稳定运行的基础。

## 一、网络拓扑类型

### 1. 星型拓扑（Star Topology）
```
        [中心节点]
        /    |    \
       /     |     \
   [节点]  [节点]  [节点]
```
**特点：**
- 所有节点通过中心节点连接
- 中心节点故障会导致全网瘫痪
- 易于扩展和故障隔离
- 线缆消耗较多

**适用场景：** 小型办公网络、家庭网络

### 2. 环型拓扑（Ring Topology）
```
    [节点] ──── [节点]
      │          │
      │          │
    [节点] ──── [节点]
```
**特点：**
- 节点首尾相连形成环
- 数据单向传输
- 任一节点故障影响全网
- 线缆消耗少

**适用场景：** 早期令牌环网络、工业控制网络

### 3. 总线型拓扑（Bus Topology）
```
    [节点]──[节点]──[节点]──[节点]
         ════════════════════
              共享总线
```
**特点：**
- 所有节点连接到共享总线
- 成本低、布线简单
- 总线故障导致全网瘫痪
- 性能随节点增加下降

**适用场景：** 早期以太网、小型临时网络

### 4. 树型拓扑（Tree Topology）
```
            [根节点]
           /        \
      [分支]        [分支]
      /   \        /    \
   [叶]   [叶]   [叶]   [叶]
```
**特点：**
- 分层结构、易于管理
- 扩展性好
- 根节点故障影响范围大
- 层级越多延迟越高

**适用场景：** 企业网络、校园网络

### 5. 网状拓扑（Mesh Topology）
```
    [节点]═══════[节点]
      ╲  ╲    ╱  ╱
       ╲  ╲  ╱  ╱
        [节点]═══[节点]
```
**特点：**
- 节点之间多条路径连接
- 可靠性最高
- 成本高、复杂度高
- 故障时自动路由绕行

**适用场景：** 核心网络、数据中心、互联网骨干

## 二、网络层次模型

### OSI七层模型
| 层次 | 名称 | 功能 | 数据单位 |
|------|------|------|----------|
| 7 | 应用层 | 用户接口、应用服务 | 数据 |
| 6 | 表示层 | 数据格式转换、加密 | 数据 |
| 5 | 会话层 | 会话管理、同步 | 数据 |
| 4 | 传输层 | 端到端传输、可靠性 | 段 |
| 3 | 网络层 | 路由选择、IP寻址 | 包 |
| 2 | 数据链路层 | 帧传输、MAC寻址 | 帧 |
| 1 | 物理层 | 比特传输、物理介质 | 比特 |

### TCP/IP四层模型
| 层次 | 对应OSI | 协议示例 |
|------|---------|----------|
| 应用层 | 5-7层 | HTTP, FTP, SMTP, DNS |
| 传输层 | 4层 | TCP, UDP |
| 网络层 | 3层 | IP, ICMP, ARP |
| 网络接口层 | 1-2层 | Ethernet, WiFi |

## 三、网络设备

### 核心设备
| 设备 | 工作层次 | 功能 |
|------|----------|------|
| 中继器 | 物理层 | 信号放大、延长距离 |
| 集线器 | 物理层 | 多端口中继、共享带宽 |
| 网桥 | 数据链路层 | 连接网段、MAC过滤 |
| 交换机 | 数据链路层 | 多端口网桥、全双工 |
| 路由器 | 网络层 | 路由选择、网络互联 |
| 三层交换机 | 网络层 | 交换+路由、硬件转发 |
| 防火墙 | 网络层-应用层 | 安全过滤、访问控制 |

### 设备选型考虑
1. **端口数量**：满足当前需求+预留扩展
2. **带宽容量**：考虑峰值流量
3. **功能支持**：VLAN、QoS、ACL等
4. **可靠性**：冗余电源、堆叠能力
5. **管理方式**：CLI、Web、SNMP

## 四、IP地址规划

### IPv4地址分类
| 类别 | 范围 | 默认子网掩码 | 私有地址范围 |
|------|------|--------------|--------------|
| A类 | 1.0.0.0-126.255.255.255 | 255.0.0.0 | 10.0.0.0/8 |
| B类 | 128.0.0.0-191.255.255.255 | 255.255.0.0 | 172.16.0.0/12 |
| C类 | 192.0.0.0-223.255.255.255 | 255.255.255.0 | 192.168.0.0/16 |

### VLSM（可变长子网掩码）
```
示例：192.168.1.0/24 划分

需求1：100台主机 → 192.168.1.0/25 (126可用)
需求2：50台主机  → 192.168.1.128/26 (62可用)
需求3：25台主机  → 192.168.1.192/27 (30可用)
需求4：10台主机  → 192.168.1.224/28 (14可用)
```

### 子网划分公式
- 子网数量：2^n（n为借用的主机位）
- 主机数量：2^h - 2（h为剩余主机位，-2为网络地址和广播地址）
- 子网掩码：/n（n为网络位数）

## 五、VLAN设计

### VLAN类型
| 类型 | 编号范围 | 用途 |
|------|----------|------|
| 默认VLAN | 1 | Cisco默认 |
| 普通VLAN | 2-1001 | 用户VLAN |
| 扩展VLAN | 1002-1005 | FDDI/Token Ring |
| 扩展范围 | 1006-4094 | 扩展用途 |

### VLAN设计原则
1. **按功能划分**：管理、数据、语音、视频
2. **按部门划分**：财务、研发、市场等
3. **按安全级别**：内部、DMZ、外部
4. **控制广播域**：每个VLAN一个子网

### VLAN间路由
```
方式1：路由器子接口（单臂路由）
方式2：三层交换机（推荐）
方式3：物理路由器多接口
```

## 六、网络架构设计原则

### 1. 层次化设计
- **接入层**：用户接入、端口安全、VLAN划分
- **汇聚层**：策略实施、VLAN间路由、汇聚流量
- **核心层**：高速转发、全网互联、最小延迟

### 2. 冗余设计
- **链路冗余**：双上行、跨设备链路聚合
- **设备冗余**：VRRP/HSRP、堆叠/虚拟化
- **路由冗余**：ECMP、动态路由多路径

### 3. 可扩展性
- 预留端口和带宽
- 模块化设备选型
- 标准化配置模板
- 地址规划预留空间

### 4. 安全设计
- 分层安全策略
- 最小权限原则
- 边界防护
- 内部隔离

## 七、常见网络架构案例

### 企业网络架构
```
                [互联网]
                    │
              [边界防火墙]
                    │
              [核心交换机] ← 冗余核心
                /    |    \
          [汇聚1] [汇聚2] [汇聚3]
           /  \     /  \     /  \
      [接入] [接入] [接入] [接入]...
        │      │      │      │
     [用户]  [用户] [用户]  [用户]
```

### 数据中心架构（Spine-Leaf）
```
           [Spine交换机] ──── [Spine交换机]
              /  │  │  \         /  │  │  \
             /   │  │   \       /   │  │   \
     [Leaf]─[Leaf]─[Leaf]─[Leaf]─[Leaf]─[Leaf]
        │      │      │      │      │      │
     [服务器][服务器][服务器][服务器][服务器][服务器]
```

**Spine-Leaf优势：**
- 所有服务器间跳数相同
- 无阻塞带宽
- 易于水平扩展
- 适合虚拟化和容器网络

## 八、最佳实践

### 1. 命名规范
```
设备：[位置]-[楼层]-[设备类型]-[编号]
示例：BJ-3F-SW-01（北京3楼交换机01）

接口：[用途]-[VLAN]-[编号]
示例：Link-Core-01（连接核心链路01）
```

### 2. IP地址规划
```
管理网络：10.0.0.0/24
服务器：10.1.0.0/16
用户网段：10.100.0.0/16（按部门划分子网）
DMZ：172.16.0.0/24
```

### 3. VLAN规划
```
VLAN 10：管理
VLAN 20：服务器
VLAN 30：用户
VLAN 40：语音
VLAN 50：视频
VLAN 100：DMZ
```
