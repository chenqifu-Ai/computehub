# 网络可观测性 - 从监控到洞察

## 概述

网络可观测性（Network Observability）是对传统网络监控的升级，不仅关注"系统是否正常"，更关注"为什么异常"，通过指标、日志、追踪三支柱实现对网络状态的深度洞察。

---

## 一、监控 vs 可观测性

### 核心区别

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        监控 vs 可观测性                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  传统监控（Monitoring）                                                     │
│  ├── 目标：回答"发生了什么"                                                  │
│  ├── 方法：预定义的指标和告警                                                │
│  ├── 特点：被动、静态                                                        │
│  └── 限制：只能发现已知问题                                                  │
│                                                                             │
│  可观测性（Observability）                                                   │
│  ├── 目标：回答"为什么发生"                                                  │
│  ├── 方法：指标+日志+追踪                                                    │
│  ├── 特点：主动、动态                                                        │
│  └── 优势：可以发现未知问题                                                  │
│                                                                             │
│  对比：                                                                     │
│  ┌─────────────────┬─────────────────┬─────────────────┐                  │
│  │     维度        │     监控        │    可观测性      │                  │
│  ├─────────────────┼─────────────────┼─────────────────┤                  │
│  │     问题类型    │    已知问题    │    未知问题      │                  │
│  │     数据来源    │    预定义指标  │    三支柱数据    │                  │
│  │     分析方式    │    告警规则    │    上下文关联    │                  │
│  │     排障效率    │    低          │    高           │                  │
│  │     适用场景    │    稳定环境    │    动态环境      │                  │
│  └─────────────────┴─────────────────┴─────────────────┘                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 二、可观测性三支柱

### 1. 指标（Metrics）

```
指标是可聚合的数值数据，回答"发生了什么"。

┌─────────────────────────────────────────────────────────────────────────────┐
│                              指标类型                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  计数器（Counter）                                                          │
│  ├── 定义：只增不减的累计值                                                  │
│  ├── 示例：请求数、错误数、字节数                                            │
│  └── 用途：计算速率、增长趋势                                                │
│                                                                             │
│  仪表盘（Gauge）                                                             │
│  ├── 定义：可增可减的瞬时值                                                  │
│  ├── 示例：温度、内存使用、连接数                                            │
│  └── 用途：当前状态、波动观察                                                │
│                                                                             │
│  直方图（Histogram）                                                         │
│  ├── 定义：观测值的分布                                                      │
│  ├── 示例：请求延迟、响应大小                                                │
│  └── 用途：分位数分析、SLA计算                                               │
│                                                                             │
│  摘要（Summary）                                                            │
│  ├── 定义：预设分位数的观测值                                                │
│  ├── 示例：P50、P95、P99延迟                                                │
│  └── 用途：性能分析                                                          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

网络关键指标：

┌─────────────────────────────────────────────────────────────────────────────┐
│                           网络监控指标                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  性能指标：                                                                 │
│  ├── 带宽利用率：接口入/出带宽                                               │
│  ├── 延迟：RTT、单向延迟                                                     │
│  ├── 丢包率：接口丢包统计                                                    │
│  ├── 抖动：延迟波动                                                         │
│  └── 吞吐量：实际传输速率                                                    │
│                                                                             │
│  可用性指标：                                                               │
│  ├── 设备可用率：在线时间/总时间                                              │
│  ├── 接口状态：up/down统计                                                   │
│  ├── 连通性：ping成功率                                                      │
│  └── 服务可用性：HTTP/HTTPS探测                                              │
│                                                                             │
│  容量指标：                                                                 │
│  ├── 接口带宽：容量利用率                                                    │
│  ├── CPU利用率：设备负载                                                     │
│  ├── 内存利用率：内存占用                                                    │
│  ├── 会话数：NAT/防火墙会话                                                  │
│  └── 流表大小：SDN流表                                                      │
│                                                                             │
│  错误指标：                                                                 │
│  ├── 接口错误：CRC错误、帧错误                                               │
│  ├── 丢包统计：入/出丢包                                                    │
│  ├── 重传率：TCP重传                                                        │
│  └── DNS解析失败：DNS错误率                                                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2. 日志（Logs）

```
日志是带时间戳的离散事件记录，回答"具体发生了什么"。

┌─────────────────────────────────────────────────────────────────────────────┐
│                              日志类型                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  网络日志：                                                                 │
│  ├── Syslog：设备系统日志                                                   │

│  ├── 流量日志：NetFlow/IPFIX                                                │
│  ├── DNS日志：解析记录                                                      │
│  └── HTTP日志：Web访问日志                                                   │
│                                                                             │
│  应用日志：                                                                 │
│  ├── 应用输出：应用级别日志                                                  │
│  ├── 错误日志：异常堆栈                                                      │
│  ├── 访问日志：HTTP请求记录                                                  │

│  结构化日志：                                                               │
```

## 四、网络可观测性实践

### 1. 指标采集

```yaml
# Prometheus网络设备监控配置
scrape_configs:
  # 网络设备SNMP采集
  - job_name: 'network-devices'
    static_configs:
      - targets:
        - 192.168.1.1  # 核心交换机
        - 192.168.1.2  # 汇聚交换机
        - 192.168.1.3  # 防火墙
    metrics_path: /snmp
    params:
      module: [if_mib]  # 接口指标
    relabel_configs:
      - source_labels: [__address__]
        target_label: instance

  # 自定义网络指标
  - job_name: 'network-custom'
    static_configs:
      - targets: ['network-exporter:9100']
    metrics_path: /metrics
```

```
关键网络指标示例：

# 接口流量
node_network_receive_bytes_total{device="eth0"}  # 入流量
node_network_transmit_bytes_total{device="eth0"} # 出流量

# 接口状态
node_network_up{device="eth0"}  # 接口状态 1=up 0=down

# 接口错误
node_network_receive_errs_total{device="eth0"}   # 入错误
node_network_transmit_errs_total{device="eth0"}  # 出错误

# 接口丢包
node_network_receive_drop_total{device="eth0"}    # 入丢包
node_network_transmit_drop_total{device="eth0"}   # 出丢包

# TCP连接状态
node_tcp_connections{state="established"}  # 已建立连接
node_tcp_connections{state="time_wait"}    # TIME_WAIT状态

# DNS查询
dns_query_duration_seconds{server="8.8.8.8"}  # DNS延迟
```

### 2. 日志采集

```yaml
# Fluentd网络日志采集配置
<source>
  @type syslog
  port 514
  bind 0.0.0.0
  tag network
  
  <parse>
    @type regexp
    expression /^<%{POSINT:priority}>%{SYSLOGTIMESTAMP:timestamp} %{HOSTNAME:host} %{GREEDYDATA:message}$/
  </parse>
</source>

<source>
  @type netflow
  port 2055
  bind 0.0.0.0
  
  <parse>
    @type netflow
  </parse>
</source>

<filter network.**>
  @type parser
  key_name message
  
  <parse>
    @type regexp
    # Cisco日志格式解析
    expression /^(?<facility>\w+)-(?<severity>\d+)-(?<mnemonic>\w+): (?<description>.*)$/
  </parse>
</filter>

<match network.**>
  @type elasticsearch
  host elasticsearch
  port 9200
  index_name network-logs
  type_name _doc
</match>
```

### 3. 告警规则

```yaml
# Prometheus网络告警规则
groups:
  - name: network_alerts
    rules:
      # 接口down告警
      - alert: InterfaceDown
        expr: node_network_up == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "接口 {{ $labels.device }} down"
          description: "设备 {{ $labels.instance }} 接口 {{ $labels.device }} 已down超过1分钟"

      # 高带宽利用率告警
      - alert: HighBandwidthUsage
        expr: |
          (rate(node_network_receive_bytes_total[5m]) + 
           rate(node_network_transmit_bytes_total[5m])) 
          / node_network_speed_bytes > 0.8
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "高带宽利用率: {{ $labels.device }}"
          description: "接口 {{ $labels.device }} 带宽利用率超过80%"

      # 高丢包率告警
      - alert: HighPacketLoss
        expr: |
          rate(node_network_receive_drop_total[5m]) > 0.01
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "高丢包率: {{ $labels.device }}"
          description: "接口 {{ $labels.device }} 丢包率异常"

      # DNS延迟告警
      - alert: HighDNSLatency
        expr: histogram_quantile(0.95, dns_query_duration_seconds) > 0.1
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "DNS查询延迟过高"
          description: "95%分位DNS延迟超过100ms"

      # TCP连接异常告警
      - alert: TCPConnectionTimeWait
        expr: node_tcp_connections{state="time_wait"} > 10000
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "TIME_WAIT连接过多"
          description: "TIME_WAIT状态连接数超过10000"
```

### 典型故障案例

```
案例1：网络延迟异常

现象：用户反馈访问应用缓慢
分析流程：
├── 指标分析：查看网络延迟指标，发现RTT从5ms上升到200ms
├── 日志分析：查看网络设备日志，发现接口错误增加
├── 追踪分析：查看分布式追踪，发现网络层延迟高
├── 根因：某交换机接口CRC错误，导致大量重传
└── 修复：更换故障光模块

案例2：间歇性连接失败

现象：偶发性连接超时
分析流程：
├── 指标分析：连接失败率在特定时段升高
├── 日志分析：防火墙日志显示连接数超限
├── 追踪分析：请求在防火墙处被丢弃
├── 根因：防火墙会话数达到上限
└── 修复：调整防火墙会话数限制

案例3：DNS解析失败

现象：部分域名解析失败
分析流程：
├── 指标分析：DNS查询失败率升高
├── 日志分析：DNS服务器日志显示解析超时
├── 追踪分析：DNS请求到达但无响应
├── 根因：上游DNS服务器故障
└── 修复：切换备用DNS服务器
```

## 七、总结

### 可观测性核心

```
三支柱：
├── 指标（Metrics）：回答"发生了什么"
├── 日志（Logs）：回答"具体是什么"
└── 追踪（Traces）：回答"问题在哪里"

核心价值：
├── 快速定位问题
├── 理解系统行为
├── 预测潜在风险
└── 优化系统性能
```

### 记忆口诀

```
指标看趋势，日志看细节，追踪看路径。
三者相结合，故障无处藏。
监控告警配，可观测性强。
主动发现问题，运维效率高。
```
