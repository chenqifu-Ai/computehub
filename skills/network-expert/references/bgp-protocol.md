# BGP边界网关协议 - 互联网路由核心

## 概述

BGP（Border Gateway Protocol，边界网关协议）是互联网的核心路由协议，负责在不同自治系统（AS）之间交换路由信息。BGP是互联网能够正常运作的关键协议。

---

### 2. BGP协议特点

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          BGP协议特点                                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  协议基础：                                                                 │
│  ├── 协议类型：EGP（Exterior Gateway Protocol）                              │
│  ├── 传输层：TCP端口179                                                      │
│  ├── 算法：路径矢量路由算法                                                   │
│  └── 版本：BGP-4（RFC 4271）                                                 │
│                                                                             │
│  核心特点：                                                                 │
│  ├── 距离矢量协议：但基于路径属性                                            │
│  ├── 无环路：通过AS_Path属性防止环路                                         │
│  ├── 增量更新：只发送变化的路由                                              │
│  ├── 策略路由：支持复杂的路由策略                                            │
│  └── 可扩展：支持IPv6、VPN（MP-BGP）                                         │
│                                                                             │
│  与IGP的区别：                                                               │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │         IGP（OSPF/IS-IS）          │        BGP                      │   │
│  │  ────────────────────────────────────────────────────────────────── │   │
│  │  内部网关协议                     │  外部网关协议                    │   │
│  │  AS内部路由                       │  AS之间路由                      │   │
│  │  最短路径优先                     │  策略优先                        │   │
│  │  链路状态/距离矢量                 │  路径矢量                        │   │
│  │  快速收敛                        │  较慢收敛                        │   │
│  │  不支持策略                       │  支持复杂策略                     │   │
│  │  度量：带宽、延迟、跳数           │  属性：AS_Path、Local_Pref等     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  BGP适用场景：                                                              │
│  ├── 运营商骨干网互联                                                        │
│  ├── 企业多宿主连接                                                          │
│  ├── 数据中心互联（DCI）                                                     │
│  └── MPLS VPN                                                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 二、BGP邻居建立

### 1. BGP邻居类型

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          BGP邻居类型                                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  IBGP（Internal BGP）：                                                      │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                     │   │
│  │      AS 100                                                        │   │
│  │  ┌─────────────────────────────────────────────────────────────┐   │   │
│  │  │           ┌─────────┐         ┌─────────┐                  │   │   │
│  │  │           │ Router A│ ◄─────▶ │ Router B│                  │   │   │
│  │  │           │ (IBGP)  │         │ (IBGP)  │                  │   │   │
│  │  │           └─────────┘         └─────────┘                  │   │   │
│  │  │              ▲                     ▲                        │   │   │
│  │  │              │      IBGP邻居       │                        │   │   │
│  │  │              └─────────────────────┘                        │   │   │
│  │  └─────────────────────────────────────────────────────────────┘   │   │
│  │                                                                     │   │
│  │  特点：                                                             │   │
│  │  ├── 同一AS内的BGP邻居                                               │   │
│  │  ├── 通过IGP（OSPF/IS-IS）学习对方接口                               │   │
│  │  ├── IBGP邻居不需要直连                                              │   │
│  │  └── IBGP不修改AS_Path、Next_Hop属性                                 │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  EBGP（External BGP）：                                                      │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                     │   │
│  │      AS 100                    AS 200                               │   │
│  │  ┌─────────────┐           ┌─────────────┐                        │   │
│  │  │  Router A   │           │  Router B   │                        │   │
│  │  │             │ ◄───────▶ │             │                        │   │
│  │  │  EBGP邻居   │           │  EBGP邻居   │                        │   │
│  │  └─────────────┘           └─────────────┘                        │   │
│  │                                                                     │   │
│  │  特点：                                                             │   │
│  │  ├── 不同AS之间的BGP邻居                                             │   │
│  │  ├── 通常需要直连（可配置多跳）                                       │   │
│  │  ├── EBGP修改AS_Path（添加本地AS号）                                 │   │
│  │  └── EBGP修改Next_Hop（修改为本地接口）                              │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

### 3. BGP消息类型

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          BGP消息类型                                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. OPEN消息                                                                │
│  ├── 作用：建立BGP邻居                                                      │
│  ├── 内容：                                                                 │
│  │   ├── BGP版本（4）                                                       │
│  │   ├── 本地AS号                                                          │
│  │   ├── Hold Time（保活时间）                                              │
│  │   ├── BGP标识符（Router ID）                                             │
│  │   └── 可选参数（能力协商）                                               │
│  └── 发送时机：TCP连接建立后                                                 │
│                                                                             │
│  2. KEEPALIVE消息                                                            │
│  ├── 作用：维持BGP邻居关系                                                   │
│  ├── 内容：仅BGP头部（19字节）                                               │
│  ├── 发送时机：Hold Time间隔内                                               │
│  └── 默认间隔：60秒                                                         │
│                                                                             │
│  3. UPDATE消息                                                               │
│  ├── 作用：通告路由信息                                                      │
│  ├── 内容：                                                                 │
│  │   ├── 撤销路由（Withdrawn Routes）                                       │
│  │   ├── 路径属性（Path Attributes）                                        │
│  │   └── 网络层可达信息（NLRI）                                             │
│  └── 增量更新：只发送变化的路由                                              │
│                                                                             │
│  4. NOTIFICATION消息                                                         │
│  ├── 作用：通知错误并终止连接                                                │
│  ├── 内容：                                                                 │
│  │   ├── 错误代码（Error Code）                                             │
│  │   ├── 错误子代码（Error Subcode）                                        │
│  │   └── 错误数据（Data）                                                   │
│  └── 发送后关闭连接                                                         │
│                                                                             │
│  消息格式：                                                                 │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                        BGP消息头部                                    │   │
│  │  ┌─────────┬─────────┬─────────┬─────────┐                         │   │
│  │  │ Marker  │ Length  │ Type    │ Data    │                         │   │
│  │  │ (16字节)│ (2字节) │ (1字节) │ (变长)  │                         │   │
│  │  └─────────┴─────────┴─────────┴─────────┘                         │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 三、BGP路径属性

### BGP属性详解

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          BGP路径属性                                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  公认必选（Well-known Mandatory）：必须包含                                  │
│  ├── ORIGIN：路由来源（IGP/EGP/Incomplete）                                  │
│  ├── AS_PATH：经过的AS序列                                                  │
│  └── NEXT_HOP：下一跳地址                                                   │
│                                                                             │
│  公认任意（Well-known Discretionary）：可选包含                               │
│  ├── LOCAL_PREF：本地优先级（IBGP）                                          │
│  └── ATOMIC_AGGREGATE：原子聚合标记                                          │
│                                                                             │
│  可选传递（Optional Transitive）：可传递给其他AS                              │
│  ├── AGGREGATOR：聚合者信息                                                  │
│  └── COMMUNITY：团体属性                                                     │
│                                                                             │
│  可选非传递（Optional Non-transitive）：不传递给其他AS                        │
│  ├── MED（Multi-Exit Discriminator）：多出口鉴别器                           │
│  ├── ORIGINATOR_ID：路由起源者                                              │
│  └── CLUSTER_LIST：集群列表                                                  │
│                                                                             │
│  属性详解：                                                                 │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                     │   │
│  │  AS_PATH：                                                           │   │
│  │  ├── 作用：防止环路，影响路径选择                                      │   │
│  │  ├── 内容：路由经过的AS号序列                                         │   │
│  │  ├── 示例：100 200 300表示从AS300→AS200→AS100                        │   │
│  │  └── 选择：优先选择AS_PATH最短的路由                                  │   │
│  │                                                                     │   │
│  │  LOCAL_PREF：                                                        │   │
│  │  ├── 作用：影响出AS的路由选择                                         │   │
│  │  ├── 范围：0-4294967295                                              │   │
│  │  ├── 默认：100                                                       │   │
│  │  └── 选择：优先选择LOCAL_PREF最大的路由                               │   │
│  │                                                                     │   │
│  │  MED：                                                               │   │
│  │  ├── 作用：影响进入AS的路由选择                                       │   │
│  │  ├── 范围：0-4294967295                                              │   │
│  │  └── 选择：优先选择MED最小的路由                                       │   │
│  │                                                                     │   │
│  │  COMMUNITY：                                                         │   │
│  │  ├── 作用：标记路由，应用策略                                         │   │
│  │  ├── 格式：AS:NN 或知名团体                                          │   │
│  │  ├── 知名团体：                                                      │   │
│  │  │   ├── NO_EXPORT：不发布给EBGP邻居                                  │   │
│  │  │   ├── NO_ADVERTISE：不发布给任何邻居                               │   │
│  │  │   └── LOCAL_AS：只发布给本地AS                                    │   │
│  │  └── 应用：基于团体应用路由策略                                       │   │
│  │                                                                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 四、BGP路由选择

### BGP选路原则

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      BGP路由选择决策（13步）                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  优先级从高到低：                                                            │
│                                                                             │
│  1. Weight（权重）- Cisco私有                                               │
│     └── 优先选择Weight最大的路由（本地有效）                                  │
│                                                                             │
│  2. LOCAL_PREF（本地优先级）                                                 │
│     └── 优先选择LOCAL_PREF最大的路由（AS内有效）                              │
│                                                                             │
│  3. 本地起源                                                                │
│     └── 优先选择本地产生的路由（network/aggregate）                          │
│                                                                             │
│  4. AS_PATH长度                                                             │
│     └── 优先选择AS_PATH最短的路由                                            │
│                                                                             │
│  5. ORIGIN类型                                                              │
│     └── IGP < EGP < Incomplete（IGP优先）                                    │
│                                                                             │
│  6. MED（多出口鉴别器）                                                      │
│     └── 优先选择MED最小的路由                                                │
│                                                                             │
│  7. EBGP优先于IBGP                                                          │
│     └── 优先选择EBGP路由                                                    │
│                                                                             │
│  8. 到下一跳的IGP度量                                                        │
│     └── 优先选择IGP度量最小的路由                                            │
│                                                                             │
│  9. 最老的路由                                                               │
│     └── 优先选择最稳定的路由（最老）                                          │
│                                                                             │
│  10. ROUTER_ID                                                              │
│      └── 优先选择ROUTER_ID最小的路由                                         │
│                                                                             │
│  11. 邻居IP地址                                                              │
│      └── 优先选择邻居IP地址最小的路由                                         │
│                                                                             │
│  记忆口诀：                                                                 │
│  "Weight Local 原产地，路径Origin Med，                                      │
│   EBGP比IBGP好，IGP到下一跳，老路由优先，                                     │
│   最后比RouterID和邻居IP"                                                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 五、BGP配置示例

### 基本BGP配置

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          BGP配置示例（Cisco）                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ! 基本BGP配置                                                               │
│  router bgp 65001                                                           │
│   bgp router-id 1.1.1.1                                                     │
│   bgp log-neighbor-changes                                                  │
│   neighbor 192.168.1.2 remote-as 65002                                      │
│   neighbor 192.168.1.2 description "ISP-A"                                 │
│   neighbor 10.1.1.2 remote-as 65001                                          │
│   neighbor 10.1.1.2 description "IBGP-peer"                                 │
│   neighbor 10.1.1.2 update-source Loopback0                                 │
│   neighbor 10.1.1.2 next-hop-self                                           │
│   !                                                                          │
│   address-family ipv4                                                       │
│    network 192.168.0.0 mask 255.255.0.0                                     │
│    neighbor 192.168.1.2 activate                                            │
│    neighbor 10.1.1.2 activate                                               │
│   exit-address-family                                                       │
│                                                                             │
│  ! 路由策略配置                                                               │
│  ip prefix-list ALLOW-OUT permit 192.168.0.0/16 le 32                       │
│  !                                                                          │
│  route-map ISP-OUT permit 10                                                │
│   match ip address prefix-list ALLOW-OUT                                    │
│   set as-path prepend 65001 65001                                           │
│  !                                                                          │
│  router bgp 65001                                                           │
│   neighbor 192.168.1.2 route-map ISP-OUT out                               │
│                                                                             │
│  配置说明：                                                                 │
│  ├── router bgp 65001：启动BGP进程，AS号为65001                               │
│  ├── neighbor remote-as：配置BGP邻居                                         │

## 六、BGP最佳实践

### 1. IBGP全互联问题

```
IBGP全互联问题：

问题：IBGP邻居需要全互联（Full Mesh）
├── N个路由器需要 N(N-1)/2 个IBGP邻居
├── 扩展性差
└── 配置复杂

解决方案：

┌─────────────────────────────────────────────────────────────────────────────┐
│                          IBGP扩展方案                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  方案1：路由反射器（Route Reflector）                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                     │   │
│  │         ┌─────────────────┐                                         │   │
│  │         │      RR         │ ◀── 路由反射器                          │   │
│  │         │  (Route Reflector)│                                       │   │
│  │         └────────┬────────┘                                         │   │
│  │                  │                                                   │   │
│  │        ┌─────────┼─────────┐                                        │   │
│  │        ▼         ▼         ▼                                        │   │
│  │  ┌─────────┐ ┌─────────┐ ┌─────────┐                               │   │
│  │  │ Client  │ │ Client  │ │ Client  │ ◀── 客户端                    │   │   │
│  │  │         │ │         │ │         │                               │   │
│  │  └─────────┘ └─────────┘ └─────────┘                               │   │
│  │                                                                     │   │
│  │  原理：                                                              │   │
│  │  ├── RR将从Client学到的路由反射给其他Client                           │   │
│  │  ├── Client只需要与RR建立IBGP邻居                                     │   │
│  │  └── 无需全互联                                                       │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  方案2：BGP联盟（BGP Confederation）                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                     │   │
│  │  AS 100                                                             │   │
│  │  ┌─────────────────────────────────────────────────────────────┐   │   │
│  │  │                                                             │   │   │
│  │  │  子AS 65001          子AS 65002                             │   │   │
│  │  │  ┌──────────┐        ┌──────────┐                          │   │   │
│  │  │  │ Router A │────────│ Router D │                          │   │   │
│  │  │  │ Router B │        │ Router E │                          │   │   │
│  │  │  │ Router C │        │ Router F │                          │   │   │
│  │  │  └──────────┘        └──────────┘                          │   │   │
│  │  │  │ 全互联 │          │ 全互联 │                            │   │   │
│  │  │                                                             │   │   │
│  │  └─────────────────────────────────────────────────────────────┘   │   │
│  │                                                                     │   │
│  │  原理：                                                              │   │
│  │  ├── 将大AS分为多个子AS                                               │   │
│  │  ├── 子AS内部使用IBGP全互联                                           │   │
│  │  ├── 子AS之间使用EBGP                                                │   │
│  │  └── 对外仍表现为一个大AS                                             │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 八、总结

### BGP核心要点

```
1. BGP是互联网路由的核心协议
2. 基于TCP，端口179，使用路径矢量算法
3. 通过AS_PATH防止环路
4. 支持复杂的路由策略
5. IBGP需要全互联或使用RR/联盟
6. 路由选择遵循13步决策过程
```

### 记忆口诀

```
BGP协议要记牢，AS之间路由跑。
邻居建立五步走，OPEN消息先发送。
路径属性很重要，AS_PATH防环路。
选路规则十三条，Weight最高来决断。
安全配置不能少，认证过滤要做好。
```
