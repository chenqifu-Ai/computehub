# SDN/NFV - 软件定义网络与网络功能虚拟化

## 概述

SDN（Software Defined Networking）和NFV（Network Function Virtualization）是网络领域最重要的两个变革性技术，它们共同推动着网络架构向软件化、虚拟化方向发展。

---

## 一、SDN（软件定义网络）

### 1. 核心思想

**将控制平面与数据平面分离**：
- 控制平面：集中式控制器，负责网络决策
- 数据平面：分布式交换机，负责数据转发
- 通过开放接口（如OpenFlow）实现南北向通信

### 2. SDN架构三层模型

```
┌─────────────────────────────────────┐
│         应用层（Application）        │
│   网络应用：路由、安全、负载均衡等      │
└─────────────────────────────────────┘
                    ↕ 北向API（RESTful）
┌─────────────────────────────────────┐
│         控制层（Controller）         │
│   SDN控制器：OpenDaylight、ONOS等     │
└─────────────────────────────────────┘
                    ↕ 南向API（OpenFlow）
┌─────────────────────────────────────┐
│         基础设施层（Infrastructure）   │
│   物理交换机、虚拟交换机、路由器        │
└─────────────────────────────────────┘
```

### 3. 主流SDN控制器

| 控制器 | 开发商 | 特点 |
|--------|--------|------|
| OpenDaylight | Linux基金会 | 企业级、模块化、可扩展 |
| ONOS | ON.Lab | 运营商级、高可用 |
| Floodlight | Apache | 轻量级、易学习 |
| Ryu | NTT | Python开源、灵活 |

### 4. SDN核心优势

1. **集中管理**：全网视图，统一策略
2. **灵活编程**：通过API快速部署新服务
3. **降低成本**：通用硬件替代专用设备
4. **快速创新**：网络功能软件化，迭代更快

---

## 二、NFV（网络功能虚拟化）

### 1. 核心思想

**将网络功能从专用硬件解耦，以软件形式运行在通用服务器上**：
- 使用x86服务器替代专用网络设备
- 网络功能虚拟化为VNF（Virtual Network Function）
- 运行在虚拟机或容器中

### 2. NFV架构

```
┌─────────────────────────────────────────────────┐
│                 NFV编排器（NFVO）                │
│        负责VNF生命周期管理、资源编排              │
└─────────────────────────────────────────────────┘
                      ↕
┌─────────────────────────────────────────────────┐
│                VNFM（VNF管理器）                  │
│        管理VNF实例的创建、扩缩容、终止            │
└─────────────────────────────────────────────────┘
                      ↕
┌─────────────────────────────────────────────────┐
│            VIM（虚拟基础设施管理器）              │
│        OpenStack、VMware vSphere等              │
└─────────────────────────────────────────────────┘
                      ↕
┌─────────────────────────────────────────────────┐
│         NFV基础设施（NFVI）                      │
│   计算节点、存储节点、网络节点（COTS硬件）         │
└─────────────────────────────────────────────────┘
```

### 3. 常见VNF类型

| VNF类型 | 功能 | 传统设备 |
|---------|------|----------|
| vEPC | 核心网 | 华为/爱立信EPC |
| vIMS | 语音核心 | 传统的IMS设备 |
| vCPE | 用户端设备 | 传统CPE盒子 |
| vFW | 防火墙 | 专用防火墙 |
| vLB | 负载均衡 | F5等负载均衡器 |
| vCDN | 内容分发 | 传统CDN节点 |

### 4. NFV核心优势

1. **降低CAPEX**：通用硬件成本低
2. **降低OPEX**：自动化运维
3. **快速部署**：软件部署分钟级
4. **弹性扩展**：按需扩缩容
5. **位置灵活**：VNF可部署在任意数据中心

## 三、SDN与NFV的关系

### 互补而非替代

| 维度 | SDN | NFV |
|------|-----|-----|
| 目标 | 网络控制集中化 | 网络功能虚拟化 |
| 关键技术 | 控制器、OpenFlow | 虚拟机、容器、编排 |
| 应用场景 | 数据中心、园区网 | 核心网、边缘计算 |
| 硬件要求 | 支持OpenFlow的交换机 | x86通用服务器 |

### 协同场景

1. **数据中心网络**：SDN控制 + NFV提供网络服务
2. **5G核心网**：NFV虚拟化核心网 + SDN控制网络切片
3. **边缘计算**：NFV部署边缘应用 + SDN管理边缘网络

## 四、实际应用案例

### 1. 运营商核心网云化

```
传统核心网                    云化核心网（SDN+NFV）
┌──────────┐                 ┌──────────────┐
│ 专用硬件  │                 │ 通用x86服务器 │
│ MME/SGW  │      →          │  vEPC        │
│ HSS/PCRF │                 │  (VNF容器)   │
└──────────┘                 └──────────────┘
                                   ↑
                            SDN控制器管理
```

### 2. 企业园区网络

- SDN统一管理接入策略
- NFV提供防火墙、负载均衡等网络服务
- 实现"软件定义园区网络"

### 3. 数据中心网络

- SDN实现网络自动化、流量工程
- NFV提供虚拟负载均衡、虚拟防火墙
- 支持多租户网络隔离

## 五、关键技术要点

### 1. OpenFlow协议

```
消息类型：
- Controller-to-Switch：控制器下发流表
- Asynchronous：交换机上报事件
- Symmetric：握手、心跳
```

### 2. 服务链（Service Chaining）

```
传统流量：用户 → 防火墙 → IDS → 负载均衡 → 应用服务器

SDN/NFV服务链：
用户 → vFW(vNF1) → vIDS(vNF2) → vLB(vNF3) → 应用
      (SDN编排流量路径，VNF按需串接)
```

### 3. 网络切片（Network Slicing）

```
物理网络资源切片：
┌─────────────────────────────────────┐
│           物理网络基础设施            │
├──────────────┬──────────────────────┤
│   切片1      │   切片2    │  切片3  │
│   eMBB       │   URLLC    │  mMTC   │
│   增强移动    │   超低时延  │  大连接 │
└──────────────┴────────────┴─────────┘
```

## 七、面试要点

### 常见问题

1. **SDN和NFV的区别是什么？**
   - SDN关注控制平面集中化
   - NFV关注网络功能虚拟化
   - 两者互补，可协同部署

2. **SDN的南向接口有哪些？**
   - OpenFlow（最主流）
   - NETCONF/YANG
   - P4（可编程数据平面）

3. **NFV为什么要引入MANO？**
   - 管理复杂的VNF生命周期
   - 实现资源自动化编排
   - 提供统一的运维接口

4. **什么是VNF转发图？**
   - 定义VNF的串联顺序
   - 实现服务链功能
   - SDN控制器负责流量引导
