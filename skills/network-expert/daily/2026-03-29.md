**调查问题**: OpenClaw cron配置错误  
**专家**: 网络专家

---

## 📋 问题摘要

**错误信息**:
```
cron channel delivery config is only supported for sessionTarget="isolated"
```

**问题背景**:
- 尝试创建每10分钟的"公司脉搏"监控任务
- 配置使用 `sessionTarget="main"` + `delivery.mode="webhook"`
- Gateway报错，任务创建失败

---

## 🔍 原因分析

### 1. 核心问题：sessionTarget 与 delivery 的兼容性问题

| sessionTarget | 支持的 delivery.mode | 说明 |
|--------------|-------------------|------|
| `"isolated"` | `announce`, `webhook`, `none` | 完整支持 |
| `"main"` | `none` only | 不支持 webhook |

### 2. 为什么会出现这个限制？

**技术架构原因**:

**关键区别**:
- `isolated`: 任务在独立会话运行，完成后需要"delivery"机制将结果传回
- `main`: 任务直接在main session执行，本身就是主会话的一部分，无需delivery

### 3. 错误配置分析

**问题配置**:
```json
{
  "sessionTarget": "main",
  "payload": { "kind": "systemEvent", ... },
  "delivery": { "mode": "webhook" }  // ❌ main不支持
}
```

**问题所在**:
- sessionTarget="main" 表示在主会话中执行
- 但同时又配置了delivery，这是矛盾的
- main session的任务直接注入到主会话，不需要delivery机制

---

## ✅ 解决方案

### 方案A：改为 isolated session（推荐）

```json
{
  "name": "公司脉搏监控",
  "sessionTarget": "isolated",
  "payload": {
    "kind": "agentTurn",
    "message": "执行公司脉搏检查..."
  },
  "delivery": {
    "mode": "announce"
  },
  "schedule": {
    "kind": "every",
    "everyMs": 600000
  }
}
```

**优点**:
- 独立执行，不影响主会话
- 支持delivery输出
- 适合定期监控任务

### 方案B：保持 main session，移除 delivery

```json
{
  "name": "公司脉搏监控",
  "sessionTarget": "main",
  "payload": {
    "kind": "systemEvent",
    "text": "脉搏检查 - 请运行监控脚本"
  },
  "schedule": {
    "kind": "every",
    "everyMs": 600000
  }
}
```

## 🎯 结论与建议

### 问题结论:
cron配置错误是因为 **sessionTarget="main" 与 delivery="webhook" 不兼容**。main session的任务直接注入主会话，不需要额外的delivery机制。

### 修复建议:
1. **首选方案**: 改为 `sessionTarget="isolated"` + `payload.kind="agentTurn"`
2. **备选方案**: 保持main但移除delivery配置
3. **监控任务推荐isolated**: 避免阻塞主会话，支持独立超时控制

### 严重程度: 中等
这是一个**配置问题**，不是系统故障。只需要调整cron配置即可解决。
