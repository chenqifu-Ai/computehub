# 📜 Git 记忆铁律

## 铁律一：先 Git 查，再操作

### 查找文件
```bash
# ✅ 正确：先 Git 查
git ls-files | grep 关键词
git grep -l 关键词

# ❌ 错误：猜路径
cat /path/to/file.md
```

### 检索项目
```bash
# ✅ 正确：按项目检索
ls projects/
git ls-files projects/
git grep -l "关键词" projects/

# ❌ 错误：盲目搜索
find . -name "*computehub*"
```

## 铁律二：每次修改必须 commit

```bash
# ✅ 正确：修改后立即 commit
git add . && git commit -m "更新：文件名 - 变更说明"

# ❌ 错误：只修改不 commit
# 文件已修改但没有记录
```

## 铁律三：项目必须纳入 Git

```bash
# ✅ 正确：创建新项目后立即纳入
mkdir projects/new-project
git add projects/new-project
git commit -m "init: 项目名 - 初始化"

# ❌ 错误：项目散落在各处
# 文件在 ai_agent/code/ 和 references/ 之间游走
```

## 铁律四：目录结构必须规范

```
# ✅ 正确：标准结构
workspace/
├── projects/           # 项目目录
│   ├── INDEX.md        # 项目索引
│   └── project-name/   # 项目目录
├── memory/             # 记忆目录
│   ├── daily/          # 每日记忆
│   └── topics/         # 主题记忆
└── scripts/            # 工具脚本

# ❌ 错误：文件散乱
# 100+ 文件直接放在根目录
```

## 铁律五：善用搜索工具

```bash
# 关键词搜索 - 0.24 秒
python3 scripts/git-memory-search.py keyword "关键词"

# 提交搜索 - <0.1 秒
python3 scripts/git-memory-search.py commit "提交关键词"

# 时间搜索 - 秒级
python3 scripts/git-memory-search.py time --since="2026-04-20"

# Git grep - 极快
git grep -l "关键词"
```

## 铁律六：定期快照与备份

```bash
# 每日快照
python3 scripts/git-memory-manager.py snapshot

# 手动备份
git bundle create workspace-bundle.bundle --all
```
