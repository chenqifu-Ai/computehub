# 数据合规指南

## 概述

数据合规是指企业在收集、存储、使用、加工、传输、提供、公开、删除个人数据和企业数据过程中，遵守相关法律法规、行业标准和企业内部制度的管理活动。随着《个人信息保护法》《数据安全法》等法律的实施，数据合规已成为企业必须重视的法律风险领域。

---

## 一、核心法规框架

### 1. 三大基础法律

| 法律 | 施行时间 | 核心要点 |
|-----|---------|---------|
| 《网络安全法》 | 2017年6月 | 网络运营者义务、数据本地化 |
| 《数据安全法》 | 2021年9月 | 数据分级分类、安全保护义务 |
| 《个人信息保护法》 | 2021年11月 | 个人信息处理规则、权利保障 |

### 2. 配套法规与标准
- 《网络安全审查办法》
- 《数据出境安全评估办法》
- 《个人信息安全规范》(GB/T 35273)
- 《关键信息基础设施安全保护条例》

## 二、个人信息保护要点

### 1. 个人信息的定义
**个人信息**：以电子或者其他方式记录的与已识别或者可识别的自然人有关的各种信息。

**敏感个人信息**：包括生物识别、宗教信仰、特定身份、医疗健康、金融账户、行踪轨迹等信息，以及不满十四周岁未成年人的个人信息。

### 2. 处理个人信息的原则
1. **合法、正当、必要原则**：不得过度收集
2. **目的明确原则**：明确告知处理目的
3. **最小够用原则**：收集范围限于必要
4. **公开透明原则**：告知处理规则
5. **质量保证原则**：确保信息准确完整
6. **安全保护原则**：采取安全措施

### 3. 处理个人信息的合法依据
- 取得个人同意
- 为订立合同所必需
- 为履行法定职责所必需
- 为应对突发公共卫生事件
- 为公共利益实施新闻报道
- 法律法规规定的其他情形

### 4. 个人信息处理者的义务

```
┌────────────────────────────────────────────────────────────┐
│                    个人信息处理义务                          │
├────────────────────────────────────────────────────────────┤
│ ▸ 制定内部管理制度和操作规程                                 │
│ ▸ 对个人信息实行分类管理                                     │
│ ▸ 采取相应的加密、去标识化等安全技术措施                      │
│ ▸ 合理确定个人信息处理的操作权限                              │
│ ▸ 制定并组织实施个人信息安全事件应急预案                      │
│ ▸ 定期对从业人员进行安全教育和培训                           │
│ ▸ 定期对其个人信息处理活动进行合规审计                        │
│ ▸ 发生个人信息泄露时立即采取补救措施并通知监管部门             │
└────────────────────────────────────────────────────────────┘
```

## 三、数据分级分类

### 1. 分级标准

| 级别 | 定义 | 保护要求 |
|-----|------|---------|
| 一般数据 | 泄露后影响较小 | 基本保护措施 |
| 重要数据 | 泄露后可能影响公共利益 | 重点保护，定期评估 |
| 核心数据 | 泄露后严重危害国家安全、国民经济运行 | 最高级别保护 |

### 2. 分类维度
- 按主体：个人信息、企业数据、公共数据
- 按内容：基础信息、交易信息、行为信息
- 按敏感度：一般信息、敏感信息、特别敏感信息

### 3. 分类管理要求
- 建立数据分类目录
- 设置不同级别的访问权限
- 采取差异化的安全措施
- 定期评审分类结果

## 四、数据出境合规

### 1. 数据出境方式
- **数据跨境传输**：通过网络将数据传输至境外
- **境外访问**：境外主体访问境内数据
- **数据出境处理**：在境外对境内数据进行处理

### 2. 合规路径（三选一）

| 路径 | 适用场景 | 流程 |
|-----|---------|-----|
| 安全评估 | 处理重要数据、CII运营者、大量个人信息 | 向国家网信部门申报 |
| 个人信息保护认证 | 非关键场景、一般个人信息 | 通过认证机构认证 |
| 标准合同 | 一般个人信息，非重要数据 | 签署标准合同并备案 |

### 3. 不得出境的情形
- 关键信息基础设施运营者处理重要数据
- 处理个人信息达到一定规模
- 处理敏感个人信息
- 可能影响国家安全、公共利益

## 五、企业数据合规要点

### 1. 组织架构
```
┌─────────────────────────────────────────┐
│           董事会/管理层                  │
│         （数据安全决策）                  │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│         数据保护官(DPO)                   │
│    （监管对接、制度制定）                  │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│         数据安全部门                      │
│    （技术实施、日常管理）                  │
└─────────────────────────────────────────┘
```

### 2. 制度建设
- **数据安全管理制度**：总体框架
- **个人信息保护制度**：处理规则、权利响应
- **数据分类分级制度**：分类标准、管理措施
- **数据安全事件应急预案**：事件响应流程
- **数据出境管理制度**：出境评估、审批流程

### 3. 技术措施
- **访问控制**：身份认证、权限管理
- **加密存储**：敏感数据加密
- **脱敏处理**：开发测试环境脱敏
- **日志审计**：操作日志记录
- **安全监测**：异常行为检测

### 4. 合规审计
- 定期开展数据安全风险评估
- 每年至少进行一次合规审计
- 建立审计发现问题的整改机制
- 保留审计记录至少3年

## 六、常见违规风险

### 1. 过度收集个人信息
- **表现**：收集与业务无关的个人信息
- **风险**：行政处罚、用户投诉、声誉损失
- **整改**：最小必要原则，定期清理冗余数据

### 2. 未履行告知义务
- **表现**：未发布隐私政策、未告知处理目的
- **风险**：监管处罚、民事赔偿
- **整改**：完善隐私政策、弹窗告知

### 3. 数据泄露事件
- **表现**：系统漏洞导致数据泄露
- **风险**：监管处罚、刑事责任、民事赔偿
- **整改**：加强安全措施、建立应急响应机制

### 4. 违规数据出境
- **表现**：未经评估将数据传输至境外
- **风险**：责令整改、罚款、业务暂停
- **整改**：完成安全评估或认证

## 七、法律责任

### 1. 行政责任

| 违法行为 | 处罚措施 |
|---------|---------|
| 违法处理个人信息 | 责令改正、警告、没收违法所得 |
| 情节严重 | 罚款5000万元或年收入5% |
| 拒不改正 | 责令暂停业务、停业整顿 |
| 直接责任人员 | 罚款10-100万元 |

### 2. 民事责任
- 侵害个人信息权益造成损害的，承担赔偿责任
- 推定过错责任：处理者不能证明自己无过错的，承担侵权责任

### 3. 刑事责任
- 侵犯公民个人信息罪：情节严重的，处三年以下有期徒刑
- 情节特别严重的，处三年以上七年以下有期徒刑

## 八、合规建议清单

### 小微企业数据合规基础清单

**必须做**：
- [ ] 制定并发布隐私政策
- [ ] 明确个人信息收集范围（最小必要）
- [ ] 获得用户明确同意
- [ ] 建立数据安全管理制度
- [ ] 指定数据安全负责人

**应该做**：
- [ ] 对数据进行分类分级
- [ ] 定期进行安全培训
- [ ] 建立数据泄露应急预案
- [ ] 定期备份数据

**最好做**：
- [ ] 开展合规审计
- [ ] 获得安全认证
- [ ] 建立数据治理平台

## 总结

数据合规已成为企业法律风险管理的重点领域。企业应：

1. **建立意识**：数据安全是法律底线
2. **完善制度**：从制度层面规范数据处理
3. **技术保障**：用技术手段落实安全措施
4. **持续改进**：定期审计、动态调整

**核心原则**：合法、正当、必要、最小够用、公开透明。
