# gemma-4-31b 测试用例集

## 1. 基础能力测试
### 逻辑推理
- **用例 1.1 (经典逻辑)**: 
  - 输入: "如果所有 A 都是 B，且所有 B 都是 C，那么所有 A 都是 C 吗？请解释推理过程。"
  - 预期: 正确回答 "是"，并给出三段论推理。
- **用例 1.2 (情境推理)**: 
  - 输入: "房间里有 3 个灯泡，门外有 3 个开关。你只能进入房间一次，如何确定哪个开关对应哪个灯泡？"
  - 预期: 提供经典的“加热灯泡”解决方案。

### 数学计算
- **用例 2.1 (代数)**: 
  - 输入: "求解方程: 3x + 5 = 2x - 10"
  - 预期: 正确步骤并得出 x = -15。
- **用例 2.2 (组合数学)**: 
  - 输入: "从 10 个不同的球中随机抽取 3 个，有多少种组合方式？"
  - 预期: C(10,3) = 120。

### 代码编写
- **用例 3.1 (Python)**: 
  - 输入: "用 Python 写一个快速排序算法，并包含测试用例。"
  - 预期: 正确实现 QuickSort，时间复杂度 O(n log n)。
- **用例 3.2 (Go)**: 
  - 输入: "用 Go 语言实现一个并发的 Web 爬虫骨架，使用 channel 进行通信。"
  - 预期: 正确使用 goroutine 和 channel。
- **用例 3.3 (JS)**: 
  - 输入: "用 JavaScript 实现一个深拷贝函数，处理循环引用。"
  - 预期: 使用 Map 或 WeakMap 处理循环引用。

### 多语言翻译与理解
- **用例 4.1 (中英互译)**: 
  - 输入: "将这句话翻译成英文：'在这个瞬息万变的时代，唯有持续学习才能保持竞争力'。"
  - 预期: 自然流畅的英文翻译（如 "In this fast-changing era, only continuous learning can maintain competitiveness"）。
- **用例 4.2 (俚语理解)**: 
  - 输入: "解释英文俚语 'Piece of cake' 的含义，并举例。"
  - 预期: 解释为“极容易的事”。

## 2. 指令遵循测试
### 复杂多步骤指令
- **用例 5.1**: 
  - 输入: "1. 总结这段文本[输入一段文本]；2. 将总结内容翻译成法文；3. 用 JSON 格式输出，键名为 'summary' 和 'translation'。"
  - 预期: 严格执行三步，输出合法 JSON。

### 格式约束
- **用例 6.1**: 
  - 输入: "请列出 5 个编程语言，要求：必须使用 Markdown 表格，列名为 '语言'、'创始人'、'发布年份'，且年份必须按降序排列。"
  - 预期: 正确表格，正确降序。
