# 智能体管理体系专项能力测试方案

## 1. 测试目标
验证 gemma-4-31b 在 OpenClaw 架构中作为“调度中枢 (Dispatcher/Manager)”的可靠性，重点考察其在复杂任务编排、状态追踪、冲突解决及资源调度方面的能力。

## 2. 模拟场景设计

### 场景 A：企业投资价值分析（编排能力测试）
- **任务描述**：分析 A 公司的年度财报 $\rightarrow$ 评估法律与合规风险 $\rightarrow$ 结合市场竞争制定投资建议 $\rightarrow$ 生成最终报告。
- **预期调度路径**：
    1. 调用 `finance-expert` (财务专家) $\rightarrow$ 提取财报关键指标。
    2. 调用 `legal-advisor` (法律顾问) $\rightarrow$ 分析潜在诉讼或监管风险。
    3. 调用 `ceo-advisor` (CEO 顾问) $\rightarrow$ 进行战略定位与投资建议。
    4. 主智能体汇总 $\rightarrow$ 生成报告。
- **测试点**：任务拆解的逻辑性、专家调用的准确性、数据传递的连续性。

### 场景 B：多渠道市场推广方案（状态追踪与同步测试）
- **任务描述**：同时启动三个子任务：1. SEO 关键词分析；2. 社媒内容矩阵规划；3. 竞品广告投放分析。要求在所有结果返回后汇总成方案。
- **模拟干扰**：子任务 1 快速返回，子任务 2 延迟返回且结果碎片化，子任务 3 返回错误提示。
- **测试点**：能否维持并行状态表、是否能识别子任务 3 的失败并尝试追溯/重试、汇总结果的完整性。

### 场景 C：产品定价策略争议（冲突处理与纠偏测试）
- **任务描述**：针对新产品定价，邀请营销专家和财务专家给出建议。
- **模拟冲突**：
    - `marketing-expert`：建议低价渗透，快速抢占市场。
    - `finance-expert`：建议高价定位，确保毛利率和品牌溢价。
- **测试点**：模型是否能识别冲突 $\rightarrow$ 尝试分析两者逻辑 $\rightarrow$ 引入 `ceo-advisor` 作为仲裁者或请求用户决策。

### 场景 D：紧急安全漏洞修复（资源调度意识测试）
- **任务描述**：发现生产环境高危漏洞 $\rightarrow$ 需要快速分析影响范围 $\rightarrow$ 深度推演漏洞利用链路 $\rightarrow$ 制定修复计划。
- **模拟需求**：
    - 步骤 1 (分析范围)：要求极速响应 $\rightarrow$ 预期使用 `gemma-4-31b`。
    - 步骤 2 (深度推演)：要求极高逻辑严密性 $\rightarrow$ 预期切换至 `DeepSeek-v3.1`。
- **测试点**：能否根据任务性质（速度 vs 深度）主动选择模型，而非单一使用默认模型。

## 3. 评估指标
| 维度 | 指标 | 评分标准 (1-5) |
| :--- | :--- | :--- |
| **编排能力** | 拆解正确率 | 任务步骤是否完整，专家匹配是否精准 |
| **状态追踪** | 汇总完整度 | 碎片化结果是否丢失，缺失环节是否被识别 |
| **冲突处理** | 解决逻辑性 | 是否能客观分析冲突并采取合理纠偏手段 |
| **资源调度** | 模型选择率 | 是否在关键深度思考环节切换至强模型 |
