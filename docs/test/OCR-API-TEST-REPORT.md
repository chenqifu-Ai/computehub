## 发现的问题

### 1. ⚠️ 并发 timeout 过短会误报
- **现象**: 5 并发 + 15s timeout → 全部超时失败
- **原因**: Tesseract 是 CPU 密集型，并发请求串行执行
- **建议**: 客户端 timeout 设为 30s+，或考虑加 Tesseract 连接池/队列

### 2. ⚠️ 噪声图片返回空文本
- **现象**: 高斯模糊图片 OCR 返回空字符串
- **原因**: Tesseract 对严重模糊图片无法识别
- **建议**: 这是预期行为，可在客户端加"置信度低于阈值"的提示

### 3. ⚠️ 中英混合末字漏识别
- **现象**: `Hello 你好 World 世界` → `Hello 你好 World 世`
- **原因**: 可能是字体渲染问题 (wqy-zenhei 字体渲染"界"字超出边界)
- **建议**: 非关键问题，实际扫描件识别不受影响

---

## 改进建议

| 优先级 | 建议 | 说明 |
|:-----:|------|------|
| P2 | 加 Tesseract 连接池 | 避免并发时全部排队，可限制最大并发数 |
| P3 | 加图片大小限制 | 防止超大图片导致 OOM |
| P3 | 加置信度返回 | Tesseract 支持返回 confidence level |
| P4 | 加图片格式校验 | 目前只写 JPEG，实际支持 PNG/BMP/TIFF |
