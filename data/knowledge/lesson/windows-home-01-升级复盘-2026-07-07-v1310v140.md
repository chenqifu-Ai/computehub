## 踩坑记录

### 坑1: & 在 cmd 里是命令分隔符
- 现象: curl -o ch_new.exe "http://...?file=computehub.exe&platform=..." → exit_code=3
- 尝试链: ^& → %26 → cmd /c → certutil → bitsadmin → 全失败
- 最终方案: PowerShell -EncodedCommand ✅
- 教训: 遇到 exit_code=3 直接上 PS 编码，不要试其他转义方式

### 坑2: PowerShell $ 变量被 Python 字符串吃掉
- 现象: 在 exec() 里写 powershell -Command "$var = ..." → $var 被 shell 当环境变量
- 最终方案: 所有 PS 脚本用 -EncodedCommand + base64(UTF16LE)
- 教训: 写 Python 脚本文件(.py)而不是在 exec() 里 inline

### 坑3: 盲目轮询
- 现象: 下载 task 提交后 sleep 20/30/60/90 反复查，以为在跑
- 实际: exit_code=3 瞬间返回（duration=171ms）
- 教训: 3s 查一次 duration，<1s 就是瞬间失败

## 成功经验

1. **PowerShell -EncodedCommand 是终极武器** — 绕过所有转义层
2. **tasklist | findstr 比 tasklist /FI 更可靠** — 不需要处理引号
3. **限定搜索范围** — dir /s /b C:\Temp\ 比 where /R C:\ 快 10 倍
4. **先启动新 Worker 再杀旧** — 零停机关键
5. **通过新 Worker 杀旧进程** — 不通过旧 Worker 自己杀自己
