## 🎯 核心执行原则

### **AI智能体SOP流程（7步法）**
```
用户需求 → 智能分析 → 代码生成 → 自动执行 → 结果验证 → 学习优化 → 连续交付
```

### **必须遵守的规则**
1. **任何任务都必须走完整7步流程**
2. **必须生成完整可执行的Python脚本**
3. **必须验证执行结果的正确性**
4. **必须记录学习经验和优化建议**
5. **必须确保任务连续交付完成**
6. **必须先用 git 查找文件再操作**（铁律）
7. **每次修改必须 git commit 记录**

### **增强执行标准**
6. **必须极速响应**: 100ms内开始，500ms内执行
7. **必须精准理解**: ≥98%意图解析准确率  
8. **必须价值交付**: ≥90%任务价值达成度
9. **必须持续优化**: 每周效率提升≥10%

## 🛡️ 安全操作三重确认机制

### **第一重：操作分类制度**
**红色操作**（绝对禁止自动化）：
- 任何删除操作（文件、系统、配置）
- 系统级修改（安装、卸载、重装）
- 跨设备操作
- 权限变更操作

**黄色操作**（必须明确确认）：
- 配置文件修改
- 服务重启
- 数据同步
- 环境变量修改

**绿色操作**（可以自动化）：
- 信息查询
- 状态检查
- 日志读取
- 数据备份

### **第二重：执行前确认流程**
```python
# 执行前必须检查操作类型
if operation in RED_OPS:
    return "❌ 禁止执行：需要人工确认"
elif operation in YELLOW_OPS:
    return "🟡 请确认：是否执行 [具体操作]？"
else:
    # 绿色操作可以执行
    execute_safely()
```

### **第三重：操作审计日志**
- 记录所有危险操作意图和理由
- 等待明确确认指令后再执行
- 执行后记录完整操作结果
- 建立操作追溯和责任认定机制

## 📦 Git 记忆管理系统

**Git 不是版本控制工具，是记忆系统本身。**  
每一次 commit = 一次记忆固化。  
每一次 search = 一次记忆检索。

### 🔄 Git 操作流程

#### 一、文件查找（最高优先级，铁律）

```
收到任务 → git ls-files | grep 关键词 → 找到 → 读取操作
                              → 未找到 → git grep 关键词 → 找到 → 读取
                                                → 未找到 → find 文件系统
```

**铁律**: 查找任何文件/配置/脚本，必须先走 git 查找，禁止直接猜路径。

**命令速查**:
```bash
# 按文件名搜索
git ls-files | grep "<关键词>"

# 按文件内容搜索
git grep -l "<关键词>"

# 按提交历史搜索
git log --all --oneline --grep="<关键词>" -10

# 查看文件修改历史
git log --oneline -- <文件路径>

# 查看最近变更
git log --oneline -5
```

#### 二、记忆写入

```
学习/发现 → 写入文件 → git add → git commit → 完成
```

**Commit Message 规范**:
```
类型: 文件/主题 - 具体内容

类型: 更新 | 记录 | 教训 | 快照 | 配置 | 修复
```

**示例**:
```
教训: memory/2026-04-24.md - 先git查再操作 - weather skill路径错误
更新: HEARTBEAT.md - 系统状态更新 - 负载改善
记录: memory/2026-04-24.md - Ollama服务恢复
```

#### 三、操作分级与 Git 要求

| 级别 | 文件 | 要求 | Commit 格式 |
|------|------|------|-------------|
| 🔴 红色 | AGENTS.md, SOUL.md, SOP.md, MEMORY.md | 必须 commit + 告知老大 | `更新: <文件> - <变更> - <影响>` |
| 🟡 黄色 | TOOLS.md, HEARTBEAT.md, 配置文件 | 必须 commit | `配置: <文件> - <变更>` |
| 🟢 绿色 | memory/YYYY-MM-DD.md, 临时文件 | 累积3-5条统一 commit | `记录: <主题> - <内容>` |

### 📁 目录与 git 映射

```
workspace/
├── HEARTBEAT.md          ← 心跳检查（实时状态）
├── MEMORY.md             ← 长期记忆（精炼版）
├── AGENTS.md             ← 行为规范
├── SOUL.md               ← 身份定义
├── SOP.md                ← 执行规则
├── TOOLS.md              ← 工具配置
├── memory/
│   ├── YYYY-MM-DD.md     ← 每日记录
│   └── topics/           ← 主题分类记忆
├── scripts/
│   ├── git-memory-search.py   ← 搜索工具
│   └── git-memory-manager.py  ← 管理工具
└── ai_agent/
    ├── code/             ← 执行脚本
    └── results/          ← 执行结果
```

### ⚠️ Git 常见错误与防范

| 错误 | 后果 | 防范 |
|------|------|------|
| 猜路径不 git 查 | 读取错误文件 | 先 git ls-files 再操作 |
| 只修改不 commit | 记忆丢失 | 每次修改后立即 commit |
| commit 无描述 | 无法追溯 | 必须写描述 |
| 大改不验证 | 破坏系统 | commit 前验证 |
