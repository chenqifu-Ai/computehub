## 📦 Git 记忆管理系统

**Git 不是版本控制工具，是记忆系统本身。**  
每一次 commit = 一次记忆固化。  
每一次 search = 一次记忆检索。

### 🔄 Git 操作流程

#### 一、文件查找（最高优先级，铁律）

```
收到任务 → git ls-files | grep 关键词 → 找到 → 读取操作
                              → 未找到 → git grep 关键词 → 找到 → 读取
                                                → 未找到 → find 文件系统
```

**铁律**: 查找任何文件/配置/脚本，必须先走 git 查找，禁止直接猜路径。

**命令速查**:
```bash
# 按文件名搜索
git ls-files | grep "<关键词>"

# 按文件内容搜索
git grep -l "<关键词>"

# 按提交历史搜索
git log --all --oneline --grep="<关键词>" -10

# 查看文件修改历史
git log --oneline -- <文件路径>

# 查看最近变更
git log --oneline -5
```

#### 二、记忆写入

```
学习/发现 → 写入文件 → git add → git commit → 完成
```

**Commit Message 规范**:
```
类型: 文件/主题 - 具体内容

类型: 更新 | 记录 | 教训 | 快照 | 配置 | 修复
```

**示例**:
```
教训: memory/2026-04-24.md - 先git查再操作 - weather skill路径错误
更新: HEARTBEAT.md - 系统状态更新 - 负载改善
记录: memory/2026-04-24.md - Ollama服务恢复
```

#### 三、操作分级与 Git 要求

| 级别 | 文件 | 要求 | Commit 格式 |
|------|------|------|-------------|
| 🔴 红色 | AGENTS.md, SOUL.md, SOP.md, MEMORY.md | 必须 commit + 告知老大 | `更新: <文件> - <变更> - <影响>` |
| 🟡 黄色 | TOOLS.md, HEARTBEAT.md, 配置文件 | 必须 commit | `配置: <文件> - <变更>` |
| 🟢 绿色 | memory/YYYY-MM-DD.md, 临时文件 | 累积3-5条统一 commit | `记录: <主题> - <内容>` |

### ⚠️ Git 常见错误与防范

| 错误 | 后果 | 防范 |
|------|------|------|
