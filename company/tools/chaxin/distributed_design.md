# 分布式茶信系统设计

## 架构

### 1. 本地节点
每个智能体运行自己的茶信节点：
- 本地消息队列服务
- 本地数据库
- 本地健康监控

### 2. Ollama 云端服务
- **身份认证**: API Key 作为身份标识
- **服务发现**: 注册节点，发现其他节点
- **消息中转**: 内网不通时的消息中转

### 3. 通信模式

#### 点对点 (P2P)
当双方都在线且网络可达时：
```
节点A ──直接──→ 节点B
```

#### 云端中转
当网络不通时：
```
节点A ──→ Ollama云端 ──→ 节点B
```

### 消息格式
```json

  "from": "xiaozhi",

  "body": "...",
```

## 安全考虑

1. **认证**: API Key 验证身份
2. **加密**: HTTPS 传输
3. **授权**: 只能发给已授权的节点
4. **限流**: 防止消息轰炸
